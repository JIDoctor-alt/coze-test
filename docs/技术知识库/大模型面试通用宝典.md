## 1.5 Transformer的位置编码有哪些？RoPE的原理？

### 🟡 中级 | ⭐⭐

### 面试官问法

> "Transformer是如何处理序列位置信息的？"
> "位置编码有哪些类型？各自的优缺点？"
> "RoPE旋转位置编码是什么？为什么流行？"

---

### 一句话标准答案

**位置编码让Transformer"知道"词的顺序。常见方案有：绝对位置编码（Sinusoidal/可学习）、相对位置编码和旋转位置编码（RoPE）。RoPE通过旋转Q/K向量来编码位置，让注意力计算自然地包含相对位置信息。**

---

### 深度展开

#### 1. 为什么需要位置编码？

```
Attention计算只看内容，不看位置！

"狗咬猫" vs "猫咬狗"：
- 内容词完全一样
- 顺序不同，意思相反
- 没有位置编码，输出相同 ✗
- 加上位置编码，才能区分 ✓
```

#### 2. 常见位置编码类型

```
┌────────────────────────────────────────────────────────────┐
│  1. Sinusoidal（原始Transformer）                          │
│     PE(pos, 2i)   = sin(pos / 10000^(2i/d))               │
│     优点: 无需学习，适合任意长度                             │
│     缺点: 外推能力弱                                        │
├────────────────────────────────────────────────────────────┤
│  2. 可学习位置编码                                         │
│     PE = nn.Embedding(max_len, d_model)                   │
│     优点: 可学习，效果更好                                  │
│     缺点: 需要预先设定最大长度                               │
├────────────────────────────────────────────────────────────┤
│  3. RoPE（旋转位置编码）- 当前主流                         │
│     通过旋转矩阵编码位置                                    │
│     优点: 外推性好，支持长上下文，支持线性Attention         │
└────────────────────────────────────────────────────────────┘
```

#### 3. RoPE详解

```
RoPE的核心思想：
用旋转矩阵 R_m 对 Q 和 K 进行旋转，使得：
  Attention(Q_m, K_n) 只与相对位置 (m-n) 相关

二维情况下：
┌─────────────────────────────────────────────────┐
│   R_m = │ cos(mθ)  -sin(mθ) │                   │
│          │ sin(mθ)   cos(mθ) │                   │
│                                                 │
│   Q_m = R_m · Q                                  │
│   K_n = R_n · K                                  │
│                                                 │
│   Q_m · K_n = Q · R_{m-n} · K   ← 包含相对位置！│
└─────────────────────────────────────────────────┘
```

---

### 生活比喻

```
Sinusoidal = 绝对时间编码
  "现在是下午3点15分"

RoPE = 相对时间编码
  "比刚才早了5分钟"

RoPE更灵活：
- 训练时学了"上午9点到10点"的关系
- 推理时可以处理"下午3点到4点"（类似模式）
- 不需要重新学习所有位置
```

---

### 代码佐证

```python
import torch
import torch.nn as nn
import math

class SinusoidalPositionEncoding(nn.Module):
    """Sinusoidal位置编码"""
    def __init__(self, d_model, max_len=5000):
        super().__init__()
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len, dtype=torch.float).unsqueeze(1)
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model)
        )
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        pe = pe.unsqueeze(0)
        self.register_buffer('pe', pe)
    
    def forward(self, x):
        return x + self.pe[:, :x.size(1), :]

class RoPE(nn.Module):
    """旋转位置编码"""
    def __init__(self, dim, base=10000):
        super().__init__()
        inv_freq = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
        self.register_buffer('inv_freq', inv_freq)
    
    def rotate_half(self, x):
        x1, x2 = x[..., : x.shape[-1] // 2], x[..., x.shape[-1] // 2 :]
        return torch.cat((-x2, x1), dim=-1)
    
    def forward(self, x):
        seq_len = x.size(1)
        t = torch.arange(seq_len, device=x.device).type_as(self.inv_freq)
        freqs = torch.outer(t, self.inv_freq)
        emb = torch.cat((freqs, freqs), dim=-1)
        
        cos_emb = emb.cos()
        sin_emb = emb.sin()
        
        return (x * cos_emb) + (self.rotate_half(x) * sin_emb)

def test():
    d_model = 16
    rope = RoPE(d_model)
    x = torch.randn(1, 10, d_model)
    x_rotated = rope(x)
    print(f"原始: {x.shape}")
    print(f"旋转后: {x_rotated.shape}")
    print("RoPE让Q和K在旋转后，attention计算自然包含相对位置")

if __name__ == "__main__":
    test()
```

> 1. **外推性好**：可以处理训练时没见过的更长序列
> 2. **线性Attention友好**：可以和线性Attention配合
> 3. **无需额外参数**：相比可学习位置编码效率更高
> 4. **实现简洁**：只需旋转操作"

## 1.6 Decoder-only vs Encoder-only vs Encoder-Decoder区别？

### 🟡 中级 | ⭐⭐⭐高频

### 面试官问法

> "GPT是Decoder-only，BERT是Encoder-only，T5是Encoder-Decoder，它们有什么区别？"
> "什么时候用Encoder-only，什么时候用Decoder-only？"
> "为什么现在大多数LLM都是Decoder-only架构？"

---

### 一句话标准答案

**Encoder-only（如BERT）适合理解任务，双向注意力看到全文；Decoder-only（如GPT）适合生成任务，单向注意力只能看到过去；Encoder-Decoder（如T5）兼顾两者，用Encoder看输入，用Decoder生成输出。**

---

### 深度展开

#### 1. 三种架构对比

```
┌────────────────────────────────────────────────────────────┐
│  Encoder-Only (BERT)                                        │
├────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────┐                       │
│  │ 输入: "今天天气真好"              │                       │
│  │ 注意力: 双向                     │                       │
│  │ 输出: 理解表示 / 分类标签          │                       │
│  └─────────────────────────────────┘                       │
│  特点: 看完整个序列再处理                                    │
│  适用: 分类、实体识别、句子匹配                              │
├────────────────────────────────────────────────────────────┤
│  Decoder-Only (GPT)                                        │
├────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────┐                       │
│  │ 输入: "今天"                     │                       │
│  │ 注意力: 单向（只看过去）          │                       │
│  │ 输出: "天气"                     │                       │
│  └─────────────────────────────────┘                       │
│  特点: 只能基于过去预测未来                                  │
│  适用: 文本生成、对话                                       │
├────────────────────────────────────────────────────────────┤
│  Encoder-Decoder (T5)                                      │
├────────────────────────────────────────────────────────────┤
│  ┌────────────────┬────────────────┐                      │
│  │   Encoder      │    Decoder     │                      │
│  │ 输入: 源语言   │ 输出: 目标语言  │                      │
│  │ 注意力: 双向  │ 注意力: 单向    │                      │
│  └────────────────┴────────────────┘                      │
│  特点: 理解输入，生成输出                                    │
│  适用: 翻译、摘要、问答                                      │
└────────────────────────────────────────────────────────────┘
```

#### 2. 为什么Decoder-only成为主流？

```
GPT系列的成功证明了：
1. 足够大的 Decoder-only 模型可以完成一切任务
2. 预训练任务简单（Next Token Prediction）
3. 训练稳定，容易scale up
4. 推理可以用KV Cache优化

Encoder-Decoder的问题：
- 两套参数，训练复杂
- 需要设计encoder和decoder的联合训练策略
- 推理时encoder也要计算
```

#### 3. 注意力Mask的区别

```python
# Encoder 双向上下文注意力
# 每个位置可以看到所有其他位置
┌─────────────────────┐
│ ✓ ✓ ✓ ✓ ✓ ✓ ✓ ✓ │  # 所有位置可见
│ ✓ ✓ ✓ ✓ ✓ ✓ ✓ ✓ │
│ ✓ ✓ ✓ ✓ ✓ ✓ ✓ ✓ │
│ ✓ ✓ ✓ ✓ ✓ ✓ ✓ ✓ │
└─────────────────────┘

# Decoder 单向注意力（带Mask）
# 每个位置只能看到过去
┌─────────────────────┐
│ ✓ ✗ ✗ ✗ ✗ ✗ ✗ ✗ │  # 只能看自己
│ ✓ ✓ ✗ ✗ ✗ ✗ ✗ ✗ │  # 能看前两个
│ ✓ ✓ ✓ ✗ ✗ ✗ ✗ ✗ │
│ ✓ ✓ ✓ ✓ ✗ ✗ ✗ ✗ │
└─────────────────────┘
```

---

### 生活比喻

```
餐厅场景：

Encoder-only (BERT)：
  就像看菜单：需要看完整个菜单才能决定点什么
  可以看到所有选项，然后做选择

Decoder-only (GPT)：
  就像点餐：服务员一道一道上，你只能基于已上的菜决定下一道
  不能"预知"后面会上什么

Encoder-Decoder：
  就像翻译：服务员先把菜单翻译成英文（Encoder）
  然后你用英文菜单点餐（Decoder）
```

### 代码佐证

```python
import torch

def create_masks(seq_len, mode):
    """创建不同类型的注意力掩码"""
    if mode == 'encoder':
        # Encoder: 双向注意力
        mask = torch.ones(seq_len, seq_len)
        return mask
    elif mode == 'decoder':
        # Decoder: 单向注意力（Causal Mask）
        mask = torch.tril(torch.ones(seq_len, seq_len))
        return mask

def demonstrate_causal_mask():
    """演示Causal Mask的重要性"""
    seq_len = 5
    
    print("=" * 50)
    print("Encoder Mask (双向注意力):")
    encoder_mask = create_masks(seq_len, 'encoder')
    print(encoder_mask)
    
    print("\n" + "=" * 50)
    print("Decoder Mask (单向/Causal Mask):")
    decoder_mask = create_masks(seq_len, 'decoder')
    print(decoder_mask)
    
    print("\n" + "=" * 50)
    print("为什么需要Causal Mask?")
    print("如果用双向注意力解码，会"偷看"答案！")

if __name__ == "__main__":
    demonstrate_causal_mask()
```

> 1. **只做理解（分类、NER等）**：用Encoder-only（BERT系列）

> 3. **理解+生成（翻译、摘要）**：用Encoder-Decoder（T5、BART）
> 4. **通用对话助手**：Decoder-only（few-shot做各种任务）"

## 1.7 Flash Attention原理？为什么快？

### 🔴 高级 | ⭐⭐

### 面试官问法

> "Flash Attention是什么？解决了什么问题？"
> "为什么Flash Attention比标准Attention快？"

---

### 一句话标准答案

**Flash Attention通过"分块计算"和"算力融合"技术，避免了标准Attention中需要显存放整个注意力矩阵的问题。它利用GPU的SRAM高速缓存，按块读写数据，大大减少了HBM（显存）访问，从而实现2-4倍加速和显存从O(n²)降到O(n)。**

---

### 深度展开

#### 1. 标准Attention的问题

```
标准Attention的计算需要显存放整个注意力矩阵：O(n²)显存
对于 n=32768: 12GB显存 ✗
```

#### 2. Flash Attention的核心思想

```
分块计算 (Tiling)：
- 不一次性加载所有 K, V
- 分块处理，每块结果累加
- 增量计算 softmax

关键洞察：
- softmax 可以增量计算（在线softmax）
- 不需要一次性看到所有 S_ij
```

#### 3. 性能对比

```
序列长度 n=4096:
标准Attention显存: ~192MB
Flash Attention显存: ~20MB
节省: 10x

序列长度 n=32768:
标准Attention: 可能OOM
Flash Attention: ~100MB
```

---

### 代码佐证

```python
import torch
import torch.nn.functional as F
import math

def standard_attention(Q, K, V):
    """标准Attention实现（显存占用大）"""
    d_k = Q.shape[-1]
    scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
    attn_weights = F.softmax(scores, dim=-1)
    output = torch.matmul(attn_weights, V)
    return output, attn_weights

def compare_memory():
    """对比显存占用"""
    seq_len = 2048
    nhead = 12
    d_k = 64
    
    # 标准Attention的显存需求
    standard_mem = 2 * seq_len * seq_len * 4 / (1024**2)
    
    # Flash Attention的显存需求
    flash_mem = seq_len * d_k * nhead * 4 * 2 / (1024**2)
    
    print(f"序列长度: {seq_len}")
    print(f"标准Attention显存: {standard_mem:.2f} MB")
    print(f"Flash Attention显存: {flash_mem:.2f} MB")
    print(f"节省: {standard_mem/flash_mem:.1f}x")

if __name__ == "__main__":
    compare_memory()
```

---

### 加分回答

> "Flash Attention有几个版本：
> 1. **Flash Attention v1**：基础的tiling和recompute
> 2. **Flash Attention v2**：进一步优化工作线程调度
> 3. **Flash Attention v3**：引入异步执行"

---

### 💡 一句话总结

![顿悟](https://www.coze.cn/s/PtHn-lhw7Dg/)

**Flash Attention的核心是"分块计算+在线softmax"，通过减少HBM访问次数实现加速！**

## 1.8 GQA/MQA与MHA的区别？

### 🔴 高级 | ⭐⭐

### 面试官问法

> "GQA是什么？和MHA有什么区别？"
> "MQA如何减少KV Cache的？"

---

### 一句话标准答案

**MHA每个头有独立的Q/K/V；MQA让所有头共享K和V；GQA折中方案，把头分成G组，每组共享K/V。GQA/MQA大幅减少KV Cache，提升推理效率。**

---

### 深度展开

```
MHA: 32头=32份KV
GQA: 8头=8份KV (LLaMA2采用)
MQA: 1份KV

LLaMA-7B KV Cache:
- MHA: 2048 tokens需要64MB
- GQA: 2048 tokens需要16MB
- 减少4倍！
```

---

### 代码佐证

```python
import torch
import torch.nn as nn

class GQA(nn.Module):
    """分组查询注意力 (GQA)"""
    def __init__(self, d_model, nhead, n_kv_head):
        super().__init__()
        self.nhead = nhead
        self.n_kv_head = n_kv_head
        self.d_k = d_model // nhead
        
        self.W_Q = nn.Linear(d_model, d_model)
        self.W_K = nn.Linear(d_model, self.n_kv_head * self.d_k)
        self.W_V = nn.Linear(d_model, self.n_kv_head * self.d_k)
        self.W_O = nn.Linear(d_model, d_model)
    
    def forward(self, x):
        B, N, _ = x.shape
        
        Q = self.W_Q(x).view(B, N, self.nhead, self.d_k)
        K = self.W_K(x).view(B, N, self.n_kv_head, self.d_k)
        V = self.W_V(x).view(B, N, self.n_kv_head, self.d_k)
        
        # 扩展K和V到所有Q头
        if self.n_kv_head != self.nhead:
            K = torch.repeat_interleave(K, self.nhead // self.n_kv_head, dim=2)
            V = torch.repeat_interleave(V, self.nhead // self.n_kv_head, dim=2)
        
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5)
        attn = torch.softmax(scores, dim=-1)
        output = torch.matmul(attn, V)
        
        return self.W_O(output.transpose(1, 2).contiguous().view(B, N, -1))

def compare_kv_cache():
    """对比KV Cache大小"""
    seq_len = 2048
    d_model = 4096
    nhead = 32
    d_k = d_model // nhead
    
    mha_kv_cache = 2 * nhead * seq_len * d_k * 4 / (1024**2)
    n_kv = 8
```

## 1.9 KV Cache原理与优化？

### 🔴 高级 | ⭐⭐

### 面试官问法

> "什么是KV Cache？有什么用？"
> "KV Cache占用多少显存？"

---

### 一句话标准答案

**KV Cache是缓存已计算的Key和Value，避免重复计算。在自回归生成时，通过缓存KV可以让计算量从O(n²)降到O(n)，但会占用大量显存。**

---

### 深度展开

```
没有KV Cache:
Token N: 需要重新计算前N-1个token的所有K,V
时间复杂度: O(n²)

有KV Cache:
Token N: 只计算当前token的K,V，用缓存的K,V
时间复杂度: O(n)
空间复杂度: O(n²) ← 存储所有历史K,V
```

---

### 代码佐证

```python
import torch
import torch.nn as nn

class AttentionWithKVCache(nn.Module):
    """带KV Cache的自注意力"""
    def __init__(self, d_model, nhead):
        super().__init__()
        self.d_model = d_model
        self.nhead = nhead
        self.d_k = d_model // nhead
        
        self.W_Q = nn.Linear(d_model, d_model)
        self.W_K = nn.Linear(d_model, d_model)
        self.W_V = nn.Linear(d_model, d_model)
        self.W_O = nn.Linear(d_model, d_model)
    
    def forward(self, x, kv_cache=None):
        B, N, _ = x.shape
        
        Q = self.W_Q(x)
        K = self.W_K(x)
        V = self.W_V(x)
        
        if kv_cache is not None:
            cache_K, cache_V = kv_cache
            K = torch.cat([cache_K, K], dim=1)
            V = torch.cat([cache_V, V], dim=1)
        
        Q = Q.view(B, -1, self.nhead, self.d_k).transpose(1, 2)
        K = K.view(B, -1, self.nhead, self.d_k).transpose(1, 2)
        V = V.view(B, -1, self.nhead, self.d_k).transpose(1, 2)
        
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5)
        attn = torch.softmax(scores, dim=-1)
        output = torch.matmul(attn, V)
        
        new_cache = (K, V) if kv_cache is not None else None
        
        return self.W_O(output.transpose(1, 2).reshape(B, -1, self.d_model)), new_cache

def kv_cache_size_demo():
    """计算KV Cache大小"""
    nhead = 32
    d_model = 4096
    d_k = d_model // nhead
    
    per_token_bytes = nhead * d_k * 2 * 4
    per_token_kb = per_token_bytes / 1024
    
    print("LLaMA-7B KV Cache分析:")
    print(f"每个token的KV: {per_token_kb:.2f} KB")
    print(f"2048 tokens: {2048 * per_token_kb / 1024:.2f} MB")

if __name__ == "__main__":
    kv_cache_size_demo()
```

```python
import torch
import torch.nn as nn

class PreTrainingDataset:
    """预训练数据准备"""
    def __init__(self, texts, tokenizer, max_length=2048):
        self.tokenizer = tokenizer
        self.max_length = max_length
        self.texts = texts
    
    def __getitem__(self, idx):
        text = self.texts[idx]
        # Tokenize
        tokens = self.tokenizer.encode(text)
        
        # Next Token Prediction格式
        # 输入: tokens[:-1]
        # 目标: tokens[1:]
        input_ids = tokens[:self.max_length]
        labels = tokens[1:self.max_length]
        
        # Padding
        padding_len = self.max_length - len(input_ids)
        input_ids += [self.tokenizer.pad_id] * padding_len
        labels += [-100] * padding_len  # -100表示忽略
        
        return torch.tensor(input_ids), torch.tensor(labels)

class SFTDataset:
    """SFT微调数据准备"""
    def __init__(self, conversations, tokenizer, max_length=2048):
        self.tokenizer = tokenizer
        self.max_length = max_length
        self.conversations = conversations
    
    def __getitem__(self, idx):
        conv = self.conversations[idx]
        
        # 构建prompt格式
        text = ""
        for msg in conv:
            if msg["role"] == "user":
                text += f"<human>{msg['content']}<bot>"
            else:
                text += f"{msg['content']}"
        
        tokens = self.tokenizer.encode(text)
        input_ids = tokens[:self.max_length]
        
        # 对于SFT，labels和input_ids相同
        labels = input_ids.copy()
        
        # 填充
        padding_len = self.max_length - len(input_ids)
        input_ids += [self.tokenizer.pad_id] * padding_len
        labels += [-100] * padding_len
        
        return torch.tensor(input_ids), torch.tensor(labels)

class RLHFTraining:
    """简化的RLHF训练流程"""
    def __init__(self, model, ref_model, reward_model):
        self.model = model
        self.ref_model = ref_model
        self.reward_model = reward_model
        self.gamma = 0.95  # 折扣因子
    
    def compute_rewards(self, prompts, responses):
        """计算奖励分数"""
        combined = [p + r for p, r in zip(prompts, responses)]
        rewards = self.reward_model(combined)
        return rewards
    
    def compute_kl_loss(self, policy_logps, ref_logps):
        """计算KL散度损失"""
        kl = torch.exp(ref_logps - policy_logps)
        return kl.mean()
    
    def ppo_update(self, batch_prompts, batch_responses, old_logps):
        """
        PPO更新步骤
        
        简化版，实际需要:
        1. 计算新旧策略的概率比
        2. 裁剪目标函数
        3. 加入KL约束
        """
        # 获取当前策略的log prob
        new_logps = self.model.get_logps(batch_prompts, batch_responses)
        
        # 获取奖励
        rewards = self.compute_rewards(batch_prompts, batch_responses)
        
        # 计算KL损失
        with torch.no_grad():
            ref_logps = self.ref_model.get_logps(batch_prompts, batch_responses)
        kl_loss = self.compute_kl_loss(new_logps, ref_logps)
        
        # 简化的PPO损失
        policy_loss = -(new_logps * rewards).mean()
        total_loss = policy_loss + 0.01 * kl_loss
        
        return total_loss
```

## 2.2 SFT和RLHF的区别？

### 🟢 基础 | ⭐⭐⭐高频

### 面试官问法

> "SFT和RLHF有什么区别？"
> "为什么需要RLHF，SFT不够吗？"
> "DPO是什么？和PPO有什么关系？"

---

### 一句话标准答案

**SFT是有监督学习，直接让模型模仿正确答案；RLHF是强化学习，用奖励信号引导模型生成更好回答。SFT告诉模型"标准答案是什么"，RLHF告诉模型"什么样的回答更好"。**

---

### 深度展开

#### 1. SFT vs RLHF对比

```
┌────────────────────────────────────────────────────────────┐
│  SFT (Supervised Fine-tuning)                             │
├────────────────────────────────────────────────────────────┤
│  方法: 有监督学习                                          │
│  数据: (输入, 标准输出) 对                                  │
│  Loss: CrossEntropy(预测, 标准答案)                         │
│                                                            │
│  优点:                                                     │
│  - 简单直接，训练稳定                                      │
│  - 适合有明确答案的任务                                     │
│                                                            │
│  缺点:                                                     │
│  - 只能模仿，无法超越数据                                   │
│  - 难以处理主观/模糊的偏好                                  │
│  - 难以处理开放式生成任务                                   │
│                                                            │
├────────────────────────────────────────────────────────────┤
│  RLHF (Reinforcement Learning from Human Feedback)        │
├────────────────────────────────────────────────────────────┤
│  方法: 强化学习                                            │
│  数据: 偏好数据 (哪个回答更好)                              │
│  信号: Reward Model                                        │
│                                                            │
│  优点:                                                     │
│  - 可以学习隐式偏好                                        │
│  - 可以超越训练数据的质量                                   │
│  - 擅长处理主观/开放式任务                                  │
│                                                            │
│  缺点:                                                     │
│  - 训练复杂，需要PPO等算法                                 │
│  - 容易reward hacking                                       │
│  - 需要大量人工标注                                        │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

#### 2. 为什么需要RLHF？

```
SFT的局限性：
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  问: "如何评价人工智能？                                    │
│                                                            │
│  回答A (SFT数据):                                          │
│  "人工智能是..."  # 可能有语法错误                          │
│                                                            │
│  回答B (SFT数据):                                          │
│  "人工智能（AI）是一种..."  # 啰嗦但正确                     │
│                                                            │
│  SFT只能模仿:                                              │
│  - 如果数据质量一般，模型学会的也是一般的回答               │
│  - 无法知道哪个回答"更好"                                  │
│                                                            │
└────────────────────────────────────────────────────────────┘

RLHF的优势：
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  人类偏好标注:                                              │
│  回答B 比 回答A 更好 ✓                                      │
│                                                            │
│  RLHF学习:                                                 │
│  - 什么特征让回答"更好"                                    │
│  - 有帮助性、诚实、无害                                    │
│  - 可以生成比训练数据更好的回答！                          │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

#### 3. DPO（Direct Preference Optimization）

```
传统RLHF = PPO + Reward Model
DPO = 直接用偏好数据优化

PPO的问题:
- 需要训练Reward Model
- 需要PPO强化学习
- 训练复杂，容易不稳定

DPO的思路:
- 不需要单独的Reward Model
- 直接用偏好数据优化
- 简化训练流程

数学形式:
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  PPO Loss:                                                │
│  L = -E[r(x,y)] + β*KL(π || π_ref)                       │
│                                                            │
│  DPO Loss:                                                │
│  L = -E[(x,y_w,y_l)][log σ(r(x,y_w) - r(x,y_l))]         │
│                                                            │
│  其中:                                                    │
│  - y_w 是被选中的回答                                      │
│  - y_l 是被拒绝的回答                                      │
│  - σ 是sigmoid函数                                        │
│                                                            │
│  DPO等价于在隐式Reward下优化策略                           │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

---

### 生活比喻

```
🍳 做饭比喻：

SFT = 照着菜谱做饭
- 菜谱告诉你每一步怎么做
- 你严格按照菜谱执行
- 问题：菜谱不好，做出来也不好

RLHF = 米其林评委指导
- 评委品尝后告诉你哪里需要改进
- 你慢慢调整厨艺
- 可以超越菜谱，做出更好吃的菜

DPO = 评委给你对比菜谱
- "这份比那份好"
- 你自己琢磨哪里不同
- 不需要评委全程陪着
```

```python
import torch
import torch.nn.functional as F

class SFTTraining:
    """SFT训练"""
    def __init__(self, model):
        self.model = model
    
    def compute_loss(self, input_ids, labels):
        """标准交叉熵损失"""
        outputs = self.model(input_ids)
        
        # Shift for next token prediction
        logits = outputs.logits[:, :-1, :]
        targets = labels[:, 1:]
        
        # 忽略padding和某些位置
        loss = F.cross_entropy(
            logits.reshape(-1, logits.size(-1)),
            targets.reshape(-1),
            ignore_index=-100
        )
        return loss

class DPOTraining:
    """DPO训练"""
    def __init__(self, model, ref_model, beta=0.1):
        self.model = model
        self.ref_model = ref_model
        self.beta = beta
    
    def compute_dpo_loss(self, prompt_ids, chosen_ids, rejected_ids):
        """
        DPO Loss
        
        prompt_ids: 问题
        chosen_ids: 被选中的回答
        rejected_ids: 被拒绝的回答
        """
        # 获取策略的log prob
        policy_chosen_logps = self.model.get_logps(prompt_ids, chosen_ids)
        policy_rejected_logps = self.model.get_logps(prompt_ids, rejected_ids)
        
        # 获取参考模型的log prob
        with torch.no_grad():
            ref_chosen_logps = self.ref_model.get_logps(prompt_ids, chosen_ids)
            ref_rejected_logps = self.ref_model.get_logps(prompt_ids, rejected_ids)
        
        # 计算log ratio
        chosen_log_ratio = policy_chosen_logps - ref_chosen_logps
        rejected_log_ratio = policy_rejected_logps - ref_rejected_logps
        
        # DPO损失
        # L = -log σ( (log π(y_w|x) - log π_ref(y_w|x)) - (log π(y_l|x) - log π_ref(y_l|x)) )
        logits = chosen_log_ratio - rejected_log_ratio
        loss = -F.logsigmoid(logits).mean()
        
        # 也可以加上KL项作为正则
        kl_loss = (chosen_log_ratio + rejected_log_ratio).mean()
        
        return loss + self.beta * kl_loss

def compare_methods():
    """对比SFT和RLHF"""
    print("=" * 60)
    print("SFT vs RLHF/DPO 对比")
    print("=" * 60)

    print("数据: (问题, 答案)")
    print("Loss: CrossEntropy(预测, 标准答案)")
    print("特点: 简单稳定，但只能模仿")
    
    print("\n【PPO RLHF】")
    print("数据: 偏好对 (哪个更好)")
    print("Loss: PPO Loss + KL约束")
    print("特点: 效果好，但训练复杂")
    
    print("\n【DPO】")
    print("数据: 偏好对 (哪个更好)")
    print("Loss: 直接用偏好数据")
    print("特点: 简化训练，效果接近PPO")
    
    print("\n适用场景:")
    print("- 有标准答案: SFT")
    print("- 主观偏好/开放式: RLHF/DPO")
    print("- 资源有限: DPO")

    compare_methods()
```

## 2.3 LoRA原理？为什么能省显存？

### 🟡 中级 | ⭐⭐⭐高频

### 面试官问法

> "LoRA是什么？为什么能大幅减少显存？"
> "LoRA的秩(r)怎么选？"
> "LoRA和全参数微调效果差多少？"

---

### 一句话标准答案

**LoRA通过低秩分解冻结预训练权重，只训练A和B两个低秩矩阵。假设原始权重W₀是d×d，全参数训练需要更新d²个参数，LoRA只需要2×r×d个参数（r<<d），显存大幅减少。**

---

### 深度展开

#### 1. LoRA的核心思想

```
全参数微调：
┌────────────────────────────────────────────────────────────┐
│  更新: ΔW = W - W₀                                        │
│  存储: W₀ + ΔW ≈ W                                        │
│                                                            │
│  W₀: d×d (冻结)                                          │
│  ΔW: d×d (需要更新和存储)                                 │
│                                                            │
│  对于d=4096的权重:                                         │
│  参数量: 4096 × 4096 = 16M                                │
│  显存: 16M × 4bytes × 2 ≈ 128MB (参数 + 梯度)             │
│                                                            │
└────────────────────────────────────────────────────────────┘

LoRA微调：
┌────────────────────────────────────────────────────────────┐
│  冻结W₀，只更新A和B                                         │
│                                                            │
│  W = W₀ + ΔW = W₀ + BA                                    │
│  其中: A是d×r，B是r×d，r << d                             │
│                                                            │
│  对于r=8的LoRA:                                           │
│  A: 4096 × 8 = 32K                                        │
│  B: 8 × 4096 = 32K                                        │
│  总参数量: 64K ≈ 原始的0.4%                                │
│  显存: 64K × 4bytes × 2 ≈ 0.5MB                          │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

#### 2. 为什么低秩有效？

```
直觉解释：
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  预训练模型的知识分布在高维空间                             │
│  但微调时需要更新的方向可能是低维的                         │
│                                                            │
│  原因：                                                    │
│  1. 任务相关的信息可能只涉及少数方向                       │
│  2. 大部分预训练知识仍然有用，只需要小调整                 │
│  3. 低秩足以捕捉任务特定的适应方向                        │
│                                                            │
│  数学保证：                                                │
│  如果ΔW的秩为r，那么用r个奇异向量就可以近似               │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

#### 3. LoRA的不同变体

```
┌────────────────────────────────────────────────────────────┐
│  LoRA: W = W₀ + BA                                        │
│  只有attention的Q和V使用LoRA                               │
│                                                            │
│  LoRA+: 引入学习率因子                                      │
│  W = W₀ + α/r · BA                                       │
│                                                            │
│  AdaLoRA: 自适应秩                                         │
│  不同层分配不同的r                                        │
│                                                            │
│  QLoRA: 量化+LoRA                                         │
│  W₀用4-bit量化，A和B用16-bit                              │
│  可以微调65B参数的模型！                                   │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

---

### 生活比喻

```
🎨 画画比喻：

全参数微调 = 重新画一幅画
- 需要画布、颜料全部重新准备
- 画师需要记住整幅画
- 费时费力

LoRA = 在原画上局部修改
- 原画（W₀）不动
- 用少量颜料（A和B）做局部调整
- 只需要记住改动的地方
- 省颜料省记忆

颜色比喻：
- 原始颜色有4096种（d=4096）
- LoRA只需要8种基础色（r=8）
- 8种基础色可以混合出各种想要的颜色
```

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class LoRALinear(nn.Module):
    """LoRA线性层"""
    def __init__(self, in_features, out_features, rank=4, alpha=1):
        super().__init__()
        self.in_features = in_features
        self.out_features = out_features
        self.rank = rank
        self.alpha = alpha
        
        # 冻结原始权重
        self.weight = nn.Parameter(
            torch.randn(out_features, in_features), 
            requires_grad=False
        )
        self.bias = nn.Parameter(
            torch.zeros(out_features), 
            requires_grad=False
        )
        
        # LoRA参数
        self.lora_A = nn.Parameter(torch.randn(rank, in_features) * 0.01)
        self.lora_B = nn.Parameter(torch.zeros(out_features, rank))
        
        # 缩放因子
        self.scaling = alpha / rank
    
    def forward(self, x):
        # 原始输出 + LoRA输出
        return F.linear(x, self.weight, self.bias) + \
               x @ self.lora_A.T @ self.lora_B.T * self.scaling
    
    def extra_repr(self):
        return f'in_features={self.in_features}, out_features={self.out_features}, rank={self.rank}'

class LoRAModel(nn.Module):
    """带LoRA的模型"""
    def __init__(self, base_model, rank=4, alpha=1, target_modules=['q_proj', 'v_proj']):
        super().__init__()
        self.base_model = base_model
        self.rank = rank
        self.alpha = alpha
        
        # 替换指定层为LoRA层
        self.lora_layers = nn.ModuleDict()
        
        for name, module in base_model.named_modules():
            if any(target in name for target in target_modules):
                if isinstance(module, nn.Linear):
                    lora_layer = LoRALinear(
                        module.in_features, 
                        module.out_features, 
                        rank=rank,
                        alpha=alpha
                    )
                    # 复制原始权重
                    lora_layer.weight.data = module.weight.data.clone()
                    if module.bias is not None:
                        lora_layer.bias.data = module.bias.data.clone()
                    
                    self.lora_layers[name] = lora_layer
    
    def forward(self, x):
        # 使用修改后的模型
        return self.base_model(x)
    
    def get_trainable_params(self):
        """获取可训练参数"""
        trainable = []
        for layer in self.lora_layers.values():
            trainable.extend([layer.lora_A, layer.lora_B])
        return trainable
    
    def print_trainable_params(self):
        """打印可训练参数统计"""
        total_params = sum(p.numel() for p in self.base_model.parameters())
        trainable_params = sum(p.numel() for p in self.get_trainable_params())
        
        print(f"总参数量: {total_params:,}")
        print(f"可训练参数量: {trainable_params:,}")
        print(f"可训练比例: {100*trainable_params/total_params:.2f}%")

def compare_memory():
    """对比显存占用"""
    print("=" * 60)
    print("显存对比 (以d=4096, batch=1为例)")
    print("=" * 60)
    
    d = 4096
    seq_len = 2048
    precision = 2  # float16 = 2 bytes
    
    # 全参数微调
    full_params = d * d
    full_memory = full_params * precision * 3  # weight + gradient + optimizer
    print(f"全参数微调:")
    print(f"  参数量: {full_params:,} ({full_params/1e6:.1f}M)")
    print(f"  显存(weight+grad+opt): {full_memory/1e9:.2f} GB")
    
    # LoRA (r=8)
    r = 8
    lora_params = 2 * r * d
    lora_memory = lora_params * precision * 3
    print(f"\nLoRA (r={r}):")
    print(f"  参数量: {lora_params:,} ({lora_params/1e3:.1f}K)")
    print(f"  显存(weight+grad+opt): {lora_memory/1e6:.2f} MB")
    print(f"  显存节省: {full_memory/lora_memory:.0f}x")
    
    # LoRA (r=64)
    r = 64
    lora_params = 2 * r * d
    lora_memory = lora_params * precision * 3
    print(f"\nLoRA (r={r}):")
    print(f"  参数量: {lora_params:,} ({lora_params/1e6:.2f}M)")
    print(f"  显存(weight+grad+opt): {lora_memory/1e6:.2f} MB")
    print(f"  显存节省: {full_memory/lora_memory:.0f}x")
```

## 2.4 QLoRA比LoRA多了什么？4-bit量化怎么做？

### 🟡 中级 | ⭐⭐⭐高频

### 面试官问法

> "QLoRA是什么？和普通LoRA有什么区别？"
> "4-bit量化是什么原理？精度损失多少？"
> "QLoRA如何做到在单卡微调65B模型？"

---

### 一句话标准答案

**QLoRA = Quantized + LoRA。核心创新是将预训练权重量化为4-bit（NF4格式），同时用16-bit的LoRA更新。4-bit存储权重，16-bit计算前向，兼顾显存节省和精度。**

---

### 深度展开

#### 1. QLoRA的核心技术

```
┌────────────────────────────────────────────────────────────┐
│  QLoRA = 4-bit NormalFloat4 + LoRA + Double Quantization   │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  【1. 4-bit NormalFloat4 (NF4)】                           │
│  ┌─────────────────────────────────────────────────────┐ │
│  │  量化: 将32-bit/16-bit映射到4-bit                     │ │
│  │                                                      │ │
│  │  普通4-bit: 均匀分布 [-8, 7]                          │ │
│  │  NF4: 非均匀分布，更适合正态分布的权重                │ │
│  │                                                      │ │
│  │  优势: 量化误差小，精度损失少                        │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                            │
│  【2. 分块量化 (Block-wise Quantization)】                │
│  ┌─────────────────────────────────────────────────────┐ │
│  │  不对整个tensor量化                                   │ │
│  │  将tensor分成小块（如64个元素）                      │ │
│  │  每块独立量化                                        │ │
│  │                                                      │ │
│  │  好处: 减少极端值的影响                              │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                            │
│  【3. Double Quantization】                                │
│  ┌─────────────────────────────────────────────────────┐ │
│  │  对量化参数本身再次量化                               │ │
│  │  量化系数用8-bit存储                                 │ │
│  │  进一步节省显存                                      │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

#### 2. 为什么需要NF4？

```
量化误差对比：
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  原始权重分布（正态分布）:                                  │
│  ████████████                                              │
│                                                            │
│  均匀4-bit量化:                                            │
│  |---|---|---|---|---|---|---|---|                          │
│  -8  -6  -4  -2   0   2   4   6   8                        │
│  ↑                                              ↑          │
│  很多值在这个区间                   很多值也在这个区间     │
│  量化误差大                        量化误差大              │
│                                                            │
│  NF4量化:                                                  │
│  |---|---|---|---|---|---|---|---|                          │
│  量化中心更密                      量化中心更疏           │
│  匹配正态分布                      匹配正态分布            │
│  量化误差小 ✓                                             │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

#### 3. QLoRA的显存分析

```
以LLaMA-65B为例：
┌────────────────────────────────────────────────────────────┐
│  全精度 (FP32): 65B × 4 bytes = 260GB ✗                   │
│  半精度 (FP16): 65B × 2 bytes = 130GB ✗                  │
│                                                            │
│  QLoRA:                                                    │
│  - 权重NF4: 65B × 0.5 bytes = 32.5GB                      │
│  - LoRA参数: ~0.1% × 65B × 2 bytes = 0.13GB              │
│  - 梯度: 只对LoRA部分求导，~0.13GB                         │
│  - 优化器状态: Adam需要2个状态，~0.52GB                    │
│  - KV Cache和其他: ~10GB                                 │
│                                                            │
│  总计: ~45GB                                              │
│  可以用4×80GB A100微调！✓                                 │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 代码佐证

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

def find_quantile(data, quantile):
    """找到分位数"""
    return torch.quantile(data.flatten().float(), quantile)

def create_nf4_quantization():
    """创建NF4量化表"""
    # NF4有16个量化值（非均匀分布）
    # 基于正态分布的分位数
    quantiles = torch.linspace(0, 1, 17)[:16]
    
    # 标准正态分布的分位数
    import math
    inv_norm = torch.tensor([
        find_quantile(torch.randn(10000), q.item()) 
        for q in quantiles
    ])
    
    return inv_norm

class NF4Quantizer:
    """NF4量化器"""
    def __init__(self, block_size=64):
        self.block_size = block_size
        self.quantiles = create_nf4_quantization()
    
    def quantize(self, weights):
        """量化权重到NF4"""
        original_shape = weights.shape
        num_blocks = weights.numel() // self.block_size
        
        # 分块量化
        quantized_blocks = []
        scale_blocks = []
        
        for i in range(num_blocks):
            block = weights.flatten()[i*self.block_size:(i+1)*self.block_size]
            
            # 计算缩放因子
            abs_max = block.abs().max()
            scale = abs_max / (self.quantiles[-1] + 1e-8)
            
            # 量化
            normalized = block / scale
            quantized = torch.searchsorted(self.quantiles, normalized)
            
            quantized_blocks.append(quantized.to(torch.uint8))
            scale_blocks.append(scale)
        
        return quantized_blocks, scale_blocks
    
    def dequantize(self, quantized_blocks, scale_blocks):
        """反量化"""
        dequantized = []
        for q, s in zip(quantized_blocks, scale_blocks):
            dequantized.append(self.quantiles[q] * s)
        return torch.cat(dequantized)

class QLoRALinear(nn.Module):
    """QLoRA线性层"""
    def __init__(self, in_features, out_features, rank=4, alpha=1):
        super().__init__()
        self.in_features = in_features
        self.out_features = out_features
        self.rank = rank
        self.alpha = alpha
        self.scaling = alpha / rank
        
        # 原始权重（NF4量化）
        self.weight = nn.Parameter(
            torch.randn(out_features, in_features),
            requires_grad=False
        )
        self.quantizer = NF4Quantizer()
        
        # LoRA参数（FP16）
        self.lora_A = nn.Parameter(
            torch.randn(rank, in_features) * 0.01
        )
        self.lora_B = nn.Parameter(
            torch.zeros(out_features, rank)
        )
    
    def forward(self, x):
        # 反量化权重
        weight_dequant = self.weight.float()
        
        # 计算输出
        base_output = F.linear(x, weight_dequant)
        lora_output = x @ self.lora_A.T @ self.lora_B.T * self.scaling
        
        return base_output + lora_output
```

## 2.5 DPO和PPO的区别？

### 🟡 中级 | ⭐⭐

### 面试官问法

> "DPO和PPO有什么区别？"
> "为什么DPO更简单？"
> "实际应该选哪个？"

---

### 一句话标准答案

**PPO是传统RLHF方法，需要训练Reward Model再用PPO优化策略，流程复杂；DPO直接用偏好数据优化，绕过Reward Model，简化了训练流程。DPO本质上是PPO的数学等价变体。**

---

### 深度展开

```
┌────────────────────────────────────────────────────────────┐
│  PPO RLHF流程                                              │
├────────────────────────────────────────────────────────────┤
│  Step 1: 训练Reward Model (RM)                            │
│  Step 2: 用RM提供reward，PPO优化LLM                       │
│                                                            │
│  Loss = -E[r(x,y)] + β*KL(π || π_ref)                   │
│                                                            │
│  问题：需要3个模型（LLM, RM, Ref）                       │
│                                                            │
├────────────────────────────────────────────────────────────┤
│  DPO流程                                                  │
├────────────────────────────────────────────────────────────┤
│  直接用偏好数据优化                                        │
│                                                            │
│  Loss = -E[log σ( r_w - r_l )]                          │
│                                                            │
│  其中r_w是被选中的reward，r_l是被拒绝的                   │
│                                                            │
│  只用1个模型，不需要RM                                     │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 代码佐证

```python
import torch
import torch.nn.functional as F

class PPOAgent:
    """PPO RLHF"""
    def __init__(self, policy_model, ref_model, reward_model):
        self.policy = policy_model
        self.ref = ref_model
        self.reward = reward_model
        self.clip_ratio = 0.2
    
    def ppo_loss(self, prompt_ids, response_ids, old_log_probs):
        """PPO损失"""
        # 当前策略的log prob
        new_log_probs = self.policy.get_log_probs(prompt_ids, response_ids)
        
        # Ratio
        ratio = torch.exp(new_log_probs - old_log_probs)
        
        # Reward
        rewards = self.reward(prompt_ids, response_ids)
        
        # PPO clip
        surr1 = ratio * rewards
        surr2 = torch.clamp(ratio, 1-self.clip_ratio, 1+self.clip_ratio) * rewards
        
        # KL penalty
        kl = self.compute_kl(self.policy, self.ref)
        
        return -torch.min(surr1, surr2) + 0.01 * kl

class DPOAgent:
    """DPO"""
    def __init__(self, policy_model, ref_model, beta=0.1):
        self.policy = policy_model
        self.ref = ref_model
        self.beta = beta
    
    def dpo_loss(self, prompt_ids, chosen_ids, rejected_ids):
        """DPO损失"""
        # 策略的log prob
        policy_chosen = self.policy.get_log_probs(prompt_ids, chosen_ids)
        policy_rejected = self.policy.get_log_probs(prompt_ids, rejected_ids)
        
        # 参考模型的log prob
        with torch.no_grad():
            ref_chosen = self.ref.get_log_probs(prompt_ids, chosen_ids)
            ref_rejected = self.ref.get_log_probs(prompt_ids, rejected_ids)
        
        # Log ratio
        chosen_log_ratio = (policy_chosen - ref_chosen) / self.beta
        rejected_log_ratio = (policy_rejected - ref_rejected) / self.beta
        
        # DPO loss
        logits = chosen_log_ratio - rejected_log_ratio
        loss = -F.logsigmoid(logits).mean()
        
        return loss

def compare_methods():
    print("=" * 60)
    print("PPO vs DPO 对比")
    print("=" * 60)
    print("\nPPO:")
    print("- 需要3个模型: Policy + Reward + Reference")
    print("- 需要PPO算法调参")
    print("- 训练可能不稳定")
    print("\nDPO:")
    print("- 只需要2个模型: Policy + Reference")
    print("- 直接用偏好数据优化")
    print("- 训练更稳定")

if __name__ == "__main__":
    compare_methods()
```

```python
        alpha_ratio = sum(c.isalpha() for c in text) / len(text)
        if alpha_ratio < 0.5:
            return False
        
        # 检查重复
        if self._has_repetition(text):
            return False
```

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class EWC:
    """Elastic Weight Consolidation"""
    def __init__(self, model, dataloader, device):
        self.model = model
        self.device = device
        self.fisher = {}
        self.params = {}
        self._compute_fisher(dataloader)
    
    def _compute_fisher(self, dataloader):
        """计算Fisher信息矩阵"""
        self.model.eval()
        for name, param in self.model.named_parameters():
            if param.requires_grad:
                self.fisher[name] = torch.zeros_like(param)
                self.params[name] = param.data.clone()
        
        # 近似计算Fisher
        for batch in dataloader:
            self.model.zero_grad()
            output = self.model(batch['input'])
            loss = output.loss
            loss.backward()
            
            for name, param in self.model.named_parameters():
                if param.requires_grad and param.grad is not None:
                    self.fisher[name] += param.grad.data ** 2
    
    def penalty(self):
        """EWC惩罚项"""
        loss = 0
        for name, param in self.model.named_parameters():
            if name in self.fisher:
                loss += (self.fisher[name] * (param - self.params[name]) ** 2).sum()
        return loss

class KnowledgeDistillation:
    """知识蒸馏缓解遗忘"""
    def __init__(self, teacher_model, student_model, temperature=2.0):
        self.teacher = teacher_model
        self.student = student_model
        self.T = temperature
    
    def distillation_loss(self, student_logits, teacher_logits, labels):
        """蒸馏损失"""
        # KL散度损失
        soft_teacher = F.softmax(teacher_logits / self.T, dim=-1)
        soft_student = F.log_softmax(student_logits / self.T, dim=-1)
        kl_loss = F.kl_div(soft_student, soft_teacher, reduction='batchmean') * (self.T ** 2)
        
        # 硬标签损失
        hard_loss = F.cross_entropy(student_logits, labels)
        
        return 0.5 * kl_loss + 0.5 * hard_loss

def mitigation_strategies():
    """对比缓解策略"""
    print("=" * 60)
    print("灾难性遗忘缓解策略")
    print("=" * 60)
    
    strategies = {
        "LoRA": "只更新少量参数，效果最好",

        "EWC": "对重要参数惩罚",
        "KD": "蒸馏保留知识"
```

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class MoELayer(nn.Module):
    """MoE层"""
    def __init__(self, d_model, d_ff, n_experts, top_k=2):
        super().__init__()
        self.n_experts = n_experts
        self.top_k = top_k
        
        # Router
        self.router = nn.Linear(d_model, n_experts)
        
        # Experts
        self.experts = nn.ModuleList([
            nn.Sequential(
                nn.Linear(d_model, d_ff),
                nn.SiLU(),
                nn.Linear(d_ff, d_model)
            )
            for _ in range(n_experts)
        ])
    
    def forward(self, x):
        batch_size, seq_len, d_model = x.shape
        x_flat = x.view(-1, d_model)
        
        # Router计算
        router_logits = self.router(x_flat)
        router_probs = F.softmax(router_logits, dim=-1)
        
        # 选择top-k专家
        top_k_probs, top_k_indices = torch.topk(router_probs, self.top_k, dim=-1)
        top_k_probs = top_k_probs / top_k_probs.sum(dim=-1, keepdim=True)
        
        # 初始化输出
        output = torch.zeros_like(x_flat)
        
        # 计算每个专家的输出
        for i in range(self.top_k):
            expert_idx = top_k_indices[:, i]
            expert_weight = top_k_probs[:, i]
            
            for e in range(self.n_experts):
                mask = (expert_idx == e)
                if mask.any():
                    expert_input = x_flat[mask]
                    expert_output = self.experts[e](expert_input)
                    output[mask] += expert_output * expert_weight[mask].unsqueeze(-1)
        
        return output.view(batch_size, seq_len, d_model)

class LoadBalancingLoss(nn.Module):
    """负载均衡损失"""
    def __init__(self, n_experts, alpha=0.01):
        super().__init__()
        self.n_experts = n_experts
        self.alpha = alpha
    
    def forward(self, router_probs, expert_indices):
        """
        router_probs: (batch*seq, n_experts)
        expert_indices: (batch*seq, top_k)
        """
        # 计算每个expert被选中的频率
        one_hot = F.one_hot(expert_indices, self.n_experts).float()
        expert_counts = one_hot.sum(dim=(0, 1))  # 每个expert被选中的次数
        
        # 计算路由概率的平均值
        expert_probs = router_probs.mean(dim=0)
        
        # 损失 = n_experts * Σ(f_i * p_i)
        # f_i: expert i被选中的频率
        # p_i: expert i的平均路由概率
        loss = self.n_experts * (expert_counts * expert_probs).sum()
        
        return self.alpha * loss

def moe_analysis():
    """MoE分析"""
    print("=" * 60)
    print("Mixtral 8x7B 分析")
    print("=" * 60)
    
    n_experts = 8
    top_k = 2
    d_model = 4096
    
    # 参数量
    ffn_params = 4 * d_model ** 2 / 1e9
    moe_params = (n_experts * 4 * d_model ** 2) / 1e9
    
    print(f"\n总参数量: {moe_params:.1f} B")
    print(f"激活参数量: {top_k/n_experts * moe_params:.1f} B")
    print(f"稀疏度: {(n_experts - top_k) / n_experts * 100:.0f}%")
    print(f"\n理论上可节省: {(1 - top_k/n_experts) * 100:.0f}% FLOPs")

if __name__ == "__main__":
    moe_analysis()
```

```python
import numpy as np
from typing import List, Tuple

class SimpleRAG:
    """简化RAG系统"""
    def __init__(self, documents: List[str], embedding_model):
        self.documents = documents
        self.embedding_model = embedding_model
        self.doc_embeddings = None
        self._index_documents()
    
    def _index_documents(self):
        """索引文档"""
        self.doc_embeddings = self.embedding_model.encode(self.documents)
    
    def retrieve(self, query: str, top_k: int = 3) -> List[Tuple[str, float]]:
        """检索相关文档"""
        query_embedding = self.embedding_model.encode([query])
        
        # 计算相似度
        similarities = np.dot(self.doc_embeddings, query_embedding.T).flatten()
        
        # 取top-k
        top_indices = np.argsort(similarities)[-top_k:][::-1]
        
        return [(self.documents[i], similarities[i]) for i in top_indices]
    
    def generate(self, query: str, context: List[str]) -> str:
        """生成回答"""
        prompt = f"""基于以下信息回答问题：

问题: {query}

相关信息:
{chr(10).join(context)}

回答:"""
        return prompt
```

```python
import re
from typing import List

class TextChunker:
    """文本分块器"""
    
    def fixed_size_chunk(self, text: str, chunk_size: int = 512, overlap: int = 50):
        """固定大小分块"""
        tokens = text.split()
        chunks = []
        
        for i in range(0, len(tokens), chunk_size - overlap):
            chunk = ' '.join(tokens[i:i + chunk_size])
            chunks.append(chunk)
        
        return chunks
    
    def sentence_chunk(self, text: str, max_sentences: int = 5):
        """句子级别分块"""
        # 简单句子分割
        sentences = re.split(r'[。！？.!?]+', text)
        sentences = [s.strip() for s in sentences if s.strip()]
        
        chunks = []
        current_chunk = []
        current_len = 0
        
        for sent in sentences:
            if current_len + len(sent) > max_sentences * 50 and current_chunk:
                chunks.append(''.join(current_chunk))
                current_chunk = [sent]
                current_len = len(sent)
            else:
                current_chunk.append(sent)
                current_len += len(sent)
        
        if current_chunk:
            chunks.append(''.join(current_chunk))
        
        return chunks
    
    def recursive_chunk(self, text: str, separators: List[str] = None):
        """递归分块"""
        if separators is None:
            separators = ['\n\n', '\n', '. ', ' ']
        
        def split(text, sep):
            parts = text.split(sep)
            result = []
            for p in parts:
                if len(p) > 500:
                    result.extend(split(p, separators[separators.index(sep)+1] if sep in separators[:-1] else separators[-1]))
                else:
                    result.append(p)
            return result
        
        return [p for p in split(text, separators[0]) if p.strip()]

        "固定大小": "简单快速，但可能切断语义",
```

```python
import numpy as np
from collections import Counter
import math

class BM25:
    """BM25关键词检索"""
    def __init__(self, k1=1.5, b=0.75):
        self.k1 = k1
        self.b = b
        self.documents = []
        self.avgdl = 0
    
    def fit(self, documents: List[str]):
        """构建索引"""
        self.documents = [doc.split() for doc in documents]
        self.avgdl = sum(len(doc) for doc in self.documents) / len(self.documents)
        self.N = len(self.documents)
        
        # 计算IDF
        self.doc_freq = Counter()
        for doc in self.documents:
            for word in set(doc):
                self.doc_freq[word] += 1
        
        self.idf = {
            word: math.log((self.N - df + 0.5) / (df + 0.5))
            for word, df in self.doc_freq.items()
        }
    
    def score(self, query: str, doc_idx: int) -> float:
        """计算BM25分数"""
        doc = self.documents[doc_idx]
        doc_len = len(doc)
        doc_counter = Counter(doc)
        
        score = 0
        for word in query.split():
            if word not in self.idf:
                continue
            
            tf = doc_counter.get(word, 0)
            idf = self.idf[word]
            
            # BM25公式
            numerator = tf * (self.k1 + 1)
            denominator = tf + self.k1 * (1 - self.b + self.b * doc_len / self.avgdl)
            score += idf * numerator / denominator
        
        return score

def hybrid_search(query, vector_results, bm25_results, k=5):
    """混合检索（RRF融合）"""
    # RRF: Reciprocal Rank Fusion
    rrf_k = 60  # 通常设为60
    
    scores = {}
    
    # 向量检索结果
    for i, (doc, score) in enumerate(vector_results[:k]):
        scores[doc] = scores.get(doc, 0) + 1 / (rrf_k + i + 1)
    
    # BM25结果
    for i, (doc, score) in enumerate(bm25_results[:k]):
        scores[doc] = scores.get(doc, 0) + 1 / (rrf_k + i + 1)
    
    # 排序
    sorted_results = sorted(scores.items(), key=lambda x: x[1], reverse=True)
    return sorted_results[:k]

def retrieval_comparison():
    """检索方法对比"""
    print("=" * 60)
    print("检索方法对比")
    print("=" * 60)
    
    print("\nBM25: 精确匹配，快速，适合关键词搜索")
    print("向量检索: 语义理解，适合意图搜索")
    print("混合检索: 两者结合，效果最好")

if __name__ == "__main__":
    retrieval_comparison()
```

### 深度展开

```
┌────────────────────────────────────────────────────────────┐
│                    RAGAS评估指标                             │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  【Faithfulness 忠实度】                                    │
│  生成答案是否忠实于检索内容                                  │
│  - 计算生成内容与检索内容的重叠程度                         │
│                                                            │
│  【Answer Relevance 答案相关性】                            │
│  答案是否针对问题                                          │
│  - 使用embedding相似度评估                                  │
│                                                            │
│  【Context Relevance 上下文相关性】                        │
│  检索内容是否与问题相关                                    │
│  - 评估检索到的chunk对问题的覆盖程度                       │
│                                                            │

│  检索内容中关键信息的位置                                    │

│                                                            │

```

### 代码佐证

```python
import numpy as np

class RAGASEvaluator:
    """RAGAS评估"""
    
    def faithfulness(self, question, context, answer):
        """
        计算忠实度
        简化版本：计算答案中从context中抽取的句子比例
        """
        context_sentences = set(context.split('。'))
        answer_sentences = answer.split('。')
        
        faithful_count = sum(1 for s in answer_sentences if any(c in s for c in context_sentences))
        
        return faithful_count / max(len(answer_sentences), 1)
    
    def answer_relevance(self, question, answer):
        """
        计算答案相关性
        简化版本：基于关键词重叠
        """
        q_words = set(question.lower().split())
        a_words = set(answer.lower().split())
        
        overlap = len(q_words & a_words)
        return overlap / len(q_words)
    
    def context_relevance(self, question, contexts):
        """
        计算上下文相关性
        """
        relevance_scores = []
        for ctx in contexts:
            q_words = set(question.lower().split())
            c_words = set(ctx.lower().split())
            overlap = len(q_words & c_words)
            score = overlap / len(q_words)
            relevance_scores.append(score)
        
        return np.mean(relevance_scores)
    
    def evaluate(self, question, contexts, answer):
        """综合评估"""
        return {
            'faithfulness': self.faithfulness(question, ' '.join(contexts), answer),
            'answer_relevance': self.answer_relevance(question, answer),
            'context_relevance': self.context_relevance(question, contexts)
        }

def ragas_demo():
    """RAGAS评估示例"""
    print("=" * 60)
    print("RAGAS评估示例")
    print("=" * 60)
    
    evaluator = RAGASEvaluator()
    
    question = "苹果公司的CEO是谁？"
    contexts = ["苹果公司是一家美国公司。", "蒂姆·库克是苹果公司的CEO。"]
    answer = "苹果公司的CEO是蒂姆·库克。"
    
    metrics = evaluator.evaluate(question, contexts, answer)
    
    for metric, score in metrics.items():
        print(f"{metric}: {score:.3f}")

if __name__ == "__main__":
    ragas_demo()
```

## 4.6 自动提示词优化(OPRO/APE)原理？

**自动提示词优化（OPRO/APE）是让LLM根据任务表现自动迭代优化Prompt。通过不断生成新的Prompt、测试效果、选择更好的Prompt，逐步提升性能。**

```python
class AutoPromptOptimizer:
    """自动Prompt优化"""
    def __init__(self, evaluator):
        self.evaluator = evaluator
        self.history = []
    
    def generate_variations(self, prompt, n=5):
        """生成Prompt变体"""
        variations = [
            f"{prompt}\n\n请简洁回答。",
            f"{prompt}\n\n分点回答。",
            f"作为一个专家，{prompt}",
            f"在回答之前，请仔细思考。{prompt}",
            f"假设你是大学教授，{prompt}"
        ]
        return variations[:n]
    
    def evaluate(self, prompt, test_cases):
        """评估Prompt效果"""
        scores = []
        for case in test_cases:
            response = self.evaluator(prompt, case)
            score = self.evaluator.score(response, case['expected'])
            scores.append(score)
        return sum(scores) / len(scores)
    
    def optimize(self, initial_prompt, test_cases, iterations=10):
        """优化Prompt"""
        current_prompt = initial_prompt
        best_score = self.evaluate(current_prompt, test_cases)
        
        for i in range(iterations):
            # 生成变体
            variations = self.generate_variations(current_prompt)
            
            # 评估所有变体
            for var in variations:
                score = self.evaluate(var, test_cases)
                if score > best_score:
                    current_prompt = var
                    best_score = score
            
            print(f"Iteration {i+1}: Best score = {best_score:.3f}")
        
        return current_prompt, best_score

def auto_prompt_demo():
    """自动Prompt优化示例"""
    print("=" * 60)

    print("=" * 60)
    
    print("\n迭代过程:")
    print("Iteration 1: Best score = 0.65")
    print("Iteration 2: Best score = 0.71")
    print("Iteration 3: Best score = 0.78")
    print("...")
    print("Final Prompt优化效果: +20%")

if __name__ == "__main__":
```

```python
                "parameters": {
                    "type": "object",
                    "properties": {
                        "city": {"type": "string", "description": "城市名称"}
                    },
                    "required": ["city"]
                }
            },
            "calculate": {
                "name": "calculate",
                "description": "计算数学表达式",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "expression": {"type": "string"}
                    }
                }
            }
        }
    
    def call_function(self, name, arguments):
        """执行函数"""
        if name == "get_weather":
            return self.get_weather(arguments.get("city"))
        elif name == "calculate":
            return self.calculate(arguments.get("expression"))
        return None
    
    def get_weather(self, city):
        return f"{city}今天晴天，25度"
    
    def calculate(self, expression):
        try:
            result = eval(expression)
            return str(result)
        except:
            return "计算错误"
    
    def chat(self, user_message):
        """模拟LLM的Function Calling"""
        # 简化版：直接解析关键词
        if "天气" in user_message:
            city = user_message.replace("天气", "").strip()
            return {
                "function": "get_weather",
                "arguments": {"city": city}
            }
        elif "计算" in user_message or any(c in user_message for c in "+-*/"):
            import re
            expr = re.search(r'[\d\+\-\*/\.\s]+', user_message)
            if expr:
                return {
                    "function": "calculate",
                    "arguments": {"expression": expr.group().strip()}
                }
        return None

def function_calling_demo():
    """Function Calling演示"""
    print("=" * 60)
    print("Function Calling演示")
    print("=" * 60)
    
    agent = FunctionCalling()
    
    # 用户提问
    messages = [
        "北京天气怎么样？",
        "帮我计算 2+3*4"
    ]
    
    for msg in messages:
        print(f"\n用户: {msg}")
        
        # LLM决定调用哪个函数
        call = agent.chat(msg)
        if call:
            print(f"调用函数: {call['function']}")
            print(f"参数: {call['arguments']}")
            
            # 执行函数
            result = agent.call_function(call['function'], call['arguments'])
            print(f"结果: {result}")

if __name__ == "__main__":
    function_calling_demo()
```

### 🟡 中级 | ⭐⭐⭐高频

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class SelfAttention(nn.Module):
    """手写Self-Attention"""
    
    def __init__(self, d_model, nhead=8):
        super().__init__()
        assert d_model % nhead == 0
        
        self.d_model = d_model
        self.nhead = nhead
        self.d_k = d_model // nhead
        
        # Q, K, V 投影
        self.W_Q = nn.Linear(d_model, d_model)
        self.W_K = nn.Linear(d_model, d_model)
        self.W_V = nn.Linear(d_model, d_model)
        
        # 输出投影
        self.W_O = nn.Linear(d_model, d_model)
    
    def forward(self, x, mask=None):
        """
        x: (batch, seq_len, d_model)
        mask: (batch, seq_len, seq_len) or None
        """
        batch_size, seq_len, _ = x.shape
        
        # Step 1: 线性投影得到 Q, K, V
        Q = self.W_Q(x)  # (B, N, D)
        K = self.W_K(x)
        V = self.W_V(x)
        
        # Step 2: 分成多头
        # (B, N, D) -> (B, N, nhead, d_k) -> (B, nhead, N, d_k)
        Q = Q.view(batch_size, seq_len, self.nhead, self.d_k).transpose(1, 2)
        K = K.view(batch_size, seq_len, self.nhead, self.d_k).transpose(1, 2)
        V = V.view(batch_size, seq_len, self.nhead, self.d_k).transpose(1, 2)
        
        # Step 3: 计算注意力分数
        # Q @ K^T / sqrt(d_k)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)
        
        # Step 4: 应用mask（如果有）
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        
        # Step 5: Softmax得到注意力权重
        attn_weights = F.softmax(scores, dim=-1)
        
        # Step 6: 加权求和
        # attn_weights @ V
        context = torch.matmul(attn_weights, V)
        
        # Step 7: 合并多头
        # (B, nhead, N, d_k) -> (B, N, nhead, d_k) -> (B, N, D)
        context = context.transpose(1, 2).contiguous().view(batch_size, seq_len, self.d_model)
        
        # Step 8: 输出投影
        output = self.W_O(context)
        
        return output, attn_weights

def test_attention():
    """测试Self-Attention"""
    batch_size = 2
    seq_len = 10
    d_model = 64
    nhead = 8
    
    # 创建模型
    attention = SelfAttention(d_model, nhead)
    
    # 随机输入
    x = torch.randn(batch_size, seq_len, d_model)
    
    # 前向传播
    output, weights = attention(x)
    
    print(f"输入形状: {x.shape}")
    print(f"输出形状: {output.shape}")
    print(f"注意力权重形状: {weights.shape}")
    
    # 检查注意力权重的性质
    print(f"注意力权重和(每行): {weights[0, 0].sum(dim=-1)}")  # 应该全为1

if __name__ == "__main__":
    test_attention()
```

## 10.2 手写LoRA（PyTorch）

### 🟡 中级 | ⭐⭐

### 面试官问法

> "请手写LoRA"
> "LoRA如何节省显存？"
> "解释低秩分解原理"

---

### 代码答案

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class LoRALinear(nn.Module):
    """LoRA线性层"""
    
    def __init__(self, in_features, out_features, rank=4, alpha=1):
        super().__init__()
        self.in_features = in_features
        self.out_features = out_features
        self.rank = rank
        self.alpha = alpha
        
        # 冻结原始权重
        self.weight = nn.Parameter(
            torch.randn(out_features, in_features),
            requires_grad=False
        )
        
        # LoRA可训练参数
        self.lora_A = nn.Parameter(torch.randn(rank, in_features) * 0.01)
        self.lora_B = nn.Parameter(torch.zeros(out_features, rank))
        
        # 缩放因子
        self.scaling = alpha / rank
    
    def forward(self, x):
        """
        W = W_0 + (alpha / rank) * B @ A
        
        原始计算: y = x @ W^T
        LoRA计算: y = x @ (W_0 + BA)^T = x @ W_0^T + x @ A^T @ B^T
        """
        # 原始输出
        original_output = F.linear(x, self.weight)
        
        # LoRA输出
        lora_output = x @ self.lora_A.T @ self.lora_B.T * self.scaling
        
        return original_output + lora_output
    
    def merge_weights(self):
        """合并LoRA权重到原始权重"""
        delta_W = self.lora_B @ self.lora_A * self.scaling
        merged_weight = self.weight + delta_W
        return merged_weight

class LoRAModel(nn.Module):
    """带LoRA的模型"""
    
    def __init__(self, base_model, rank=4, alpha=1, target_modules=['q_proj', 'v_proj']):
        super().__init__()
        self.base_model = base_model
        self.rank = rank
        self.alpha = alpha
        
        # 替换指定层为LoRA层
        self.lora_layers = nn.ModuleDict()
        
        for name, module in base_model.named_modules():
            if any(target in name for target in target_modules):
                if isinstance(module, nn.Linear):
                    # 创建LoRA层
                    lora_layer = LoRALinear(
                        module.in_features,
                        module.out_features,
                        rank=rank,
                        alpha=alpha
                    )
                    
                    # 复制原始权重
                    lora_layer.weight.data = module.weight.data.clone()
                    
                    # 存储原始层引用
                    self.lora_layers[name] = {
                        'lora': lora_layer,
                        'original': module
                    }
    
    def forward(self, x):
        """简化的前向传播"""
        return self.base_model(x)
    
    def get_trainable_params(self):
        """获取可训练参数"""
        params = []
        for layer_info in self.lora_layers.values():
            params.extend([layer_info['lora'].lora_A, layer_info['lora'].lora_B])
        return params
    
    def print_trainable_params(self):
        """打印可训练参数统计"""
        total_params = sum(p.numel() for p in self.base_model.parameters())
        trainable_params = sum(p.numel() for p in self.get_trainable_params())
        
        print(f"总参数量: {total_params:,}")
        print(f"可训练参数量: {trainable_params:,}")
        print(f"可训练比例: {100*trainable_params/total_params:.4f}%")

def compare_lora_params():
    """对比LoRA参数节省"""
    print("=" * 60)
    print("LoRA参数对比")
    print("=" * 60)
    
    for d_model in [512, 1024, 4096]:
        for rank in [4, 8, 16]:
            full_params = d_model * d_model
            lora_params = 2 * rank * d_model
            ratio = lora_params / full_params * 100
            
            print(f"d_model={d_model}, rank={rank}:")
            print(f"  全参数: {full_params:,}, LoRA: {lora_params:,}, 比例: {ratio:.2f}%")

if __name__ == "__main__":
    compare_lora_params()
```

```python
import numpy as np
from typing import List, Tuple

class SimpleVectorStore:
    """简单向量数据库"""
    
    def __init__(self):
        self.documents = []
        self.embeddings = []
    
    def add(self, text: str, embedding: np.ndarray):
        self.documents.append(text)
        self.embeddings.append(embedding)
    
    def search(self, query_embedding: np.ndarray, top_k: int = 3) -> List[Tuple[str, float]]:
        """余弦相似度搜索"""
        embeddings = np.array(self.embeddings)
        query = query_embedding / (np.linalg.norm(query_embedding) + 1e-8)
        docs = embeddings / (embeddings.norm(axis=1, keepdims=True) + 1e-8)
        
        similarities = np.dot(docs, query)
        
        top_indices = np.argsort(similarities)[-top_k:][::-1]
        
        return [(self.documents[i], similarities[i]) for i in top_indices]

class SimpleEmbedding:
    """简化Embedding（实际应使用真实模型）"""
    
    def __init__(self, dim=768):
        self.dim = dim
    
    def encode(self, texts: List[str]) -> np.ndarray:
        """简化的编码（实际应使用模型）"""
        np.random.seed(42)
        return np.random.randn(len(texts), self.dim)

        embeddings = self.embedding_model.encode(documents)

        """检索相关文档"""
        query_embedding = self.embedding_model.encode([query])[0]
        return self.vector_store.search(query_embedding, top_k)
    
    def generate(self, query: str, context: str) -> str:
        """生成回答（简化版）"""
        # 实际应调用LLM
        return f"基于以下信息回答：\n{context}\n\n问题：{query}"
    
    def query(self, question: str) -> str:

        # 1. 检索相关文档
        retrieved_docs = self.retrieve(question, top_k=3)

        context = '\n'.join([f"[文档{i+1}] {doc}" for i, (doc, _) in enumerate(retrieved_docs)])
```

## 10.4 手写cosine similarity和向量检索

### 🟡 中级 | ⭐⭐

### 面试官问法

> "请手写cosine similarity"
> "如何实现向量检索？"
> "暴力搜索和ANN的区别？"

---

### 代码答案

```python
import numpy as np

def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float:
    """
    计算余弦相似度
    
    cos(θ) = (A · B) / (||A|| × ||B||)
    """
    norm_a = np.linalg.norm(a)
    norm_b = np.linalg.norm(b)
    
    if norm_a == 0 or norm_b == 0:
        return 0
    
    return np.dot(a, b) / (norm_a * norm_b)

def cosine_similarity_batch(query: np.ndarray, vectors: np.ndarray) -> np.ndarray:
    """
    批量计算余弦相似度
    
    query: (d,) 查询向量
    vectors: (n, d) 文档向量
    """
    # 归一化
    query_norm = query / (np.linalg.norm(query) + 1e-8)
    vectors_norm = vectors / (vectors.norm(axis=1, keepdims=True) + 1e-8)
    
    # 点积就是余弦相似度
    return np.dot(vectors_norm, query_norm)

def brute_force_search(query: np.ndarray, vectors: np.ndarray, top_k: int = 5) -> np.ndarray:
    """
    暴力最近邻搜索
    
    时间复杂度: O(n × d)
    空间复杂度: O(n)
    """
    similarities = cosine_similarity_batch(query, vectors)
    top_indices = np.argsort(similarities)[-top_k:][::-1]
    return top_indices, similarities[top_indices]

class SimpleANN:
    """简化的近似最近邻"""
    
    def __init__(self, n_clusters=10):
        self.n_clusters = n_clusters
        self.centroids = None
        self.vectors = None
        self.labels = None
    
    def fit(self, vectors: np.ndarray):
        """聚类"""
        from sklearn.cluster import KMeans
        
        self.vectors = vectors
        kmeans = KMeans(n_clusters=self.n_clusters, random_state=42)
        self.labels = kmeans.fit_predict(vectors)
        self.centroids = kmeans.cluster_centers_
    
    def search(self, query: np.ndarray, top_k: int = 5) -> np.ndarray:
        """搜索"""
        # 1. 找到最近的簇中心
        centroid_sims = cosine_similarity_batch(query, self.centroids)
        nearest_cluster = np.argmax(centroid_sims)
        
        # 2. 在该簇中搜索
        cluster_mask = self.labels == nearest_cluster
        cluster_vectors = self.vectors[cluster_mask]
        
        if len(cluster_vectors) == 0:
            return [], []
        
        # 暴力搜索该簇
        indices, sims = brute_force_search(query, cluster_vectors, top_k)
        
        # 映射回原始索引
        original_indices = np.where(cluster_mask)[0][indices]
        
        return original_indices, sims

def demo_search():
    """搜索演示"""
    print("=" * 60)
    print("向量检索演示")
    print("=" * 60)
    
    # 创建随机数据
    np.random.seed(42)
    n = 1000
    d = 128
    vectors = np.random.randn(n, d)
    
    # 查询向量
    query = np.random.randn(d)
    
    # 暴力搜索
    indices, sims = brute_force_search(query, vectors, top_k=5)
    print(f"\n暴力搜索结果 (top 5):")
    for i, (idx, sim) in enumerate(zip(indices, sims)):
        print(f"  {i+1}. 索引={idx}, 相似度={sim:.4f}")
    
    # ANN搜索
    ann = SimpleANN(n_clusters=10)
    ann.fit(vectors)
    indices, sims = ann.search(query, top_k=5)
    print(f"\nANN搜索结果 (top 5):")
    for i, (idx, sim) in enumerate(zip(indices, sims)):
        print(f"  {i+1}. 索引={idx}, 相似度={sim:.4f}")

if __name__ == "__main__":
    demo_search()
```

```
═══════════════════════════════════════════════════════════════
                    模拟面试对话
═══════════════════════════════════════════════════════════════

【第一轮：自我介绍 + 基础概念】

面试官：请简单介绍一下你自己，以及为什么想从事大模型相关工作？

之所以想从事大模型工作，是因为：
1. 大模型是AI发展的趋势，有广阔前景
2. 我对预训练、微调等技术很感兴趣
3. 希望能在实际项目中应用这些技术"

面试官追问：
"你能简单介绍一下BERT和GPT的区别吗？"

候选人：
"BERT是Encoder-only架构，采用双向上下文注意力，
适合理解任务如文本分类、NER。
GPT是Decoder-only架构，采用单向注意力，适合生成任务。
主要区别在于：
1. 架构：BERT看全文，GPT看过去
2. 任务：BERT理解，GPT生成
3. 训练：BERT用MLM，GPT用NTP"

═══════════════════════════════════════════════════════════════

【第二轮：深度追问】

面试官：那你知道为什么Transformer能够并行处理，而RNN需要串行吗？

候选人：
"这是因为RNN的结构决定的。RNN每个时间步的输出依赖
当前的输入和上一步的隐藏状态，而上一步的隐藏状态又
依赖上上步...这就形成了一个链式依赖，必须一步步计算。

Transformer通过自注意力机制解决了这个问题。每个位置的
表示可以同时计算，只需要一次矩阵乘法 QK^T 就能得到
所有位置之间的关系，所以可以并行。

关键在于：RNN的时间依赖是串行的，Transformer的空间依赖是并行的。"

面试官追问：
"那你提到注意力计算，这里面为什么要除以√d_k？"

候选人：
"除以√d_k主要是为了防止梯度消失/爆炸。

当d_k较大时，QK^T的值会很大，导致softmax进入饱和区，
梯度接近于0，模型难以训练。

具体来说：
- 假设Q、K元素均值为0、方差为1
- QK^T每个元素的方差是d_k
- 除以√d_k后，方差变为1

这样可以保证softmax的输入方差稳定，梯度流通顺畅。"

═══════════════════════════════════════════════════════════════

【第三轮：实战问题】

   - Embedding：用BAAI/bge-large-zh生成向量
   - 存储到向量数据库（Milvus/Pinecone）

2. 【检索层】
   - 用户query向量化
   - 向量检索 top_k=10
   - 混合检索（可选：关键词+向量）
   - 重排序（用Cross-Encoder）

3. 【生成层】
   - 构建prompt：System + Context + Query
   - 调用LLM生成回答
   - 后处理：引用标注

4. 【优化方向】
   - 查询改写（HyDE）

   - 缓存机制"

面试官追问：
```
