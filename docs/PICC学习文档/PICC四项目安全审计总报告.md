### 二、共性问题详解

#### 🔴 共性问题1：密钥硬编码（影响3/4项目）

**问题描述**：  
多个项目将加密密钥直接硬编码在源代码中，如同"把钥匙藏在门口地毯下"。

**生活化比喻**：  
> 你把家门钥匙用胶带粘在门口的脚垫下面，告诉客人"放心，藏在隐蔽处"。但小偷只要翻脚垫就能开门。

**具体发现**：

| 项目 | 硬编码密钥类型 | 发现位置 | 风险 |
|-----|--------------|---------|-----|
| user | SM4、AES、RSA密钥 | SM4Util.java、AesUtil.java | 用户密码可被解密 |
| server | FTP密码、第三方API密钥 | FTPFileUtil_xcx.java | FTP服务器可被入侵 |
| gateway | SM2、AES、SFTP密钥 | Sm2Util.java、SFTPUtils.java | 全部加密数据可被解密 |

**统一问题分析**：
```yaml
# 所有密钥必须通过环境变量或配置中心管理
# 推荐使用Apollo配置中心或HashiCorp Vault

# application.yml 示例
encryption:
  sm4:
    key: ${SM4_SECRET_KEY}        # 必须从环境变量读取
  aes:
    key: ${AES_SECRET_KEY}
  sm2:
    private-key: ${SM2_PRIVATE_KEY}
    public-key: ${SM2_PUBLIC_KEY}

ftp:
  password: ${FTP_PASSWORD}        # 必须从环境变量读取

# 启动脚本
# start.sh
export SM4_SECRET_KEY="$(openssl rand -base64 32)"
export AES_SECRET_KEY="$(openssl rand -base64 32)"
export FTP_PASSWORD="$(openssl rand -base64 24)"
```

#### 🔴 共性问题2：CORS跨域配置漏洞（影响3/4项目）

**问题描述**：  
CORS配置允许任意来源访问，如同"门卫对谁都喊请进"。

**生活化比喻**：  
> 你开了一家餐厅，告诉门卫"对所有人都说请进"，还让他们"不用查身份证"、"可以复制钥匙带走"。这意味着任何人都能进你的餐厅假装成你。

**具体发现**：

| 项目 | CORS配置 | 风险等级 |
|-----|---------|---------|
| server | 未明确配置CORS | 🟠 中危 |
| gateway | `allowedOrigins("*")` + `allowCredentials(true)` | 🔴 致命 |
| agent | `Access-Control-Allow-Origin: *` | 🟠 中危 |

**gateway危险代码示例**：
```java
// GlobalCorsConfig.java - 这是严重漏洞！
registry.addMapping("/**")                    // 对所有路径开放
        .allowedOrigins("*")                 // 允许所有来源！致命！
        .allowCredentials(true)               // 允许携带Cookie
        .allowedMethods("GET","POST","PUT","DELETE")
        .allowedHeaders("*");                // 允许所有请求头
```

**统一问题分析**：
```java
@Override
public void addCorsMappings(CorsRegistry registry) {
    registry.addMapping("/**")
            // 只允许受信任的域名，绝不使用 *
            .allowedOriginPatterns(
                "https://www.picchealth.com",
                "https://mzmtb.picchealth.com",
                "https://mzmtb-h5.picchealth.com"
            )
            .allowCredentials(true)
            .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
            .allowedHeaders("Content-Type", "Authorization", "X-Requested-With")
            .exposedHeaders("Content-Disposition")
            .maxAge(3600);
}
```

#### 🟠 共性问题3：敏感信息日志泄露（影响2/4项目）

**问题描述**：  
在日志中输出手机号、Token等敏感信息，如同"把客户银行卡号写在便利贴上贴公告"。

**生活化比喻**：  
> 银行柜员把客户的账号密码写在便签纸上，贴在公告栏供大家"参考"。

**具体发现**：

| 项目 | 泄露位置 | 泄露内容 |
|-----|---------|---------|
| user | ApiInterceptor | 权限校验时打印用户手机号 |
| server | 多个Service | 异常日志中打印请求参数 |
| gateway | APIAuthorityFilter | Token验证失败时打印手机号 |

**危险代码示例**：
```java
// gateway - 这是泄露用户隐私！
} catch (Exception e) {
    log.info("手机号与Token不匹配", xcxString);  // 泄露手机号
    sendErrorResponse(response, 1002, "手机号与Token不匹配");
    return;
}
```

**统一问题分析**：
```java
// 方案1：日志脱敏
private String maskPhoneNumber(String phone) {
    if (phone == null || phone.length() < 11) {
        return "***";
    }
    return phone.substring(0, 3) + "****" + phone.substring(7);
}

// 方案2：日志中不打印敏感字段
log.warn("Token validation failed for user: {}", maskPhoneNumber(phone));
```

#### 2. BCrypt密码迁移脚本

```java
// ============================================================
// 密码迁移脚本 - 将MD5密码迁移到BCrypt
// ============================================================

@Service
public class PasswordMigrationService {
    
    @Autowired
    private PasswordEncoder passwordEncoder;
    
    @Autowired
    private UserMapper userMapper;
    
    /**
     * 迁移所有用户密码从MD5到BCrypt
     * 
     * 注意：此操作不可逆，执行前必须：
     * 1. 备份数据库
     * 2. 在测试环境验证
     * 3. 选择低峰期执行
     */
    @Transactional
    public MigrationResult migrateAllPasswords() {
        MigrationResult result = new MigrationResult();
        List<User> users = userMapper.selectAllUsers();
        
        for (User user : users) {
            try {
                // 1. 验证MD5密码格式
                if (!isMD5Hash(user.getPassword())) {
                    // 已经是新格式，跳过
                    result.addSkipped();
                    continue;
                }
                
                // 2. 用MD5解密获取明文
                String plainPassword = decodeMD5(user.getPassword());
                
                // 3. 用BCrypt重新加密
                String newPassword = passwordEncoder.encode(plainPassword);
                
                // 4. 更新数据库
                user.setPassword(newPassword);
                userMapper.updateById(user);
                
                result.addMigrated();
            } catch (Exception e) {
                log.error("密码迁移失败: userId={}", user.getId(), e);
                result.addFailed();
            }
        }
        
        return result;
    }
}
```

### 二、自动化安全检查脚本

```bash
#!/bin/bash
# ============================================================
# PICC门诊慢特病系统 - 上线前安全检查脚本
# ============================================================

PROJECT_DIR="/path/to/picc-mzmtb"
EXIT_CODE=0

echo "========================================"
echo "PICC安全检查开始"
echo "========================================"

# 检查1：硬编码密钥扫描
echo "[检查1] 扫描硬编码密钥..."
KEYWORDS=(
    "1234567812345678"  # SM4测试密钥
    "abcdefgabcdefg12"  # AES测试密钥
    "hellolevy"         # FTP测试密码
    "hmlink@123"        # FTP测试密码
    "00ced30f88fc9187"  # SM2私钥前缀
)

for keyword in "${KEYWORDS[@]}"; do
    result=$(grep -r "$keyword" "$PROJECT_DIR" --include="*.java" --include="*.js" --include="*.vue" 2>/dev/null)
    if [ -n "$result" ]; then
        echo "❌ 发现硬编码密钥: $keyword"
        echo "$result"
        EXIT_CODE=1
    fi
done

# 检查2：CORS配置
echo "[检查2] 检查CORS配置..."
cors_result=$(grep -r 'allowedOrigins.*"*"' "$PROJECT_DIR" --include="*.java" 2>/dev/null)
if [ -n "$cors_result" ]; then
    echo "❌ 发现CORS全开配置"
    echo "$cors_result"
    EXIT_CODE=1
fi

# 检查3：console.log残留
echo "[检查3] 检查console.log残留..."
log_count=$(grep -r "console\.log" "$PROJECT_DIR" --include="*.vue" --include="*.js" 2>/dev/null | wc -l)
if [ "$log_count" -gt 0 ]; then
    echo "❌ 发现$log_count处console.log"
    EXIT_CODE=1
fi

# 检查4：MD5密码存储
echo "[检查4] 检查MD5密码存储..."
md5_result=$(grep -r "MD5.*password\|password.*MD5" "$PROJECT_DIR" --include="*.java" --include="*.js" 2>/dev/null)
if [ -n "$md5_result" ]; then
    echo "❌ 发现MD5密码处理"
    echo "$md5_result"
    EXIT_CODE=1
fi

# 检查5：URL传递Token
echo "[检查5] 检查URL传递Token..."
url_token=$(grep -r "query.*token\|token.*query" "$PROJECT_DIR" --include="*.vue" --include="*.js" 2>/dev/null)
if [ -n "$url_token" ]; then
    echo "❌ 发现URL传递Token"
    echo "$url_token"
    EXIT_CODE=1
fi

echo "========================================"
if [ $EXIT_CODE -eq 0 ]; then
    echo "✅ 所有安全检查通过"
else
    echo "❌ 存在安全隐患，请修复后再上线"
fi
echo "========================================"

exit $EXIT_CODE
```
