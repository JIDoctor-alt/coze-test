## 一、项目概述

### 1.1 项目背景

PICC人保健康门诊慢特病业务管理信息系统是面向全国16个地市的医保慢特病管理平台，覆盖宝鸡、阜新、商洛、张家口、延安、达州、晋中、榆林、满洲里、定州、淮南、新余、杨凌、德州、九江、晋城等地市，为各地市医保局、定点医疗机构提供统一的慢特病业务管理服务。

### 1.2 核心业务

**picc-mzmtb-user** 是整个系统的权限管理服务（Privilege Server），负责：

- **用户身份认证**：登录登出、Token管理、密码加密
- **权限访问控制**：基于角色的访问控制(RBAC)、菜单权限、数据权限
- **机构层级管理**：多级组织架构、16地市差异化配置
- **系统菜单管理**：动态菜单、菜单与机构关联
- **接口鉴权认证**：API级别权限校验、URL白名单

### 3.2 Token机制（Redis存储）

```
┌─────────────────────────────────────────────────────────────────┐
│                        Token认证流程                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │

│            │ 验证通过                                           │

│   │ 生成Token        │────▶│ Redis存储        │                   │
│   │ UUID            │     │ Key: API_TOKEN:{token}              │
│   └────────┬────────┘     │ TTL: 1440分钟    │                   │
│            │              └────────┬────────┘                   │
│            │                       │                             │
│            ▼                       ▼                             │
│   ┌─────────────────┐     ┌─────────────────┐                   │
│   │ 返回Token        │     │ 缓存用户权限     │                   │
│   │ 给客户端         │     │ Key: UrlPermission:{userId}        │
│   └─────────────────┘     └─────────────────┘                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**Redis Key规范**：
- Token缓存：`API_TOKEN:{token}` → User对象，TTL 1440分钟
- URL权限：`UrlPermission:{userId}` → List<String>，存储用户可访问URL列表

### 3.3 密码加密体系

项目采用了多层次的加密策略：

| 加密场景 | 算法 | 说明 |
|---------|------|------|
| 密码存储 | MD5 + SHA-256 | 双重加密存储 |
| 敏感字段 | AES |身份证号等敏感数据加密存储 |
| 配置文件密钥 | AES/ECB/PKCS5Padding | 敏感配置加密 |

**关键代码**：

```java
// AES加密工具 (AesUtil.java)

- 密钥: ${utils.aes.KEY} (Apollo配置)

// MD5加密 (EncryptionUtil.java)
public static String encodeMD5(String inStr) {
    // 标准MD5加密
}

public static String getPassword(String pwd) {
    // SHA-256加密
}
```

### 3.4 已知安全风险与修复计划

| 风险等级 | 风险描述 | 当前措施 | 修复建议 |
|---------|---------|---------|----------|
| 🔴 高 | Token续期使用固定值1440分钟 | 已实现 | 建议根据用户级别动态设置 |
| 🟠 中 | AES密钥硬编码在配置中 | Apollo配置 | 建议迁移至密钥管理服务(KMS) |
| 🟡 低 | 日志可能打印敏感Token | log.info | 建议脱敏处理 |
| 🟡 低 | URL权限使用字符串contains匹配 | stream().filter | 建议使用精确路径匹配 |

## 七、性能考量

### 7.1 已知性能问题

| 问题类型 | 位置 | 影响 | 严重程度 |
|---------|------|------|---------|
| N+1查询 | 菜单树查询 | 机构树层级深时查询次数激增 | 🟠 中 |
| BFS机构树 | 递归查询 | 大数据量时递归深度影响性能 | 🟡 低 |
| 无分页限制 | 列表查询 | 大数据量时内存溢出风险 | 🟠 中 |
| Redis同步锁 | APIAuthorityFilter | 高并发时线程等待 | 🟡 低 |

        UNION ALL
        SELECT p.id, p.name, p.parent_id, t.level + 1

## 九、技术债务

### 9.1 代码质量问题清单

| 序号 | 问题类型 | 位置 | 描述 | 建议修复 |
|------|---------|------|------|----------|
| 1 | ⚠️ 魔法值 | 多处 | 使用硬编码的权限码"88" | 提取为常量 |
| 2 | ⚠️ 魔法值 | 多处 | 分页默认值为null | 设置默认值 |
| 3 | ⚠️ 同步锁 | APIAuthorityFilter | synchronized(this)可能无效 | 使用分布式锁 |
| 4 | ⚠️ SQL注入风险 | Mapper XML | 动态SQL需注意 | 使用参数化查询 |
| 5 | ⚠️ 异常捕获 | 多处Service | 通Exception捕获 | 细分异常类型 |
| 6 | ⚠️ 日志泄露 | TokenInterceptorConfig | System.out.println | 统一使用log |
| 7 | ⚠️ 注释缺失 | 部分方法 | 关键方法缺少注释 | 补充注释 |

### 9.2 安全风险清单

| 序号 | 风险等级 | 风险描述 | 当前状态 | 修复计划 |
|------|---------|---------|---------|----------|
| 1 | 🔴 高 | Token固定1440分钟续期 | 已实现 | 下个迭代优化 |
| 2 | 🔴 高 | AES密钥存储在配置中 | Apollo配置 | 迁移至KMS |
| 3 | 🟠 中 | 密码使用MD5(已知不安全) | 历史遗留 | 计划迁移至BCrypt |
| 4 | 🟠 中 | 敏感数据日志打印 | 部分存在 | 统一脱敏处理 |
| 5 | 🟡 低 | URL权限使用contains匹配 | 性能一般 | 优化为精确匹配 |
| 6 | 🟡 低 | 无API调用频率限制 | 未实现 | 建议添加限流 |
