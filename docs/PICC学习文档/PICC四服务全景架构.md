### 1.2 四个服务详细信息对比表

| 服务 | 花名（项目名） | 工牌号（端口） | 代码规模 | 核心职责 | 小白比喻 |
|------|---------------|---------------|---------|---------|---------|
| **Vue前端** | picc-mzmtb-agent | 80/443 | 652个Vue文件，74个API文件 | 用户界面交互、状态管理、API调用 | 菜单和餐桌，客人直接看到 |
| **前台服务** | picc-mzmtb-gateway | 9001 | 809个Java文件，104个API，361个VO | HTTP请求转发、VO层转换、统一入口 | 服务员，接待客人传菜 |
| **权限服务** | picc-mzmtb-user | 9092 | 136个Java文件，47个API，17张表 | 用户认证、权限管理、机构管理 | 保安室，检查谁能进 |
| **业务服务** | picc-mzmtb-server | 9091 | 2647个Java文件，894个API，213个Mapper | 核心业务处理、数据库操作 | 后厨，真正做菜的地方 |

## 二、架构关系：四个服务是怎么住在一起的？

```
┌─────────────────────────────────────────────────────────────────────────────────┐

│   │   ┌─────────────────┐  ┌─────────────────┐                             │  │

└────────────┼────────────────────┼────────────────────┼────────────────────────┘

             │  ← 各自独立部署，互不依赖Pod重启  →
```

**特点**：
- ✅ **前端独立Pod**：Vue前端有自己的Pod，Nginx提供静态资源
- ✅ **前台服务独立Pod**：作为统一入口，独立部署
- ✅ **后端服务独立Pod**：权限和业务服务各自在自己的Pod中
- ✅ **Redis共享Pod**：三个后端服务共用Redis缓存

### 5.2 Nginx配置详解

```
┌─────────────────────────────────────────────────────────────────────────────────┐

│                                                                                 │
│  📄 nginx.conf 配置结构                                                         │
│  ┌─────────────────────────────────────────────────────────────────────────┐  │
│  │                                                                          │  │

│  │      # 前端静态资源                                                        │  │

│  │                                                                          │  │
│  │      # API请求代理到前台服务                                               │  │

│  │          proxy_pass http://gateway-svc:9001;  # 转发到前台服务              │  │
│  │          proxy_set_header Host $host;                                    │  │
│  │          proxy_set_header X-Real-IP $remote_addr;                        │  │
│  │          proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;    │  │
│  │      }                                                                    │  │
│  │                                                                          │  │
│  │      # 小程序接口代理                                                      │  │

│  │                                                                          │  │
│  └─────────────────────────────────────────────────────────────────────────┘  │
│                                                                                 │

│  ┌─────────────────────────────────────────────────────────────────────────┐  │
│  │                                                                          │  │
│  │  1. try_files $uri $uri/ /index.html                                    │  │
│  │     → SPA单页应用路由支持                                                │  │
│  │     → 当找不到文件时，返回index.html让Vue Router处理路由                  │  │
│  │                                                                          │  │
│  │  2. proxy_set_header X-Forwarded-For                                   │  │

│  │                                                                          │  │
```

### 10.2 常用场景速查

| 场景 | 前端URL | 后端接口 | 涉及服务 |
|------|--------|----------|---------|
| 用户登录 | /api/user/login | POST /user/login | 前台→权限 |
| 查询申报列表 | /api/mb/declare/list | GET /mb/declare/list | 前台→业务 |
| 创建申报 | /api/mb/declare/create | POST /mb/declare/create | 前台→业务→权限 |
| 审批通过 | /api/mb/approve | POST /mb/approve | 前台→业务 |
| 药店结算 | /api/pharmacy/settle | POST /pharmacy/settle | 前台→业务 |
| 小程序查询 | /api/wx/card/query | GET /wx/card/query | 前台→业务 |

### 10.3 核心问题速答

| 问题 | 答案 |
|------|------|
| 前端挂了怎么办 | 后端服务正常，但用户无法访问 |
| 前台服务挂了怎么办 | **所有服务都不可用**（统一入口） |
| 权限服务挂了怎么办 | 新用户无法登录，已登录用户部分功能受影响 |
| 业务服务挂了怎么办 | 业务功能不可用，登录等基础功能正常 |
| Redis挂了怎么办 | **所有服务都受影响**（Token无法验证） |
| Apollo挂了怎么办 | 服务启动失败，已运行的服务可能异常 |
