# 需求2：榆林申报综合查询新增初审驳回意见展示

## 功能需求概述
- 点击"备案列表"进入详情页，在"申报状态"为"初审驳回"时，增加"驳回原因"展示
- 存在多条"初审驳回"流程时，分别显示对应驳回原因
- 申报列表展示字段不作调整
- 仅修改榆林慢病管理模块，其余地市保持不变
- 新增字段保持与"申报状态"的字体、大小、颜色保持一致
- 保证历史"申报状态"为"初审驳回"的单据，驳回原因不为空

private String getInitialnote(VipMbdeclareInfoDto vdto) {
    VipMbdeclareApproval vipMbdeclareApproval = new VipMbdeclareApproval();
    vipMbdeclareApproval.setApprovaltype(ApprovalTypeEnum.APPROVALTYPE_FIRST.getValue());
    vipMbdeclareApproval.setDeclareid(vdto.getId());
    List<VipMbdeclareApproval> list = vipMbdeclareApprovalService.selectAll(vipMbdeclareApproval);
    for (VipMbdeclareApproval note:list){
        if (note.getApprovalnote()!=null){
            String approvalnote = note.getApprovalnote();
            return approvalnote;  // 只返回第一条有意见的记录
        }
    }
    return null;
}

@Override
public ApiResponse<VipMbdeclareApprovalDto> queryDetailsList(QueryMbDeclareListVo queryMbDeclareListVo) {
    // ... 
    
    // 榆林地区特色：查询并处理初审驳回记录（第81-106行）
    VipMbdeclareApproval vipMbdeclareApproval = new VipMbdeclareApproval();
    vipMbdeclareApproval.setDeclareid(vipMbdeclareInfoDto.getId());
    vipMbdeclareApproval.setApprovaltype("1"); // 1：初审
    vipMbdeclareApproval.setDr(DrEnum.VALID.getValue());
    List<VipMbdeclareApproval> firstTrialList = vipMbdeclareApprovalService.selectAll(vipMbdeclareApproval);

    // 处理初审驳回记录
    for (VipMbdeclareApproval firstTrial : firstTrialList) {
        VipMbdeclareApprovalDto dto = new VipMbdeclareApprovalDto();
        BeanUtils.copyProperties(firstTrial, dto);
        // 标记为初审记录
        dto.setFlag("YL_FIRST_TRIAL");
        // 处理驳回原因
        if (firstTrial.getApprovalstatus() == 1) { // 1：不通过
            String rejectionReason = firstTrial.getApprovalnote();
            // 确保驳回原因不为空
            if (rejectionReason == null || rejectionReason.isEmpty()) {
                rejectionReason = "资料审核不通过";
            }
            dto.setApprovalnote("初审驳回原因：" + rejectionReason);
        } else if (firstTrial.getApprovalstatus() == 0) { // 0：通过
            dto.setApprovalnote("初审通过");
        }
        vipMbdeclareApprovalDtos.add(dto);
    }
    
    // ... 后续专家分组逻辑 ...
}

## 发现的问题与建议

### 问题1：只显示一条驳回原因
**现状：**
- 后端 `getInitialnote` 方法只返回第一条有意见的初审记录
- 前端也只显示一个驳回原因

**需求要求：**
> 当"申报进度详情"存在多条"申报状态"为"初审驳回"的流程时，分别显示对应申报流程初审驳回原因。

**修改建议：**
如需支持多条驳回记录展示，需要：
1. 后端修改：返回所有驳回原因列表
2. 前端修改：以列表形式展示多条驳回原因

### 问题2：详情接口返回的数据前端未处理
**现状：**
- 后端 `queryDetailsList` 返回带 `flag="YL_FIRST_TRIAL"` 标记的初审记录
- 前端没有专门处理这些记录

**建议：**
- 如果需要更详细的驳回记录展示，前端可以额外处理 `flag="YL_FIRST_TRIAL"` 的记录
