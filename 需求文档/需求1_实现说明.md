# 榆林慢病管理 - 申报综合查询导出敏感信息权限控制

## 一、需求概述

### 功能名称
新增敏感信息导出隐藏权限控制

### 业务场景
"榆林慢病管理" - "申报综合查询"功能菜单

### 需求描述
1. "申报综合查询"功能菜单，点击【导出Excel】按钮时，根据当前操作人账号判断是否为配置了导出权限的账号
2. 配置了导出权限的账号，导出信息不做加密处理
3. 未配置导出权限的账号，导出信息中的敏感字段做加密处理

### 涉及字段
- 参保人姓名
- 手机号码
- 身份证号
- 专家姓名
- 专家手机号

### 加密规则
| 字段 | 原始值 | 加密后 | 说明 |
|------|--------|--------|------|
| 姓名 | 张三 | 张** | 保留姓氏 + 两个星号 |
| 身份证号 | 130180199802180918 | 130182********0918 | 前6位 + 8个星号 + 后4位 |
| 手机号 | 15409287181 | 154****2617 | 前3位 + 4个星号 + 后4位 |

### 影响范围
仅"榆林慢病管理"模块，其余地市保持不变

### 修改文件列表

| `mtb-yh/mtb-yl/src/main/java/com/picchealth/module/mtb/service/impl/MtbDeclareListYLServiceImpl.java` | 修改 | 重写导出方法，添加权限检查和加密处理 |

### 核心代码说明

#### 1. 重写导出方法
```java
@Override
public ApiResponse<ExportMbdeclareInfoDto> exportMbdeclareInfo(QueryMbDeclareListVo queryMbDeclareListVo) {
    // 1. 获取当前用户账号
    String userAccount = UserUtils.getUser().getUserAccount();
    
    // 2. 检查当前用户是否有榆林地区导出权限
    boolean hasExportPermission = checkYlExportPermission(userAccount);
    
    // 3. 调用父类方法获取查询数据
    ApiResponse<ResultPage<VipMbdeclareInfoDto>> page = this.queryVipMbdeclareInfoList(queryMbDeclareListVo);
    List<VipMbdeclareInfoDto> vipMbdeclareInfoList = page.getData().getData();
    
    // 4. 如果没有导出权限，对敏感字段进行加密处理
    if (!hasExportPermission) {
        handleSensitiveInfoEncryption(vipMbdeclareInfoList);
    }
    
    // 5. 生成Excel文件
    // ...
}
```

#### 2. 权限检查
```java
private boolean checkYlExportPermission(String userAccount) {
    PersonDivision personDivision = new PersonDivision();
    personDivision.setAccount(userAccount);
    personDivision.setFlag(UnitConfigEnum.YL.getFlag());
    personDivision.setDr(DrEnum.VALID.getValue());
    personDivision.setIsexport("1"); // isexport = "1" 表示有导出权限
    
    List<PersonDivision> personDivisions = tPersonDivisionService.selectAll(personDivision);
    return personDivisions != null && !personDivisions.isEmpty();
}
```

#### 3. 加密处理
```java
private void handleSensitiveInfoEncryption(List<VipMbdeclareInfoDto> vipMbdeclareInfoList) {
    for (VipMbdeclareInfoDto dto : vipMbdeclareInfoList) {
        // 加密参保人姓名
        if (StringUtils.isNotBlank(dto.getName())) {
            dto.setName(encryptName(dto.getName()));
        }
        // 加密手机号码
        if (StringUtils.isNotBlank(dto.getMobile())) {
            dto.setMobile(encryptMobile(dto.getMobile()));
        }
        // 加密身份证号
        if (StringUtils.isNotBlank(dto.getIdcard())) {
            dto.setIdcard(encryptIdCard(dto.getIdcard()));
        }
        // 加密专家姓名
        if (StringUtils.isNotBlank(dto.getSecondusername())) {
            dto.setSecondusername(encryptName(dto.getSecondusername()));
        }
        // 加密专家手机号
        if (StringUtils.isNotBlank(dto.getSecondmobile())) {
            dto.setSecondmobile(encryptMobile(dto.getSecondmobile()));
        }
    }
}
```

## 三、权限配置说明

### 数据库表
`PersonDivision` 表

### 字段说明
| 字段名 | 类型 | 说明 |
|-------|------|------|
| account | VARCHAR | 用户账号 |
| flag | VARCHAR | 地区标识（15=榆林） |
| isexport | VARCHAR | 导出权限标识（1=有权限） |
| dr | SHORT | 逻辑删除（0=有效） |

### 配置方式
```sql
-- 为指定账号配置导出权限
UPDATE PersonDivision 
SET isexport = '1' 
WHERE account = '账号名' AND flag = '15' AND dr = 0;
```

## 四、测试要点

### 测试场景

#### 场景1：无导出权限账号导出
1. 前置条件：账号未配置导出权限（PersonDivision.isexport != '1'）
2. 操作：进入"榆林慢病管理" -> "申报综合查询"，点击【导出Excel】
3. 预期结果：
   - 导出的Excel中敏感字段已加密
   - 姓名显示为"张**"格式
   - 身份证号显示为"130182********0918"格式
   - 手机号显示为"154****2617"格式
   - 专家姓名已加密
   - 专家手机号已加密

#### 场景2：有导出权限账号导出
1. 前置条件：账号已配置导出权限（PersonDivision.isexport = '1'）
2. 操作：进入"榆林慢病管理" -> "申报综合查询"，点击【导出Excel】
3. 预期结果：
   - 导出的Excel中敏感字段显示原始值，未加密

#### 场景3：其他地市不受影响
1. 前置条件：使用非榆林地区的账号登录
2. 操作：进入其他地市的"申报综合查询"，点击【导出Excel】
3. 预期结果：
   - 导出功能正常，不进行权限检查和加密处理

### 边界测试

| 测试项 | 测试数据 | 预期结果 |
|-------|---------|---------|
| 空姓名 | name = null 或 "" | 不处理，保持原值 |
| 单字姓名 | name = "张" | 加密为"张**" |
| 长姓名 | name = "欧阳慕容" | 加密为"欧**" |
| 短身份证号 | idcard = "12345" | 不处理，保持原值 |
| 短手机号 | mobile = "123456" | 不处理，保持原值 |
