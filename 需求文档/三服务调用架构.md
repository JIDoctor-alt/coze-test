# 三服务调用架构详解

## 服务端口与职责

| 服务 | 端口 | 职责 | 是否转发请求 |
|------|------|------|-------------|
| **gateway** | 9001 | API 网关，接收外部请求 | ✅ 转发到 server |
| **auth** | 9093 | 认证服务，登录/登出/用户信息 | ✅ 转发到 server |
| **server** | 9091 | 核心业务服务 | ❌ 不转发 |

## 核心实现方式：HttpForwardUtil + RestTemplate

### 1. HttpForwardUtil 工具类

**位置：** 
- `gateway/src/main/java/com/picchealth/utils/HttpForwardUtil.java`
- `auth/src/main/java/com/picchealth/utils/HttpForwardUtil.java`

```java
@Component
@Slf4j
public class HttpForwardUtil {
    
    @Value("${forwardUrl:http://localhost:9091}")
    private String prefix;  // 转发目标地址
    
    @Autowired
    private RestTemplate restTemplate;

    /**
     * POST 请求转发
     * @param url 目标接口路径
     * @param request 原始请求（用于传递 Header）
     * @param requestVo 请求体
     * @param clazz 返回类型
     */
    public <T> T post(String url, HttpServletRequest request, Object requestVo, Class<T> clazz) {
        HttpHeaders headers = getHttpHeaders(request);
        HttpEntity requestEntity = new HttpEntity(requestVo, headers);
        ResponseEntity<T> responseEntity = restTemplate.exchange(
            prefix + url,  // 完整URL = prefix + url
            HttpMethod.POST, 
            requestEntity, 
            clazz
        );
        return responseEntity.getBody();
    }
    
    // 获取原始请求的 Header
    private HttpHeaders getHttpHeaders(HttpServletRequest request) {
        HttpHeaders headers = new HttpHeaders();
        Enumeration<String> names = request.getHeaderNames();
        while (names.hasMoreElements()) {
            String name = names.nextElement();
            headers.add(name, request.getHeader(name));
        }
        return headers;
    }
}
```

### 2. Gateway 调用 Server 示例

**gateway/src/main/java/com/picchealth/module/mb/api/LoginApi.java**

```java
@RestController
@RequestMapping(value = "/Login")
public class LoginApi extends BaseApi {

    @Resource
    private HttpForwardUtil httpForwardUtil;
    
    private String baseUrl = "/Login";  // 目标服务接口路径

    @PostMapping("/doLogin")
    ApiResponse login(@RequestBody LoginVo loginVo, HttpServletRequest request) {
        ParameterizedTypeReference<ApiResponse> responseType = 
            new ParameterizedTypeReference<ApiResponse>(){};
        
        // 转发到 server 的 /Login/doLogin
        return httpForwardUtil.post(baseUrl + "/doLogin", request, loginVo, responseType);
    }
    
    @PostMapping("/getUser")
    ApiResponse getUser(@RequestBody GetUserVo loginVo) {
        ParameterizedTypeReference<ApiResponse> responseType = 
            new ParameterizedTypeReference<ApiResponse>(){};
        
        // 转发到 server 的 /Login/getUser
        return httpForwardUtil.post(baseUrl + "/getUser", request, loginVo, responseType);
    }
}
```

```

              │

              │ 3. HttpForwardUtil.post("/Login/doLogin", ...)
              │
              ▼

              │

              │ 2. 操作数据库
              │ 3. 返回结果
              │
              ▼
         Gateway 返回给用户
```

## 如何实现新的服务间调用

### 场景：Server 需要调用 Auth 的接口

**方法一：添加 HttpForwardUtil（推荐）**

1. 在 server 添加配置：
```yaml
# application.yml
authUrl: http://localhost:9093
```

2. 创建工具类：
```java
@Component
public class AuthServiceClient {
    
    @Value("${authUrl:http://localhost:9093}")
    private String authUrl;
    
    @Autowired
    private RestTemplate restTemplate;
    
    public ApiResponse getUserInfo(String userId) {
        String url = authUrl + "/Login/getUser";
        // 构造请求
        Map<String, String> body = new HashMap<>();
        body.put("userId", userId);
        
        // 发送请求
        ResponseEntity<ApiResponse> response = restTemplate.postForEntity(
            url, body, ApiResponse.class
        );
        return response.getBody();
    }
}
```

3. 使用：
```java
@Service
public class SomeService {
    @Autowired
    private AuthServiceClient authServiceClient;
    
    public void doSomething() {
        ApiResponse user = authServiceClient.getUserInfo("123");
        // ...
    }
}
```

**方法二：使用 Feign（需要添加依赖）**

1. 添加依赖：
```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-openfeign</artifactId>
</dependency>
```

2. 启动类添加注解：
```java
@EnableFeignClients
@SpringBootApplication
public class ServerApplication { ... }
```

3. 创建 Feign Client：
```java
@FeignClient(name = "auth-service", url = "${authUrl:http://localhost:9093}")
public interface AuthClient {
    
    @PostMapping("/Login/getUser")
    ApiResponse getUser(@RequestBody GetUserVo vo);
    
    @PostMapping("/Login/logOut")
    ApiResponse logOut(@RequestBody GetUserVo vo);
}
```

4. 使用：
```java
@Service
public class SomeService {
    @Autowired
    private AuthClient authClient;
    
    public void doSomething() {
        GetUserVo vo = new GetUserVo();
        vo.setUserId("123");
        ApiResponse user = authClient.getUser(vo);
    }
}
```

## 总结

| 调用方向 | 实现方式 | 配置项 |
|---------|---------|--------|
| Gateway → Server | HttpForwardUtil + RestTemplate | `forwardUrl` |
| Auth → Server | HttpForwardUtil + RestTemplate | `forwardUrl` |
| Server → Auth | 无（可添加 RestTemplate 或 Feign） | - |

**当前架构特点：**
1. Gateway 和 Auth 作为网关层，负责转发请求到 Server
2. Server 是核心业务服务，不主动调用其他服务
3. 三个服务共享同一个数据库，数据操作不需要服务间调用
4. 使用 RestTemplate 进行 HTTP 调用，简单直接
