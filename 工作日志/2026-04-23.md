## 一、RAGFlow Wiki 集成调研

### 1. RAGFlow Wiki 功能
- RAGFlow v0.17+ 支持外部数据同步
- 支持平台：Confluence、Notion、Discord 等
- 可将外部知识库内容同步到 RAGFlow 进行 RAG 检索

### 2. Wiki 概念
- Wiki 是一种协作式知识管理平台
- 与 RAG 结合可实现：知识沉淀 + 智能检索 + 问答增强

### 3. PandaWiki 项目调研
- **项目地址**：https://github.com/chaitin/PandaWiki
- **开发者**：长亭科技
- **定位**：AI 大模型驱动的 Wiki 知识库系统
- **特点**：
  - 开箱即用，Wiki 编辑 + AI 问答一体化
  - 支持钉钉、飞书、企业微信集成
  - 持续维护更新

**推荐 PandaWiki** 而非 Ragflow-Plus，理由：

## 二、若依集成 RAGFlow 方案


| 流式 | 实时对话 | 逐字返回，用户体验好 |

## 三、RAG 技术方案调研

### 传统 RAG 之外的方案

| 方案 | 核心思路 | 适用场景 |
|------|----------|----------|
| **LongRAG** | 长上下文直接输入，减少切片损失 | 长文档问答 |
| **GraphRAG** | 构建知识图谱，多跳推理 | 复杂关联查询 |
| **Self-RAG** | 自我反思检索质量，迭代优化 | 高准确率要求 |
| **Hybrid RAG** | BM25 + 向量检索混合 | 通用企业知识库 |
| **Long Context** | 大模型直接处理长文本 | 无需检索的简单场景 |

### 选型建议
- 简单问答 → 传统 RAG
- 长文档 → LongRAG
- 多跳推理 → GraphRAG
- 高准确率 → Self-RAG
- 通用场景 → Hybrid RAG

## 五、exportYL 接口流程分析

### 完整流程
```
1. 医保账号检查
   └── 查询 PersonDivision 表判断用户是否绑定医保账号
   └── 有绑定 → 返回"数据导出需审批"

2. 查询申报数据
   └── queryVipMbdeclareInfoYlListNew()

3. 导出权限判断
   └── 查询 PersonDivision 表 isexport 字段
   └── 有权限 → 不脱敏
   └── 无权限 → 敏感信息加密

4. 敏感信息处理
   └── handleSensitiveInfoEncryption()

5. Excel 导出
   └── 全量导出 / 按ID导出
```

### 核心代码逻辑
```java
// 导出权限判断
personDivision.setIsexport("1");
List<PersonDivision> personDivisions = personDivisionService.selectAll(personDivision);
Boolean hasExportPermission = personDivisions != null && personDivisions.size() >= 1;

// 敏感信息处理
handleSensitiveInfoEncryption(vipMbdeclareInfoList, hasExportPermission);
```

## 六、代码学习

### 1. 反编译代码识别
```java
// Source code recreated from a .class file by IntelliJ IDEA
// (powered by FernFlower decompiler)
```
含义：打开的是 IDEA 反编译后的字节码文件，不是原始源码

### 2. DrEnum 枚举类
```java
VALID((short)0)    // 有效
INVALID((short)1)  // 无效
```
用途：数据库逻辑删除标识（dr 字段）

### 3. 行级权限（Row-Level）

- 示例：经办员只能查看本地区的申报数据
