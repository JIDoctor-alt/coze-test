小新：知道！就是妈妈看手机的时候，爸爸叫她，她听不见！

美伢：（生气）小新！你说谁呢！

妮妮：风间君，我懂了！就像我们读书的时候，会重点关注关键词一样！

正男：嗯嗯...我也明白了一点...

风间：好的，让我来详细解释！

### 深度展开

```
┌────────────────────────────────────────────────────────────┐
│  注意力机制原理                                            │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  Attention = Query × Key^T → Softmax → Value              │
│                                                            │
│  就像找餐厅：                                              │
│  Query(我想吃什么) → 查询意图                              │
│  Key(各餐厅特点) → 餐厅标签                                │
│  Value(餐厅信息) → 最终推荐                                │
│                                                            │
│  相似度计算 → 注意力权重 → 加权求和                        │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 一句话标准答案

**注意力机制让模型在处理每个词时，能自动关注到与它最相关的其他词。通过Query-Key-Value计算，得到加权和作为输出。**

---

**小新记忆法**

小新：哈哈，我懂了！就是看菜吃饭嘛！

风间：什么意思？

小新：想吃拉面就找拉面店，想吃汉堡就找汉堡店。这就是Query找Key嘛！

妮妮：哇，小新居然能理解！

风间：（无奈）虽然比喻不太准确...但大概意思对。

---

**生活比喻**

```
Attention = 图书馆找书系统

Query: "我想找一本关于恐龙的科普书"
1. 图书馆目录(Query)匹配图书标签(Key)
2. 计算相似度得到注意力分数
3. 根据分数加权返回书籍(Value)
4. 得到最相关的几本书

Attention就是：让每个词去找和它相关的词！
```

---

### 代码佐证

```python
import torch
import torch.nn.functional as F
import math

def attention(Q, K, V):
    """
    注意力机制核心计算
    
    Q: Query - 我要找什么
    K: Key - 对方是什么
    V: Value - 对方有什么
    """
    # 1. 计算Q和K的相似度
    scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(Q.size(-1))
    
    # 2. Softmax得到注意力权重
    attn_weights = F.softmax(scores, dim=-1)
    
    # 3. 加权求和得到输出
    output = torch.matmul(attn_weights, V)
    
    return output, attn_weights
```

**💡 一句话总结**

```
注意力机制 = 找相关 + 加权 = 让每个词"注意"重要的词！
```

## 1.3 为什么Decoder-only成为主流？

**面试官问法**
> "为什么现在的LLM都是Decoder-only架构？"
> "GPT和BERT有什么区别？"

---

**小新课堂**

风间：美伢阿姨你懂吗？

美伢：我天天用，当然懂！

### 三种架构对比

```
┌────────────────────────────────────────────────────────────┐
│  Encoder-Only (BERT) - 理解任务                             │
├────────────────────────────────────────────────────────────┤
│  输入: "这部电影很好看"                                      │
│  输出: 情感分类（正面）                                      │
│  特点: 双向注意力，看完整句话再判断                          │
│  适用: 分类、实体识别、句子匹配                              │
├────────────────────────────────────────────────────────────┤
│  Decoder-Only (GPT) - 生成任务                              │
├────────────────────────────────────────────────────────────┤
│  输入: "今天天气"                                            │
│  输出: "很好，适合出去玩"                                   │
│  特点: 单向注意力，只能看过去预测未来                        │
│  适用: 文本生成、对话                                       │
├────────────────────────────────────────────────────────────┤
│  Encoder-Decoder (T5) - 理解+生成                           │
├────────────────────────────────────────────────────────────┤
│  输入: 中文"苹果"                                           │
│  输出: 英文"Apple"                                          │
│  特点: 编码器理解输入，解码器生成输出                        │
│  适用: 翻译、摘要、问答                                      │
└────────────────────────────────────────────────────────────┘
```

### 一句话标准答案

**Decoder-only成为主流是因为：1）足够简单，训练稳定；2）足够强大，可以完成各种任务；3）推理可以用KV Cache优化；4）Next Token Prediction任务简单，容易scale up。**

**💡 一句话总结**

### SFT vs RLHF对比

```
┌────────────────────────────────────────────────────────────┐
│  SFT (Supervised Fine-tuning)                             │
├────────────────────────────────────────────────────────────┤
│  方法: 有监督学习                                          │
│  数据: (输入, 标准输出) 对                                  │
│  Loss: CrossEntropy(预测, 标准答案)                         │
│                                                            │
│  优点: 简单直接，训练稳定                                   │
│  缺点: 只能模仿，无法超越数据                              │
├────────────────────────────────────────────────────────────┤
│  RLHF (Reinforcement Learning from Human Feedback)       │
├────────────────────────────────────────────────────────────┤
│  方法: 强化学习                                            │
│  数据: 偏好数据 (哪个回答更好)                             │
│  信号: Reward Model                                       │
│                                                            │
│  优点: 可以学习隐式偏好，超越数据质量                      │
│  缺点: 训练复杂，容易reward hacking                       │
└────────────────────────────────────────────────────────────┘
```

**SFT是有监督学习，让模型模仿正确答案；RLHF是强化学习，用奖励信号引导模型生成更好回答。SFT告诉模型"标准答案是什么"，RLHF告诉模型"什么样的回答更好"。**

## 2.3 LoRA原理？

**面试官问法**

---

**小新课堂**

风间：你？

小新：比如做作业，我只做重要的题，不全做！

### LoRA核心原理

```
┌────────────────────────────────────────────────────────────┐
│  LoRA: Low-Rank Adaptation                                │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  全参数微调：                                              │
│  更新: W = W₀ + ΔW                                        │
│  W₀: d×d (冻结)                                          │
│  ΔW: d×d (需要更新)                                       │
│                                                            │
│  LoRA微调：                                                │
│  W = W₀ + BA                                              │
│  A: d×r                                                   │
│  B: r×d                                                   │
│  r << d                                                   │
│                                                            │
│  参数量对比（d=4096, r=8）：                                │
│  全参数: 4096 × 4096 = 16M                                │
│  LoRA: 2 × 8 × 4096 = 64K                                 │
│  只用0.4%的参数！                                          │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 一句话标准答案

**LoRA通过低秩分解冻结预训练权重，只训练A和B两个低秩矩阵。假设原始权重W₀是d×d，全参数训练需要更新d²个参数，LoRA只需要2×r×d个参数（r<<d），显存大幅减少。**

---

**小新记忆法**

小新：就像玩乐高！

妮妮：怎么说？

小新：
- 原来要搭一个16×16的大楼（全参数微调）
- LoRA只搭一个8×1的缩小版，然后放大
- 省了好多积木！

风间：虽然不准确，但抓住了核心思想。

---

**💡 一句话总结**

```
LoRA = 冻结W₀ + 只训BA = 低秩适应 = 省显存！
```

## 3.2 RAG vs 微调 vs 提示工程怎么选？

### 选型指南

```
┌────────────────────────────────────────────────────────────┐
│                    选型指南                                 │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  【提示工程】                                              │
│  适用: 简单任务（格式、风格控制）                           │
```

### KV Cache原理

```
┌────────────────────────────────────────────────────────────┐
│  KV Cache = Key-Value Cache                               │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  没有KV Cache:                                            │
│  Token N: 需要重新计算前N-1个token的所有K,V              │
│  时间复杂度: O(n²)                                        │
│                                                            │
│  有KV Cache:                                              │
│  Token N: 只计算当前token的K,V，用缓存的K,V               │
│  时间复杂度: O(n)                                         │
│  空间复杂度: O(n²) ← 存储所有历史K,V                      │
│                                                            │
│  LLaMA-7B KV Cache分析:                                   │
│  每个token的KV: ~1MB                                      │
│  2048 tokens: ~2GB                                        │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 一句话标准答案

**KV Cache是缓存已计算的Key和Value，避免重复计算。在自回归生成时，通过缓存KV可以让计算量从O(n²)降到O(n)，但会占用大量显存。**

## 10.1 手写Self-Attention

**面试官问法**
> "请手写Self-Attention"
> "写出完整代码"

---

**广志课堂**

广志：让我来写一下Self-Attention的代码！

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class SelfAttention(nn.Module):
    """手写Self-Attention"""
    
    def __init__(self, d_model, nhead=8):
        super().__init__()
        assert d_model % nhead == 0
        
        self.d_model = d_model
        self.nhead = nhead
        self.d_k = d_model // nhead
        
        # Q, K, V 投影
        self.W_Q = nn.Linear(d_model, d_model)
        self.W_K = nn.Linear(d_model, d_model)
        self.W_V = nn.Linear(d_model, d_model)
        
        # 输出投影
        self.W_O = nn.Linear(d_model, d_model)
    
    def forward(self, x, mask=None):
        batch_size, seq_len, _ = x.shape
        
        # Step 1: 线性投影得到 Q, K, V
        Q = self.W_Q(x)
        K = self.W_K(x)
        V = self.W_V(x)
        
        # Step 2: 分成多头
        Q = Q.view(batch_size, seq_len, self.nhead, self.d_k).transpose(1, 2)
        K = K.view(batch_size, seq_len, self.nhead, self.d_k).transpose(1, 2)
        V = V.view(batch_size, seq_len, self.nhead, self.d_k).transpose(1, 2)
        
        # Step 3: 计算注意力分数
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)
        
        # Step 4: 应用mask
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        
        # Step 5: Softmax得到注意力权重
        attn_weights = F.softmax(scores, dim=-1)
        
        # Step 6: 加权求和
        context = torch.matmul(attn_weights, V)
        
        # Step 7: 合并多头
        context = context.transpose(1, 2).contiguous().view(batch_size, seq_len, self.d_model)
        
        # Step 8: 输出投影
        output = self.W_O(context)
        
        return output, attn_weights
```

### 一句话标准答案

**Self-Attention = QKV投影 → 分头 → 计算注意力 → 加权求和 → 合并输出！**

---

**💡 一句话总结**

```
Self-Attention = QKV三步曲 = 投影→分头→加权！
```
