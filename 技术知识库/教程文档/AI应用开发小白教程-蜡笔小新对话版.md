## 第二章：API调用基础

### 📞 API是什么？

**小新**：风间，API是什么啊？

**风间**（推推眼镜）：API就像是...打电话给AI服务员！

**小新**：打电话？

**风间**：没错！你告诉AI你要什么（Prompt），然后AI回答你（Response）。

**小新**：就像我打电话给外卖店点餐一样！

**风间**：对！差不多就是这个意思！

**妮妮**：我来用代码演示一下！

```python
# 打电话给AI
client = OpenAI(api_key="小新的密钥")

# 告诉AI你要什么
messages = [
    {"role": "user", "content": "什么是Python？"}
]

# AI回答
response = client.chat.completions.create(
    model="gpt-4o",
    messages=messages
)
```

**小新**：看不懂...

**妮妮**：没关系，你只需要知道：
- API就是让程序调用AI的接口
- 需要一个密钥（就像身份证）
- 发送请求，获得回答

**小新**：哦！就像寄信和收信一样！

## 第三章：Function Calling 让AI能动起来

**小新**：风间，Function Calling是什么？

**风间**：就是让AI能够"调用工具"做事！

**小新**：什么意思？

**风间**：普通AI只会"说话"，Function Calling让AI能"做事"！

**妮妮**（举例）：比如你说"帮我查一下天气"——

**小新**：然后呢？

**妮妮**：
- 普通AI会说："今天天气晴，温度25度"
- Function Calling会说："我帮你查一下" → 真的调用天气API → "今天天气晴，温度25度"

**小新**：哇！差别这么大！

**正男**：因为Function Calling让AI能使用工具了！

**小新**：就像AI长出了手脚一样！

**妮妮**：没错！AI能搜索、能查数据库、能发邮件了！

## 第四章：Structured Output 格式化输出

## 第六章：对话应用开发

### 🌊 流式输出

**小新**：妮妮，为什么有些AI回答是慢慢显示出来的？

**妮妮**：那叫流式输出！就像打字机效果！

**小新**：打字机！我知道！

### 🧠 会话管理

**小新**：为什么有些AI不记得我之前说的话？

**风间**：那是因为没有会话管理！

**小新**：什么意思？

**妮妮**（举例）：

没有会话管理：
```
小新：我叫小明
AI：你好好！
小新：我叫什么？
AI：抱歉我不知道...
```

有会话管理：
```
小新：我叫小明，邮箱是xiaoming@example
AI：好的小明，我记住了！
小新：我邮箱是什么？
AI：你的邮箱是xiaoming@example
```

**小新**：原来如此！AI也要记住我说过的话！

### 📦 上下文窗口优化

**小新**：风间，AI的"记忆"有限吗？

**风间**：没错！AI的上下文窗口就像人的短期记忆，容量有限！

**妮妮**：对话太长了，AI可能忘记之前说过的事！

**小新**：那怎么办？

**风间**：有几种优化方法：
- 只保留最近的对话
- 把早期对话总结成摘要
- 提取并保留关键信息

**小新**：就像整理房间一样，把重要的留下，不重要的扔掉！

### 🏷️ 向量化

**小新**：风间，什么是向量化？

**风间**（推推眼镜）：就是给文档贴上"数学标签"！

**小新**：数学标签？

**妮妮**（举例）：就像给食物贴标签——

### 🗄️ 向量数据库

**小新**：广志，向量数据库是什么？
