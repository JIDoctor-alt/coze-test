## 1.1 什么是大模型微调？

### 📢 一句话人话

**微调就是"调教"一个已经学会说话的大模型，让它更懂你的专业，更听话、更靠谱。**

---

### 🎯 生活比喻

想象你新招了一个**全能秘书**。这位秘书已经：
- 学会了所有基础办公技能（打字、文档排版、邮件撰写）
- 懂得基本职场礼仪和沟通话术
- 能处理各种通用问题

但是！这位秘书对你的**具体业务一无所知**：
- 不知道你们公司的产品叫什么
- 不懂你们行业的专业术语
- 不了解你们老板的说话风格

**微调就相当于**：花几天时间专门培训这个秘书，让它熟悉你的公司、你的业务、你的风格。培训完成后，它就能帮你处理各种具体工作了，而且用起来特别顺手！

---

### 💻 核心概念

大模型微调（Fine-tuning）是指在**已经预训练好的大语言模型（基座模型）**基础上，使用**特定领域或特定任务的数据**进行额外训练，使模型获得特定能力的过程。

#### 三个关键术语澄清：

| 术语 | 解释 | 类比 |
|------|------|------|
| **预训练（Pre-training）** | 用海量通用数据训练模型，让它学会"语言"和"常识" | 九年义务教育，学的是基础知识 |
| **基座模型（Base Model）** | 预训练完成后的原始模型，还没对齐人类偏好 | 刚毕业的大学生，知识面广但没经验 |
| **微调（Fine-tuning）** | 用专业数据教会模型特定技能 | 新员工入职培训 |

#### 技术细节：

![编码](https://www.coze.cn/s/PIa3Uh8C3zw/)

```python
# 微调的本质是：继续训练，但有讲究
# 1. 不是从零开始训练（否则之前学的就忘了）
# 2. 用特定数据教它新技能
# 3. 控制好学习程度（学太少没效果，学太多会把原来会的忘掉）

# 伪代码表示：
base_model = load_pretrained_model()  # 加载已经学好的基座模型
for epoch in range(num_epochs):
    for batch in professional_data:  # 用专业数据训练
        loss = model.forward(batch)
        optimizer.step()  # 更新模型参数
```

| 结果 | 可能得到一个平庸的通用模型 | 可能得到一个优秀的专业模型 |

**❓ 为什么不直接训练新模型？**

## 1.2 为什么要微调？

### 📢 一句话人话

**微调能让通用AI变成你的专属AI，更懂你的业务、你的风格、你的需求。**

---

### 🎯 生活比喻

我们继续用秘书的比喻：

**没有微调的AI** = 刚毕业的名校生
- ✅ 什么都会一点
- ✅ 礼貌得体
- ❌ 不懂你的专业
- ❌ 回答太泛泛
- ❌ 风格不对

**微调后的AI** = 你亲手培养的资深助理
- ✅ 专业知识扎实
- ✅ 懂你的行话
- ✅ 风格符合你的喜好
- ✅ 回答精准实用
- ⚠️ 需要适度监督（可能学歪）

---

### 💻 核心概念

#### 微调能解决的问题：

1. **垂直领域专业化**
   ```
   通用ChatGPT → 医疗助手、法律顾问、金融分析
   ```

2. **特定风格塑造**
   ```
   严肃公文 → 活泼客服
   学术论文风 → 通俗科普风
   ```

3. **私有知识注入**
   ```
   模型 + 你的公司文档 = 私有知识助手
   ```

4. **特定任务优化**
   ```
   通用对话 → 专门做病史采集 / 合同审核 / 客服话术优化
   ```

#### 什么时候必须微调？

| 场景 | 提示工程能解决吗？ | 需要微调吗？ |
|------|-------------------|--------------|
| 简单问答 | ✅ 可以 | ❌ 不需要 |
| 风格调整 | ✅ 可以 | ❌ 不太需要 |
| 领域专业术语 | ⚠️ 勉强 | ✅ 建议 |
| 必须输出特定格式 | ⚠️ 勉强 | ✅ 需要 |
| 私有知识问答 | ❌ 不行 | ✅ 必须 |
| 持续多轮交互 | ⚠️ 不稳定 | ✅ 需要 |
| 延迟/成本敏感 | ❌ 提示太长太慢 | ✅ 需要 |

### ⚠️ 小白易懵点

**❓ 我听说提示工程（Prompt Engineering）也能实现类似效果，为什么要微调？**

| 对比维度 | 提示工程 | 微调 |
|----------|----------|------|
| 每次调用成本 | 高（提示越长，token越多，费用越高） | 低（几乎没有额外成本） |
| 响应速度 | 慢（提示太长影响生成） | 快（正常生成） |
| 效果稳定性 | 较差（依赖提示质量） | 稳定（能力内化到模型） |
| 保密性 | 差（每次都要传数据） | 好（数据可以完全本地） |
| 适用场景 | 简单、临时、实验性需求 | 稳定、长期、核心业务 |

**简单判断**：
- 如果你只是临时用用 → 用提示工程
- 如果这是你的核心业务 → 建议微调

## 1.3 微调 vs 提示工程 vs RAG vs Agent

### 📢 一句话人话

**这四种技术是递进关系：提示工程是"说话技巧"，微调是"培训上岗"，RAG是"随身带资料"，Agent是"有了手脚"。**

---

### 🎯 生活比喻

用一个医生的例子来解释：

| 技术 | 比喻 | 解释 |
|------|------|------|
| **提示工程** | 教医生"怎么跟病人说话" | 改变提问方式，让AI表现更好 |
| **微调** | 送医生去读研究生 | 让AI真正学会专业知识 |
| **RAG** | 给医生一本参考书 | AI可以从外部知识库查找信息 |
| **Agent** | 给医生配备护士和检查设备 | AI可以调用工具、采取行动 |

---

### 💻 核心概念详解

#### 1. 提示工程（Prompt Engineering）

**一句话**：通过精心设计输入提示来引导模型输出更好的结果。

![编码](https://www.coze.cn/s/PIa3Uh8C3zw/)

```python
# 不好的提示
"帮我写一封邮件"

# 好的提示（加了角色、格式、约束）
"""
你是一位资深商务律师。请帮我写一封邮件给合作方：
1. 内容：关于合同延期事宜
2. 语气：专业但不生硬
3. 格式：包含称呼、正文、落款
4. 字数：200字以内
"""
```

**优点**：零成本，立即见效
**缺点**：不稳定，效果有限

---

#### 2. 微调（Fine-tuning）

**一句话**：用专业数据训练模型，让能力"固化"进去。

```python
# 微调数据示例
{
    "instruction": "你是医疗助手",
    "input": "患者：男，45岁，持续头痛3天",
    "output": "建议：首先需要排除高血压引起的头痛，建议测量血压..."
}

# 训练后，模型就能"本能"地按照医疗助手的风格回答
```

**优点**：效果好，稳定可靠
**缺点**：需要数据和算力

---

#### 3. RAG（检索增强生成）

**一句话**：让AI在回答时能"翻书查资料"。

```
用户问题 → 检索相关文档 → 结合文档生成回答
```

```python
# RAG 工作流程
# 1. 准备知识库（你的公司文档）
documents = [
    "我们的产品叫XX，有以下功能...",
    "退货政策：7天内可退...",
    "客服联系方式：400-xxx..."
]

# 2. 用户提问时，先检索相关文档
relevant_docs = search(user_question, documents)

# 3. 让AI结合检索结果回答
response = llm.answer(question=user_question, context=relevant_docs)
```

**优点**：可以随时更新知识，不需重新训练
**缺点**：依赖检索质量，增加延迟

---

#### 4. Agent（智能体）

**一句话**：给AI装上"手脚"，让它能做实事。

```
AI 思考 → 调用工具 → 获取结果 → 继续思考 → 输出
```

```python
# Agent 示例：AI可以帮你操作电脑
user: "帮我查一下明天的天气，然后告诉我要不要带伞"

# Agent 执行流程：
# 1. 调用天气API → 获取天气数据
# 2. 分析天气 → 有雨
# 3. 输出："明天有雨，建议带伞"
```

---

### 🔄 四者对比与组合

| 维度 | 提示工程 | 微调 | RAG | Agent |
|------|----------|------|-----|-------|
| 技术难度 | ⭐ | ⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐ |
| 成本 | 免费 | GPU费用 | 向量数据库 | 工具接口费用 |
| 灵活性 | 高 | 低 | 高 | 高 |
| 知识更新 | 差 | 差 | 好 | 好 |
| 实时性 | 无 | 无 | 有 | 有 |
| 能动性 | 无 | 无 | 无 | 有 |

#### 组合使用（最常见）：

```
微调 + RAG = 最强组合
         ↓
先用微调让AI"懂行"
再配RAG让它能查最新资料
```

```python
# 实际生产环境通常这样组合：
class ProfessionalAI:
    def __init__(self):
        self.model = fine_tuned_model()      # 微调模型（专业能力）
        self.knowledge_base = RAG_system()   # RAG系统（实时知识）
    
    def answer(self, question):
        # 先查知识库
        context = self.knowledge_base.search(question)
        
        # 结合知识回答（用微调后的专业风格）
        response = self.model.answer(question, context)
        
        return response
```

## 1.4 代价收益评估

### 📢 一句话人话

**微调前先算账：花多少？能得到什么？值不值？**

---

### 🎯 生活比喻

就像装修房子：

| 装修类型 | 比喻 | 类比 |
|----------|------|------|
| 简单软装 | 提示工程 | 换个窗帘、摆件，轻投入小改变 |
| 全屋翻新 | 微调 | 换地板、刷墙，大投入大改变 |

**评估原则**：预算有限时，先做投入产出比高的！

### 💰 成本计算

#### 1. 算力成本（最大的开销）

![编码](https://www.coze.cn/s/PIa3Uh8C3zw/)

```python
# 粗略估算公式：
GPU显存需求 ≈ (模型参数量的20-30%) × 量化系数

# 例如：7B参数的模型
# 全精度(fp32)：7B × 4字节 ≈ 28GB（需要A100 40GB）
# 半精度(fp16)：7B × 2字节 ≈ 14GB（需要3090/4090）
# INT8量化：7B × 1字节 ≈ 7GB（需要3060即可）
# INT4量化：7B × 0.5字节 ≈ 3.5GB（消费级显卡够用）

# 训练显存需求（通常是推理的3-5倍）
训练显存 ≈ 推理显存 × (1 + 数据并行 + 梯度存留)
```

#### 2. 时间成本

| 模型规模 | 全量微调时间 | LoRA微调时间 | QLoRA时间 |
|----------|-------------|--------------|-----------|
| 7B | 2-4小时 | 30分钟-2小时 | 1-3小时 |
| 13B | 4-8小时 | 1-4小时 | 2-5小时 |
| 70B | 1-3天 | 4-12小时 | 8-24小时 |

#### 3. 云服务价格参考

```bash
# AWS/GCP/Lambda等云GPU价格（仅供参考，实际会有变动）

# 单卡价格估算
A100 80GB: 约 $2-3/小时
A100 40GB: 约 $1.5-2/小时
3090/4090: 约 $0.5-1/小时
H100: 约 $4-6/小时

# 7B模型微调成本估算
- LoRA微调：约 $0.5-2/次
- 全量微调：约 $5-15/次
```

### 📊 收益评估

#### 量化收益维度：

| 收益类型 | 衡量指标 | 计算方式 |
|----------|----------|----------|
| **效率提升** | 节省人工时间 | 时间 × 人员成本 |
| **质量提升** | 任务准确率 | 提升比例 × 业务价值 |
| **成本节约** | API调用减少 | 原调用成本 - 微调成本 |
| **数据安全** | 隐私风险降低 | 风险敞口 × 风险权重 |

#### ROI计算示例：

```python
# 场景：开发一个医疗问答助手

# 成本项
gpu_cost = 50  # 云GPU费用（美元）
human_time = 40  # 人工数据标注时间（小时）
hourly_rate = 30  # 工程师时薪（美元）
total_cost = gpu_cost + human_time * hourly_rate  # 50 + 1200 = $1250

# 收益项
daily_queries = 500  # 每天查询量
api_cost_per_query = 0.01  # API每次调用成本（美元）
deployed_days = 365  # 部署天数
replaced_queries_ratio = 0.7  # 能被替代的查询比例

# 计算收益
daily_saving = daily_queries * api_cost_per_query * replaced_queries_ratio
yearly_saving = daily_saving * deployed_days  # 0.7 × 365 = $2555

# ROI
roi = (yearly_saving - total_cost) / total_cost * 100  # 约104%
payback_period = total_cost / daily_saving  # 约6个月回本
```

### 🎯 决策树：要不要微调？

```
开始
  │
  ├─ 这个需求是长期稳定业务吗？
  │    │
  │    ├─ 否 → 用提示工程先验证
  │    │
  │    └─ 是 → 继续问
  │
  ├─ 现有模型+提示工程能满足吗？
  │    │
  │    ├─ 能 → 暂不微调
  │    │
  │    └─ 不能 → 继续问
  │
  ├─ 评估ROI为正吗？
  │    │
  │    ├─ 是 → 推荐微调
  │    │
  │    └─ 否 → 考虑RAG或暂缓
  │
  └─ 有足够数据（100条以上高质量数据）吗？
       │
       ├─ 是 → 可以开始微调
       │

```

## 2.1 全量微调（Full Fine-tuning）

### 📢 一句话人话

**全量微调就是把所有参数都重新训练一遍，就像把整个秘书送去回炉重造。**

- 但也最容易"学傻"（过拟合）
```

**优点**：能彻底改变模型行为，效果最好

```python
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch

def freeze_parameters(model, freeze_embeddings=True, freeze_layers=None):
    """冻结模型参数
    
    Args:
        model: 模型
        freeze_embeddings: 是否冻结嵌入层
        freeze_layers: 冻结前N层，None表示全部训练
    """
    freeze_count = 0
    
    for name, param in model.named_parameters():
        should_freeze = False
        
        # 冻结嵌入层
        if freeze_embeddings and "embed" in name.lower():
            should_freeze = True
        
        # 冻结前N层
        if freeze_layers is not None:
            # 从名称中提取层号
            for i in range(freeze_layers):
                if f".{i}." in name:
                    should_freeze = True
                    break
        
        if should_freeze:
            param.requires_grad = False
            freeze_count += 1
    
    return model

# 使用示例
model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b-hf")

# 只训练最后两层
model = freeze_parameters(model, freeze_embeddings=True, freeze_layers=26)

# 检查可训练参数
trainable = []
frozen = []
for name, param in model.named_parameters():
    if param.requires_grad:
        trainable.append(name)
    else:
        frozen.append(name)

print(f"可训练层数: {len(trainable)}")
print(f"冻结层数: {len(frozen)}")
```

## 2.3 LoRA：核心原理详解

### 📢 一句话人话

**LoRA就是"只学技巧，不改根基"，像给大脑装个外挂模块，学会后还能拔掉。**

---

### 🎯 生活比喻

```
LoRA = 笔记本外挂硬盘

基座模型 = 电脑原装硬盘（已存满数据，不能轻易改动）

LoRA = 外接的移动硬盘

- 不用动原装硬盘
- 新学的知识存在外挂上
- 想用新技能就插上
- 想恢复原状就拔掉
- 随时切换，互不影响
```

**这就是LoRA的核心思想**：不直接修改原模型参数，而是添加"旁路"来学习新知识！

---

### 💻 核心概念

#### 数学原理（简化版）：

```
传统微调：直接修改权重
W' = W + ΔW  （W是原始权重，ΔW是变化量）

LoRA：不直接修改，而是添加旁路
W' = W + B × A
     （A和B是新学的两个小矩阵）

举例：
- 原始 W：[1000 × 1000] = 100万个参数
- LoRA添加： A：[1000 × r] + B：[r × 1000]
- 当 r = 8 时： A + B = 1000×8 + 8×1000 = 16000个参数
- 参数减少：100万 → 1.6万（约1/60）
```

```python
# LoRA的核心代码实现
import torch
import torch.nn as nn

class LoRALayer(nn.Module):
    def __init__(self, in_features, out_features, rank=8, alpha=16):
        super().__init__()
        self.rank = rank
        self.alpha = alpha
        self.scaling = alpha / rank
        
        # A矩阵：用随机数初始化
        self.A = nn.Parameter(torch.randn(in_features, rank))
        # B矩阵：初始化为0（很重要！）
        self.B = nn.Parameter(torch.zeros(rank, out_features))
    
    def forward(self, x):
        # LoRA的前向传播
        # 原计算: y = x @ W
        # LoRA计算: y = x @ W + x @ (A @ B) * scaling
        return x @ (self.A @ self.B) * self.scaling

# 实际上LoRA是用在注意力层
class LoRALinear(nn.Module):
    def __init__(self, linear, rank=8, alpha=16, dropout=0.1):
        super().__init__()
        self.linear = linear
        self.rank = rank
        self.alpha = alpha
        
        # 冻结原始权重
        self.linear.weight.requires_grad = False
        if self.linear.bias is not None:
            self.linear.bias.requires_grad = False
        
        # LoRA参数
        self.lora_A = nn.Parameter(linear.weight.new_zeros((rank, linear.in_features)))
        self.lora_B = nn.Parameter(linear.weight.new_zeros((linear.out_features, rank)))
        self.dropout = nn.Dropout(dropout)
        
        # 初始化
        nn.init.normal_(self.lora_A, std=1 / self.rank)
        
        self.scaling = alpha / rank
    
    def forward(self, x):
        # 原始输出 + LoRA增量
        return self.linear(x) + self.dropout(x @ self.lora_A.T @ self.lora_B.T) * self.scaling
```

---

### 📊 LoRA关键参数

| 参数 | 含义 | 推荐值 | 说明 |
|------|------|--------|------|
| `rank` (r) | 秩，决定LoRA矩阵大小 | 4-64 | 越大越强，但可训练参数越多 |
| `alpha` (α) | 缩放因子 | rank的2倍 | 控制LoRA影响强度 |
| `target_modules` | 应用LoRA的模块 | q_proj, v_proj | 通常是注意力层 |
| `dropout` | Dropout率 | 0.0-0.1 | 防止过拟合 |

```
参数关系：可训练参数 ≈ 2 × rank × input_dim × output_dim / 1M (百万)

例如：7B模型应用LoRA到QKV投影，rank=8
每个注意力头的可训练参数 ≈ 2 × 8 × 4096 × 4096 / 1M ≈ 256M
```

---

## 2.4 QLoRA：量化+LoRA的强强联合

### 📢 一句话人话

**QLoRA就是LoRA的"省内存版"，让普通显卡也能训练大模型。**

---

### 🎯 生活比喻

```
LoRA = 只买一本笔记本（不买新电脑）
QLoRA = 买一本笔记本 + 用云存储（更省钱）

本质：用"压缩"技术把大模型变小，再用LoRA学习新知识
```

---

### 💻 核心概念

#### QLoRA的三板斧：

```
1. NF4量化（Normal Float 4-bit）
   - 把32位数字压缩成4位
   - 几乎不损失精度
   
2. 双重量化（Double Quantization）
   - 对量化参数也量化
   - 进一步省显存
   
3. 分页优化（Page Optimization）
   - 处理梯度检查点溢出
   - 稳定训练大模型
```

```python
# QLoRA 技术实现
from transformers import BitsAndBytesConfig
import torch

# 4bit量化配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,                    # 4位加载

    bnb_4bit_compute_dtype=torch.bfloat16,  # 计算用16位
    bnb_4bit_use_double_quant=True,       # 双重量化
)

# 加载量化模型
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    quantization_config=bnb_config,        # 传入量化配置
    device_map="auto"
)

# 然后应用LoRA（和普通LoRA一样）
model = get_peft_model(model, lora_config)
```

| 模型规模 | 全精度(fp32) | 半精度(fp16) | QLoRA(4bit) |

| 7B | 28GB | 14GB | ~6GB |
| 13B | 52GB | 26GB | ~12GB |
| 33B | 132GB | 66GB | ~24GB |
| 70B | 280GB | 140GB | ~48GB |

**这就是QLoRA的意义**：让70B模型在消费级4090上训练成为可能！

    r=64,
    lora_alpha=128,

    # 构建对话格式
    prompts = []
    for instruction, input_text, output in zip(
        examples["instruction"],
        examples["input"],
        examples["output"]
    ):
        prompt = f"### 指令:\n{instruction}\n\n### 输入:\n{input_text}\n\n### 回答:\n{output}"
        prompts.append(prompt)
    
    # 分词
    model_inputs = tokenizer(
        prompts,
        max_length=512,
        truncation=True,
        padding="max_length"
    )
    model_inputs["labels"] = model_inputs["input_ids"].copy()
    return model_inputs

dataset = load_dataset("json", data_files="train.json")["train"]
tokenized_dataset = dataset.map(preprocess_function, batched=True)

# 6. 训练配置
training_args = TrainingArguments(
    output_dir="./qlora_output",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    optim="paged_adamw_32bit",     # 页面优化器
    learning_rate=2e-4,
    fp16=False,
    bf16=True,                     # BF16精度
    max_grad_norm=0.3,
    warmup_ratio=0.03,
    lr_scheduler_type="cosine",
    logging_steps=10,
    save_steps=100,
)

# 7. 开始训练
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=tokenized_dataset,
    data_collator=DataCollatorForLanguageModeling(tokenizer, mlm=False),
)

trainer.train()

# data_cleaning.py
import json
import re

class DataCleaner:
    def __init__(self):
        self.errors = []
    
    def clean(self, data):
        """清洗数据"""
        cleaned = []
        
        for idx, item in enumerate(data):
            try:
                # 1. 格式检查
                if not self.check_format(item):
                    self.errors.append(f"[{idx}] 格式错误")
                    continue
                
                # 2. 长度检查
                if not self.check_length(item):
                    self.errors.append(f"[{idx}] 长度异常")
                    continue
                
                # 3. 内容检查
                if not self.check_content(item):
                    self.errors.append(f"[{idx}] 内容问题")
                    continue
                
                # 4. 去重
                if self.is_duplicate(item, cleaned):
                    self.errors.append(f"[{idx}] 重复数据")
                    continue
                
                cleaned.append(item)
                
            except Exception as e:
                self.errors.append(f"[{idx}] {str(e)}")
        
        return cleaned
    
    def check_format(self, item):
        """检查格式"""
        required_fields = ["instruction", "input", "output"]
        for field in required_fields:
            if field not in item:
                return False
        return True
    
    def check_length(self, item):
        """检查长度"""
        # output长度：50-2000字为宜
        output_len = len(item["output"])
        if output_len < 10 or output_len > 4000:
            return False
        
        # instruction长度：<500字
        instruction_len = len(item["instruction"])
        if instruction_len > 500:
            return False
        
        return True
    
    def check_content(self, item):
        """检查内容质量"""
        output = item["output"]
        
        # 检查是否为空
        if not output.strip():
            return False
        
        # 检查是否包含占位符
        placeholders = ["[此处答案]", "待补充", "请填写", "xxx", "..."]
        for placeholder in placeholders:
            if placeholder in output:
                return False
        
        # 检查是否太通用（复制instruction）
        if output.strip() == item["instruction"].strip():
            return False
        
        return True
    
    def is_duplicate(self, item, existing):
        """去重（简单版）"""
        # 用前50字去重
        content_hash = hash(item["output"][:50])
        return False  # 简化版，不做完整检查

# 使用

### 💻 增强方法

#### 1. 同义词替换

```python
import random

def synonym_replacement(text, n=3):
    """同义词替换"""
    
    # 简化的同义词词典（实际应用中用更完整的）
    synonyms = {
        "非常": ["十分", "特别", "极其"],
        "重要": ["关键", "要紧", "核心"],
        "建议": ["推荐", "提倡", "主张"],
        "治疗": ["诊疗", "医治", "疗法"],
        "检查": ["检测", "查验", "诊查"],
    }
    
    words = text.split()
    replaced_words = []
    
    for word in words:
        if word in synonyms and random.random() < 0.2:  # 20%概率替换
            replaced = random.choice(synonyms[word])
            replaced_words.append(replaced)
        else:
            replaced_words.append(word)
    
    return "".join(replaced_words)
```

#### 2. 指令改写

```python
def rewrite_instruction(instruction, style="formal"):
    """改写instruction"""
    
    # 定义改写模板
    templates = {
        "formal": [
            "请详细解释{topic}",
            "关于{topic}，请给出专业说明",
            "请问{topic}相关的知识"
        ],
        "casual": [
            "说说{topic}呗",
            "{topic}是什么鬼？",
            "帮我看看{topic}的情况"
        ],
        "medical": [
            "作为医生，如何看待{topic}",
            "从医学角度解释{topic}",
            "{topic}在临床中怎么处理"
        ]
    }
    
    # 提取主题
    topic = instruction.replace("你是一名医生，请回答：", "")
    topic = topic.replace("请回答：", "")
    
    # 生成改写
    new_templates = templates.get(style, templates["formal"])
    return random.choice(new_templates).format(topic=topic)
```

def back_translation_augment(text):
    """回译增强：中文→英文→中文"""
    
    # 简化版本：使用模板生成不同表达
    templates = [
        lambda t: f"根据您的问题，{t}",

        lambda t: f"针对{t}，以下是建议",
        lambda t: f"对于{t}这个话题，{t}的情况是...",

    ]

```

#### 4. 完整数据增强流程

```python
# data_augmentation.py
import json
import random

def augment_dataset(data, augment_ratio=3):
    """完整的数据增强流程"""
    
    augmented = []
    
    for item in data:
        # 1. 原始数据
        augmented.append(item)
        
        # 2. 增强augment_ratio倍
        for _ in range(augment_ratio):
            new_item = item.copy()
            
            # 改写instruction
            if random.random() < 0.5:
                new_item["instruction"] = rewrite_instruction(item["instruction"])
            
            # 改写output开头（保持核心内容）
            if random.random() < 0.5:
                prefixes = [
                    "根据您的情况，",
                    "建议您",
                    "对于这种情况，",
                    "从专业角度分析，"
                ]
                new_item["output"] = random.choice(prefixes) + new_item["output"]
            
            # 轻微同义词替换
            if random.random() < 0.3:
                new_item["output"] = synonym_replacement(new_item["output"])
            
            augmented.append(new_item)
    
    return augmented

# 使用

## 3.5 DPO数据

### 📢 一句话人话

**DPO是一种新的训练方式，数据格式是"问题 + 好答案 + 坏答案"，让模型自己学会分辨。**

---

### 🎯 生活比喻

```
DPO = 对比学习

老师教学生：这道题有ABCD四个选项
- A和C是对的
- B和D是错的
- 选对的给奖励，选错的打手心

这样学生就学会分辨了！
```

---

### 💻 核心概念

#### DPO vs SFT 对比：

| | SFT（监督微调） | DPO（直接偏好优化） |
|---|----------------|---------------------|
| 数据格式 | 问题 + 正确答案 | 问题 + 好答案 + 坏答案 |
| 学习方式 | 模仿正确 | 对比择优 |
| 数据难度 | 需要高质量标准答案 | 只需相对偏好 |
| 效果 | 确定性高 | 风格更自然 |

```python
# DPO数据格式

    "prompt": "如何治疗高血压？",      # 用户问题
    "chosen": "治疗高血压需要：1. 低盐饮食...",  # 好答案
    "rejected": "吃点降压药就行了"       # 差答案
}
```

### 🛠️ DPO数据准备

```python
def prepare_dpo_data(sft_data):
    """将SFT数据转换为DPO格式"""
    
    dpo_data = []
    
    for item in sft_data:
        # 生成一个"差"的回答
        # 方法1：随机截取（简单粗暴）
        bad_answer_1 = item["output"][:len(item["output"])//2]
        
        # 方法2：故意省略关键信息
        bad_answer_2 = item["output"].split("。")[0] + "。"
        
        # 方法3：加一些废话
        bad_answer_3 = "这个嘛，" + item["output"] + "大概就是这样吧"
        
        # 生成DPO格式
        dpo_data.append({
            "prompt": item["instruction"] + "\n" + item["input"],
            "chosen": item["output"],
            "rejected": bad_answer_1  # 任选一个差的
        })
    
    return dpo_data
```

### ⚠️ 小白易懵点

**❓ DPO和SFT哪个更好？**

> 各有优势：
> - SFT：数据简单，效果稳定
> - DPO：数据难准备，但效果更自然

**建议**：先用SFT打基础，有余力再用DPO提升

---

### 💡 一句话总结

**DPO是"优中选优"的训练方式，数据准备麻烦，但能让模型更懂什么才是"好答案"。**

def select_base_model(scenario):
    """

        "hardware": "消费级" or "专业级" or "云端",
        "budget": "有限" or "充足"
    }
    
    # 2. 匹配推荐
    recommendations = []

    if requirements["language"] == "中文为主":
        recommendations.extend([
            "Qwen2-7B-Instruct",     # 中文强，推荐

        recommendations.extend([

            "Mistral-7B-Instruct",   # 英文强
        ])
    
    # 双语场景
    else:
        recommendations.extend([
            "Qwen2-7B-Instruct",     # 双语优秀
            "Yi-34B",               # 双语优秀
            "DeepSeek-7B",          # 双语+代码

    elif requirements["hardware"] == "云端":
        recommendations = [m for m in recommendations if "70B" in m or "72B" in m]

from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import PeftModel
import torch

def load_medical_model():
    """加载医疗问答模型"""
    
    # 加载基座模型
    base_model = AutoModelForCausalLM.from_pretrained(
        "Qwen/Qwen2-7B-Instruct",
        device_map="auto",
        torch_dtype=torch.float16,
        load_in_4bit=True
    )
    
    # 加载微调后的模型
    model = PeftModel.from_pretrained(
        base_model,
        "./output/medical_model/checkpoint-XXX"
    )
    
    # 加载分词器
    tokenizer = AutoTokenizer.from_pretrained(
        "Qwen/Qwen2-7B-Instruct",
        trust_remote_code=True
    )
    
    return model, tokenizer

def ask_medical(model, tokenizer, question):
    """提问"""
    
    prompt = f"""### 指令:
你是一位专业医生，请根据患者的描述给出建议。回答时要注意：1) 给出初步分析 2) 建议检查 3) 提供一般性建议 4) 强调及时就医的重要性。

### 输入:
{question}

### 回答:
"""

    outputs = model.generate(
        **inputs,
        max_new_tokens=1024,
        temperature=0.7,
        top_p=0.9,
        do_sample=True,
        repetition_penalty=1.1
    )
    
    response = tokenizer.decode(outputs[0], skip_special_tokens=True)
    response = response.split("### 回答:")[-1].strip()
    
    return response

# 自动评估示例
from transformers import AutoModelForCausalLM, AutoTokenizer
import json

def evaluate_model(model_path, test_data_path):
    """评估模型"""
    
    # 加载模型
    model = AutoModelForCausalLM.from_pretrained(model_path)
    tokenizer = AutoTokenizer.from_pretrained(model_path)
    
    # 加载测试数据
    with open(test_data_path) as f:
        test_data = json.load(f)
    
    correct = 0
    total = len(test_data)
    
    for item in test_data:
        # 生成答案
        prompt = item["instruction"] + "\n" + item["input"]
        inputs = tokenizer(prompt, return_tensors="pt")
        outputs = model.generate(**inputs)
        generated = tokenizer.decode(outputs[0], skip_special_tokens=True)
        
        # 判断正确（简化版）
        if item["expected"] in generated:
            correct += 1
    
    accuracy = correct / total
    return {"accuracy": accuracy, "correct": correct, "total": total}

# 使用
result = evaluate_model("./output/model", "./data/test.json")
print(f"准确率: {result['accuracy']:.2%}")
```
