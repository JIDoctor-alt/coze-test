## 第二篇：K8s架构

> 🎬 场景：野原公寓楼下，园长在给孩子们介绍公寓管理

### 第一话：为什么需要容器编排？

**园长**：小朋友们，你们知道一个人做饭和连锁餐厅的区别吗？

**小新**：一个人做饭会手忙脚乱！

**园长**：对！如果客人多了，一个人做不过来。但如果是连锁餐厅，总部可以调配人手支援。

**小新**：就像K8s一样！

**园长**：没错！Kubernetes就是容器编排的"司令部"，负责：
- 调度：哪个容器放到哪台机器
- 扩缩容：负载高了自动加机器，低了自动减
- 自愈：某个容器挂了自动重启
- 服务发现：多个容器怎么互相找到

**小新**：哇，好厉害！

---

### 第二话：K8s整体架构 - 野原家公寓

**园长**：K8s采用Master-Node架构，就像我们公寓的管理系统。

**小新**：Master是什么？

**园长**：Master就是公寓总部（控制平面），有四大组件：
- **API Server**（前台接待）：所有操作都要经过它
- **etcd**（档案室）：保存所有数据
- **Scheduler**（调度员）：决定每个任务分配给谁
- **Controller Manager**（监工）：确保任务完成

**小新**：那Node呢？

**园长**：Node就是各个楼层（工作节点），负责运行容器：
- **kubelet**（楼层管理员）：执行任务
- **kube-proxy**（网络管理员）：管理网络

**插画说明**：K8s Master-Node架构比喻
![K8s架构](./蜡笔小新插画海报版/K8s/05_containers_0.jpg)

**小新**：一个房间可以住好几个人？

**美伢**：对！一个Pod里可以运行一个或多个容器，它们共享网络和存储。

**小新**：就像小新和风间一起住在胶囊旅馆，可以共用房间里的东西！

### 第二话：Deployment - 工厂流水线

**广志**（带小新参观工厂）：小新，你看这就是工厂流水线！

**小新**：哇，好多工人在工作！

**广志**：Deployment就是流水线主管，负责：
- 招聘工人（创建Pod）
- 解雇不合格的工人（删除Pod）
- 培训新技能（更新版本）
- 调整工人数量（扩缩容）

**小新**：那滚动更新呢？

**广志**：滚动更新就是逐步替换工人！比如从v1版本升级到v2，先换1个工人试试，没问题再继续换。

**小新**：就像我们学校换新校服一样！

**插画说明**：Deployment就像工厂流水线

### 第三话：Service - 电话总机

**小新**：妈妈，我要找爸爸！

**美伢**：打前台总机就行！

**小新**：总机是什么？

**美伢**：Service就像公寓前台总机！
- 提供稳定的电话号码（IP地址）
- 自动转接到具体的房间（Pod）
- 如果某个房间的人不在，会转给其他人（负载均衡）

**小新**：哦！就像我们学校的分机一样！

**美伢**：没错！Service有四种类型：
- **ClusterIP**：内部分机（集群内部访问）
- **NodePort**：门店窗口（开发测试）
- **LoadBalancer**：外部热线（生产环境）
- **ExternalName**：呼叫转接（外部服务别名）

**插画说明**：Service就像电话总机
![Service](./蜡笔小新插画海报版/K8s/09_service_reception_0.jpg)

### 第四话：Ingress - 大楼大堂

**小新**：妈妈，大楼大堂是干什么的？

**美伢**：大堂是统一入口！根据访客要找的人，分配到不同楼层。

**园长**：在K8s里，Ingress就像大楼大堂！
- 根据域名分配：api.example.com去API服务
- 根据路径分配：/shinchan去小新相关服务
- 处理TLS证书（门口安检）

**小新**：所以Ingress是HTTP/HTTPS的统一入口？

**园长**：对！Service是四层（TCP/UDP），Ingress是七层（HTTP/HTTPS）。

**插画说明**：Ingress就像大楼大堂

### 第四话：StatefulSet - 银行柜员

**广志**：小新，你去银行办过业务吗？

**小新**：去过！有柜员1号、2号、3号！

**广志**：StatefulSet就像银行柜员！
- 每个柜员有固定工号（稳定网络标识）
- 每个柜员有自己的抽屉（独立存储）
- 银行重开，柜员还是原来的人（有序部署/删除）

**小新**：跟Deployment不一样吗？

**广志**：Deployment的Pod是随机的，StatefulSet的Pod有稳定的名字和存储！

**小新**：所以StatefulSet适合数据库这种需要稳定身份的应用？

**广志**：Bingo！

**插画说明**：StatefulSet就像银行柜员

## 总结：K8s核心概念速查表

| 概念 | 比喻 | 一句话解释 |
|------|------|------------|
| **容器** | 集装箱 | 统一规格的包装 |
| **Dockerfile** | 泡面说明书 | 构建镜像的指令 |
| **Docker Compose** | 餐厅套餐 | 管理多个容器 |
| **K8s架构** | 公寓管理 | Master调度，Node运行 |
| **Pod** | 胶囊房间 | 最小调度单位 |
| **Deployment** | 工厂流水线 | 管理无状态应用 |
| **StatefulSet** | 银行柜员 | 管理有状态应用 |
| **DaemonSet** | 物业保安 | 每节点一个Pod |
| **Service** | 电话总机 | Pod的统一入口 |
| **Ingress** | 大楼大堂 | HTTP/HTTPS路由 |
| **ConfigMap** | 公告栏 | 非敏感配置 |
| **Secret** | 保险柜 | 敏感信息 |
| **PV/PVC** | 停车场 | 持久存储 |
| **Job** | 外卖订单 | 一次性任务 |
| **CronJob** | 定时闹钟 | 定时任务 |
| **Namespace** | 楼层隔离 | 资源分组 |
| **NetworkPolicy** | 门禁系统 | 网络防火墙 |
| **CoreDNS** | 电话簿 | 服务名解析 |
