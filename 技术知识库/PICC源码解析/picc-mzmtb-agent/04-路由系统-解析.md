# 路由系统解析 - router/index.js

> 🎯 一句话说明：路由系统就是"电梯按钮"，告诉程序用户想去哪个页面

---

## 这是啥？（小白版）

想象 **路由是医院的"楼层导航"**：
- 预约挂号台 = 根路由 `/`
- 每个科室 = 子路由 `/diseaseDeclare`（申报）、`/audit`（审核）等
- 护士站 = 路由守卫，检查你有没有挂对科室的号

---

## 核心代码解析

### 1. 静态路由配置

```javascript
export const staticRoutes = [
    {
        path: "/",
        name: "_index",
        redirect: "/loginMb"  // 默认跳转到登录页
    },
    {
        path: "/home",
        name: "home",
        meta: {
            title: "首页",
            pathName: "/home"
        },
        component: () => import("@/pages/Home"),  // 首页组件
        children: [...childrenRoutes]  // 包含所有子路由
    },
    {
        path: "/loginMb",
        name: "loginMb",
        meta: {
            title: "门诊慢特病业务管理信息系统",
            pathName: "/loginMb"
        },
        component: () => import("@/components/loginMb/index"),  // 登录页组件
    }
]
```

### 2. 路由守卫（权限控制）

```javascript
router.beforeEach((to, from, next) => {
    // 1. 设置页面标题
    if (to.meta.title) {
        document.title = to.meta.title;
    }
    
    // 2. 获取用户登录信息
    let logInfoMB = util.getUserInfo();
    
    // 3. 判断是否已登录
    if (!logInfoMB) {
        // 未登录：只能访问登录页和重置密码页
        if (to.path == '/loginMb') {
            next();
            sessionStorage.clear();
        } else if (to.path == '/resetConfirm') {
            next();
        } else {
            // 其他页面 → 强制跳转登录页
            next('/loginMb');
            sessionStorage.clear();
        }
    } else {
        // 已登录：允许访问
        next();
    }
});
```

### 3. 登录页面跳转逻辑

```javascript
// 模拟登录流程
用户输入账号密码
    ↓
调用登录接口 api.loginMb.login()
    ↓
登录成功 → 保存用户信息到 sessionStorage
    ↓
跳转到首页 this.$router.push('/home')
```

### 4. 404 路由处理

```javascript
// 原型替换 push 方法，捕获路由跳转错误
const originalPush = Router.prototype.push;
Router.prototype.push = function push(location) {
    return originalPush.call(this, location).catch(err => err);
};
```

## 路由守卫流程图

```
用户访问任意页面
    ↓
路由守卫拦截 (beforeEach)
    ↓
┌─────────────────────────────────────┐
│ 检查 sessionStorage 中是否有用户信息 │
└─────────────────────────────────────┘
    ↓
┌───────────┴───────────┐
↓                       ↓
有用户信息              无用户信息
    ↓                       ↓
检查目标页面是否          ↓
是登录/重置密码页    ┌────┴────────┐
    ↓               ↓            ↓
┌────┴────┐       登录页     其他页面
↓         ↓          ↓            ↓
是        否         ↓      强制跳转登录页
↓         ↓          ↓            ↓
允许通过   ↓      允许通过      清理session
    ↓    ↓          ↓
    ↓   强制跳转登录页
    ↓       ↓
    └───────┴──→ 执行业务逻辑，显示页面
```

## 知识点

### 🔹 路由懒加载
```javascript
component: () => import("@/pages/Home")
```
- 访问到该路由时才加载组件
- 减少首屏加载时间
- 类似于"按需加载"

### 🔹 hash 模式和 history 模式
```javascript
mode: "hash"  // 使用 #/xxx 格式
```
- hash 模式：URL 带 `#`，兼容性好，不需要服务器配置
- history 模式：URL 像正常路径，但需要服务器配置

### 🔹 meta 元信息
```javascript
meta: {
    title: "慢病申报查询",  // 页面标题
    ptitle: "宝鸡慢病管理"  // 父级标题
}
```
- 存储页面的附加信息
- 可用于权限控制、面包屑导航等
