# PICC门诊慢特病业务管理信息系统 - 源码结构全景图

> 🎯 一句话说明：这是一个用于管理门诊慢性病和特殊疾病患者申报、审核、体检、专家分配等全流程业务的后台管理系统。

---

## 🏠 系统整体架构（小白版）

想象一下，这是一个**大型医院挂号大厅的智能管理系统**：

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户界面层（Vue组件）                      │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│  │ 登录页  │ │ 申报管理 │ │ 初审管理 │ │ 专家分配 │ │ 体检管理 │   │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘ └─────────┘   │
├─────────────────────────────────────────────────────────────────┤
│                        路由层（Vue Router）                       │
│              负责"导航" - 用户点击菜单跳转到哪个页面               │
├─────────────────────────────────────────────────────────────────┤
│                        状态管理层（Vuex）                        │
│              负责"记忆" - 保存用户登录信息、菜单权限等             │
├─────────────────────────────────────────────────────────────────┤
│                        API接口层                                 │
│         负责"通信" - 页面需要数据时，向后端服务器发送请求          │
├─────────────────────────────────────────────────────────────────┤
│                        后端服务器                                │
│                    真实的数据存储和处理                           │
└─────────────────────────────────────────────────────────────────┘
```

## 🔄 业务流程图（小白版）

### 慢病患者从申报到拿卡的全流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   1.申报    │ -> │   2.初审    │ -> │   3.体检    │ -> │   4.专家分配│
│             │    │             │    │   (可选)    │    │             │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
      ↓                  ↓                  ↓                  ↓
  填写患者信息      审核资料完整性     分配体检机构       分配鉴定专家
  上传证明材料      决定是否需要体检   查看体检报告       专家给出结论
  
      ↓                  ↓                  ↓                  ↓
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   5.复审    │ -> │   6.备案    │ -> │   7.发卡    │ -> │   8.使用    │
│             │    │             │    │             │    │             │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
      ↓                  ↓                  ↓
  复核专家结论      系统登记备案      制作慢病卡
  决定是否通过      记录有效期       患者领取
```

## 🔐 权限系统（小白版）

系统有三级管理：

```
┌────────────────────────────────────────┐
│           系统管理员                     │
│    能管理所有模块、用户、角色、机构       │
├────────────────────────────────────────┤
│           机构管理员                     │
│    能管理本机构下的用户和业务             │
├────────────────────────────────────────┤
│           业务操作员                      │
│    只能操作自己有权限的菜单（如初审）     │
└────────────────────────────────────────┘
```

## 🔗 关键链接规则

- 登录页: `/#/loginMb`
- 首页: `/#/home`
- 申报查询: `/#/diseaseDeclare`
- 初审管理: `/#/diseaseFirstTrial`
- 专家分配: `/#/proficientManage`
- 体检分配: `/#/diseaseCheck`

> 链接格式：`域名/#/路由路径?token=xxx&userId=xxx&flag=0`
> - flag=0: 宝鸡
> - flag=1: 阜新
> - flag=2: 商洛
