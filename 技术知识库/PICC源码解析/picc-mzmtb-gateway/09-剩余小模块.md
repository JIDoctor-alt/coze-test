## 🔧 1. base - 基础类模块 (1个文件)

### BaseApi.java

> 🎯 API基类，所有API的父类

### 这是啥？（小白版）
像"通用工具箱"，封装了所有API都需要用的通用方法。

```java
public class BaseApi {
    
    @Autowired
    private RestfulForwardUtil restfulForwardUtil;
    
    /**
     * 预处理：验证系统编码和密码
     * 就像进门的门禁卡检查
     */
    protected boolean preHandle(HttpServletRequest request, HttpServletResponse response) {
        String syscode = request.getHeader(ReadSoapHeader.SYSCODE);
        String password = request.getHeader(ReadSoapHeader.PASSWORD);
        
        XMLTable xmlTable = new XMLTable<>();
        XMLCom xmlCom = new XMLCom();
        xmlCom.setSyscode(syscode);
        xmlTable.setXmlCom(xmlCom);
        
        // 验证接口授权
        JSONObject res = restfulForwardUtil.postByJson(
            "/rest/sys/interfaceGrant", 
            JSONObject.toJSONString(xmlTable), 
            request);
        
        return Optional.ofNullable(res)
            .map(row -> res.getInteger("status"))
            .map(row -> row == 0)
            .orElse(false);
    }
    
    /**
     * fastjson转jsonstr时null值转为空串
     * 就像把空盒子填满，避免null报错
     */
    protected static ValueFilter getValueFilter() {
        if (filter == null) {
            filter = new ValueFilter() {
                @Override
                public Object process(Object obj, String s, Object v) {
                    if (v == null) return "";  // null变空串
                    return v;
                }
            };
        }
        return filter;
    }
    
    /**
     * 把ApiResponse转成XML格式返回
     * 就像把JSON装进XML信封
     */
    protected String toJSONXML(ApiResponse apiResponse) {
        if (apiResponse.getStatus() == 0) {
            return JSONObject.toJSONString(apiResponse.getData());
        } else {
            // 错误情况返回XML格式
            JSONObject xmltable = new JSONObject();
            JSONObject xmlcom = new JSONObject();
            xmlcom.put("RESULTFLAG", "100100");
            xmlcom.put("RESULTMSG", apiResponse.getStatusText());
            xmltable.put("XMLTable", xmlcom);
            return xmltable.toJSONString();
        }
    }
}
```

### 继承关系
```
BaseApi (基类)
    ↓ 继承
各API类 (MbDeclareApi, LoginApi, ...)
```

## 🗃️ 2. cache - 缓存模块 (1个文件)

### CustomFontCache.java

> 🎯 字体缓存初始化

### 这是啥？（小白版）
像"字体库预加载"，启动时把字体文件加载到内存，避免每次用时再读文件。

```java
@Service
@Slf4j
public class CustomFontCache {
    
    /**
     * 初始化字体文件到JVM
     */
    public void initFont() {
        final GraphicsEnvironment ge = GraphicsEnvironment.getLocalGraphicsEnvironment();
        try {
            // 查找字体文件
            PathMatchingResourcePatternResolver resourcePatternResolver = 
                new PathMatchingResourcePatternResolver();
            Resource[] resources = resourcePatternResolver
                .getResources("classpath*:/**/config/MicrosoftYaHeiUI-Bold.ttf");
            
            if (resources.length == 0) {
                throw new Exception("字体文件无法获取!");
            }
            
            // 注册字体到JVM
            Font font = Font.createFont(Font.TRUETYPE_FONT, resources[0].getInputStream());
            ge.registerFont(font);
            
        } catch (Exception e) {
            log.error("字体文件获取失败：" + e.getMessage(), e);
        }
    }
}
```

### 应用场景
- 生成图片验证码
- PDF水印
- Excel导出

## 📞 3. call - 服务调用模块 (1个文件)

### CallApi.java

> 🎯 通用服务调用接口

### 这是啥？（小白版）
像"万能遥控器"，通过servicecode调用不同的服务。

```java
@RestController
@RequestMapping(value = "/call")
public class CallApi {
    
    @Resource(name = "baseCallServiceImpl")
    private BaseCallService baseCallService;
    
    /**
     * 通用服务调用
     * 根据servicecode路由到不同的服务
     */
    @RequestMapping(method = {RequestMethod.POST}, value = "/service")
    public ApiResponse<LinkRuturnEntity> service(
            @RequestBody String requestJson, 
            HttpServletRequest request) {
        
        String servicecode = request.getHeader("servicecode");
        JSONObject param = JSONObject.parseObject(requestJson);
        
        // 根据servicecode调用不同的服务
        LinkRuturnEntity linkRuturnEntity = baseCallService.callService(servicecode, param);
        
        return ApiResponse.ok(linkRuturnEntity);
    }
}
```

### 调用示例
```
POST /call/service
Header: servicecode=MB_QUERY
Body: {"idCard": "110101199001011234"}
```

### servicecode类型
| servicecode | 说明 |
|-------------|------|
| MB_QUERY | 慢病查询 |
| MB_DECLARE | 慢病申报 |
| MB_CANCEL | 慢病取消 |
| ACCOUNT_QUERY | 账户查询 |

## 📚 知识点

### 1. 基类设计模式
```
BaseApi (提供通用方法)
    ↓ 继承
具体API (实现具体业务)

好处：减少代码重复，统一处理逻辑
```

### 2. servicecode路由
```
CallApi.service()
    ↓ servicecode
BaseCallService.callService()
    ↓
根据servicecode分发到不同服务
```

### 3. 字体缓存优化
```
启动时加载字体 → 内存缓存 → 使用时直接取
    ↓
避免每次从文件读取，提高性能
```

## 🎯 小白理解要点

1. **BaseApi是工具箱** - 所有API都继承它
2. **字体缓存是预加载** - 启动时加载，启动后快
3. **CallApi是万能遥控** - 一个接口调多个服务
4. **servicecode是频道** - 选哪个频道看哪个台
