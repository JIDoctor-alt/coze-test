# picchealth/module/mb - 慢病管理核心模块

## 🎯 模块一句话说明
**慢病申报与审核业务的核心模块** - 523个文件，包含登录、申报、审核、专家分配、体检、药店等全部业务流程

## 🔑 核心API接口

### 1. LoginApi.java

> 🎯 登录认证接口

### 这是啥？（小白版）
像"酒店前台登记"，验证用户身份，发房卡（Token）。

### 核心接口

| 接口 | 路径 | 说明 |
|------|------|------|
| 登录 | POST /Login/doLogin | 用户名密码登录 |
| 获取用户 | POST /Login/getUser | 根据Token获取用户信息 |
| 登出 | POST /Login/logOut | 退出登录 |
| 修改用户 | POST /Login/update | 修改用户信息 |
| 用户列表 | POST /Login/getUserList | 分页查询用户 |
| 验证码 | POST /Login/register | 获取注册验证码 |
| 拼图验证 | POST /Login/check | 滑块验证码登录 |

### 关键代码
```java
@ApiOperation(value = "登录", notes = "登录")
@RequestMapping(method = {RequestMethod.POST}, value = "/doLogin")
ApiResponse login(@RequestBody LoginVo loginVo, HttpServletRequest request) {
    // 转发到后端服务验证
    return httpForwardUtil.post(baseUrl + "/doLogin", request, loginVo, parameterizedTypeReference);
}
```

### 2. MbDeclareApi.java

> 🎯 慢病申报核心接口

### 这是啥？（小白版）
像"医院挂号窗口"，慢病患者来这里提交申报材料。

### 核心接口

| 接口 | 路径 | 说明 |
|------|------|------|
| 查询代码表 | POST /MbDeclare/getCodeName | 获取疾病类型等字典 |
| 查询申报记录 | POST /MbDeclare/query | 分页查询申报列表 |
| 查询申报材料 | POST /MbDeclare/queryMeans | 查看附件材料 |
| 提交申报 | POST /MbDeclare/declare | 新增慢病申报 |
| 删除申报 | POST /MbDeclare/declareDelete | 删除申报记录 |
| 修改申报 | POST /MbDeclare/declareUpdate | 修改申报信息 |
| 资格校验 | POST /MbDeclare/VIPQueryChronicBalance | 医保资格验证 |

### 业务流程
```
提交申报 → 系统校验资格 → 分配专家 → 初审 → 复审 → 体检 → 享受待遇
```

## 🎯 小白理解要点

1. **网关不执行业务** - 只是转发请求到后端
2. **Token验证是必须的** - 几乎所有接口都需要登录
3. **flag代表地区** - 用于区分不同地市的请求
4. **VO是参数/返回值** - 前后端数据交换的格式
5. **枚举是常量池** - 避免魔法数字
