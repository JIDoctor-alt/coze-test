## 🚀 网关核心功能

### 1️⃣ 统一入口
```
所有外部请求 → 网关 → Token验证 → 拦截器链 → 业务处理 → 响应
```

### 2️⃣ 安全防护
- **APIAuthorityFilter**: 小程序Token验证、经办端Token验证
- **XssRequestFilter**: XSS脚本过滤，防注入攻击
- **CORS跨域**: 允许前端跨域访问

### 3️⃣ 请求转发
```java
HttpForwardUtil.post(url, request, requestVo, parameterizedTypeReference)
// 把请求转发到后端服务，同时传递header(包含Token)
```

### 4️⃣ 数据加密
- **SM2**: 国密算法，身份证号等敏感数据加密
- **AES**: 对称加密，通用数据加密

## 🔄 典型业务流程

### 慢病申报流程（小白版）
```
1. 参保人登录小程序 → 获取Token
2. 提交申报材料 → 网关验证Token
3. 药店审核 → 专家分配 → 初审/复审
4. 医保局审核 → 通过/拒绝
5. 享受待遇 → 药店购药 → 理赔结算
```

### 接口调用示例
```java
// 小程序登录
POST /Login/doLogin
→ 网关验证 → 转发到后端 → 返回Token

// 提交申报
POST /MbDeclare/declare
→ Token验证 → SM2解密 → 转发后端 → 保存数据

// 查询列表
POST /MbDeclare/query
→ Token验证 → 分页查询 → 返回结果
```

## 📝 知识点汇总

### 拦截器 vs 过滤器
- **Filter**: JavaEE标准，先于Spring执行，可过滤所有请求
- **Interceptor**: Spring MVC组件，可访问Controller方法参数

### RestTemplate vs Feign
- **RestTemplate**: 手动HTTP调用，需要拼接URL
- **Feign**: 声明式调用（本项目未用）

### SOAP vs REST
- **SOAP**: XML格式的WebService，老系统常用
- **REST**: JSON格式，更轻量

## ⚠️ 注意事项
1. 网关不执行业务逻辑，只负责转发和过滤
2. 实际业务在`picchealth-api-paas`服务中
3. 定时任务由XXL-JOB调度，不在本项目中
4. 小程序登录使用SM2加密，需配合前端使用
