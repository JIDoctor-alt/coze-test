# 04 - 系统管理模块

> 🎯 一句话说明：用户、机构、系统、权限归属的完整管理系统，是系统的"人口管理"

---

## 模块概述

这是系统最核心的模块，包含**四大管理功能**：
- 👤 **用户管理** - 管理系统中的用户账号
- 🏢 **机构管理** - 管理组织架构（公司/部门）
- 💻 **系统管理** - 管理接入的业务系统
- 🎫 **权限归属管理** - 管理各地区医保的分区权限

**生活比喻**：就像一个**大型商场管理系统**
- 用户 → 商场的员工
- 机构 → 商场各部门（服装部、食品部）
- 系统 → 商场的各种系统（收银系统、库存系统）
- 权限归属 → 各地区的营业执照/经营许可

```java
@Transactional(rollbackForClassName = {"Exception"})
public String create(PrivilegeUserInfoDto privilegeUserInfoDto) {
    
    // 1. 参数校验（必填项）
    if (StringUtils.isAnyBlank(
            privilegeUserInfo.getAccount(),      // 账号不能为空
            privilegeUserInfo.getName(),          // 姓名不能为空
            privilegeUserInfo.getTel(),           // 电话不能为空
            privilegeUserInfo.getOrgId())) {      // 机构不能为空
        throw new CustomException("参数错误！人员的账号、姓名、电话、机构信息均不能为空！");
    }
    
    // 2. 重复校验（账号、电话、邮箱唯一性）
    if (privilegeUserInfoDao.queryUserInfo(...).size() > 0) {
        throw new CustomException("该手机号/邮箱已被使用");
    }
    if (privilegeUserInfoDao.selectByAccount(...).size() > 0) {
        throw new CustomException("该账号已被使用");
    }
    
    // 3. 敏感词校验（不能创建admin/user/system等敏感账号）
    List<SensitiveWords> sensitiveWordsList = sensitiveWordsDao.selectWords();
    for (SensitiveWords words : sensitiveWordsList) {
        if (userAccount.contains(words.getWord())) {
            throw new CustomException("账号包含敏感词");
        }
    }
    
    // 4. 机构校验（机构必须存在）
    PrivilegeOrgInfo privilegeOrgInfo = privilegeOrgInfoDao.selectByPrimaryKey(
        privilegeUserInfo.getOrgId());
    if (privilegeOrgInfo == null) {
        throw new CustomException("人员对应的机构信息不存在！");
    }
    
    // 5. 生成用户ID，设置默认密码（SM4加密）
    String userId = UUIDUtil.getUUID();
    privilegeUserInfo.setId(userId);
    String password = SM4Util.sm4Encrypt(defaultPassWord); // 默认密码：PICChealth@2020
    privilegeUserInfo.setPassword(password);
    
    // 6. 保存用户基本信息
    privilegeUserInfoDao.insertSelective(privilegeUserInfo);
    
    // 7. 配置用户角色
    PrivilegeUserRoleInfoDto privilegeUserRoleInfoDto = new PrivilegeUserRoleInfoDto();
    privilegeUserRoleInfoDto.setUserId(userId);
    privilegeUserRoleInfoDto.setRoleIds(privilegeUserInfoDto.getRoleIds());
    setRoles(privilegeUserRoleInfoDto);
    
    // 8. 配置权限归属（PersonDivision分区）
    if (privilegeUserInfoDto.getAuthIds() != null && 
        privilegeUserInfoDto.getAuthIds().length > 0) {
        this.setAuths(privilegeUserInfoDto, userId);
    }
    
    return userId;
}
```

##### 3.3 分配用户角色
```java
@Transactional(rollbackForClassName = {"Exception"})
public void setRoles(PrivilegeUserRoleInfoDto dto) {
    String userId = dto.getUserId();
    String[] roleIds = dto.getRoleIds();
    
    // 1. 查询用户当前角色
    PrivilegeUserRoleInfo query = new PrivilegeUserRoleInfo();
    query.setUserId(userId);
    List<PrivilegeUserRoleInfo> userRoles = 
        privilegeUserRoleInfoDao.select(query);
    
    // 2. 删除已存在但不在新列表中的角色
    for (PrivilegeUserRoleInfo userRole : userRoles) {
        if (!Arrays.asList(roleIds).contains(userRole.getRoleId())) {
            privilegeUserRoleInfoDao.deleteByPrimaryKey(userRole.getId());
        }
    }
    
    // 3. 添加新分配的角色
    for (String roleId : roleIds) {
        // 检查是否已存在
        boolean exists = userRoles.stream()
            .anyMatch(ur -> ur.getRoleId().equals(roleId));
        
        if (!exists) {
            PrivilegeUserRoleInfo userRole = new PrivilegeUserRoleInfo();
            userRole.setId(UUIDUtil.getUUID());
            userRole.setUserId(userId);
            userRole.setRoleId(roleId);
            userRole.setCreator(UserUtils.getUser().getUserAccount());
            userRole.setCreatetime(LocalDateTime.now());
            privilegeUserRoleInfoDao.insertSelective(userRole);
        }
    }
}
```

##### 7.2 删除机构（带校验）
```java
@Transactional(rollbackForClassName = {"Exception"})
public Integer deleteAtById(String id) {
    // 1. 查询机构
    PrivilegeOrgInfo privilegeOrgInfo = privilegeOrgInfoDao.selectByPrimaryKey(id);
    if (privilegeOrgInfo == null) {
        throw new CustomException("该机构不存在");
    }
    
    // 2. 检查是否已删除
    if (privilegeOrgInfo.getDeleteAt() != null) {
        throw new CustomException("该机构已经被删除");
    }
    
    // 3. 检查是否有子机构（如果有则不允许删除）
    List<PrivilegeOrgInfo> privilegeOrgInfoList = 
        privilegeOrgInfoDao.selectOrgInfoByParentId(id);
    if (CollectionUtils.isNotEmpty(privilegeOrgInfoList)) {
        throw new CustomException("当前机构存在子机构，不允许删除");
    }
    
    // 4. 软删除
    privilegeOrgInfo.setDeleteAt(LocalDateTime.now());
    privilegeOrgInfo.setModifier(UserUtils.getUser().getUserAccount());
    return privilegeOrgInfoDao.updateByPrimaryKey(privilegeOrgInfo);
}
```

```
│     - 唯一性校验（账号、电话、邮箱）                        │
│     - 敏感词校验                                          │
│     - 机构存在性校验                                       │
│                ↓                                         │
│  3. 生成用户信息                                          │
│     - UUID生成userId                                     │
│     - SM4加密默认密码                                     │
│     - 设置创建人、创建时间                                 │
│                ↓                                         │
│  4. 保存用户 (privilege_user_info)                        │
│                ↓                                         │
│  5. 配置用户角色 (privilege_user_role_info)                │
│     - 关联userId和roleId                                  │
│                ↓                                         │
│  6. 配置权限归属 (privilege_user_auth)                     │
│     - 关联userId和authId                                  │
│                ↓                                         │
│  7. 配置分区权限 (person_division)                         │
│     - 设置flag(地市标识)                                  │
│     - 设置各权限标识(ismanual, isdownload等)              │
│                ↓                                         │
│  8. 返回userId                                            │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 关键知识点

### 1. 敏感词过滤
```java
// 不能创建包含以下关键词的账号
敏感词列表: admin, user, system, manager, supervisor, root, postgres
```
> 防止恶意账号冒充系统管理员

### 2. 默认密码策略
- 默认密码: `PICChealth@2020`
- 使用SM4加密存储
- 首次登录后应强制修改

### 3. 软删除保护
- 用户删除 = 设置deleteAt
- 机构删除 = 检查是否有子机构/用户
- 角色删除 = 保留关联数据

### 4. 权限归属的五要素
| 要素 | 说明 | 示例 |
|------|------|------|
| flag | 地市标识 | 0-宝鸡, 4-延安, 7-榆林 |
| unitcode | 服务窗口编码 | 医保编码 |
| persontype | 人员身份 | 职工, 居民 |
| isupdate | 修改权限 | 1-支公司, 2-分公司 |
| isdownload | 下载权限 | 1-允许 |
