# 02 - 角色管理模块

> 🎯 一句话说明：管理系统中的"岗位"角色，分配菜单权限，控制"谁能看到什么"

---

## 模块概述

这是一个**权限控制核心模块**，类似于公司的**岗位管理制度**：
- 什么是"岗位"？→ 角色（如：管理员、普通用户、审核员）
- 岗位能做什么？→ 绑定菜单权限
- 谁是这个岗位？→ 分配给用户

**生活比喻**：就像酒店的**门禁卡系统**
- 客房部经理的卡 → 能开所有客房
- 前台的卡 → 只能开前台区域
- 保安的卡 → 只能开门禁和监控室

#### 核心代码解析

##### 3.1 创建角色
```java
@Transactional(rollbackForClassName = {"Exception"})  // 事务控制，出错回滚
public CommonReqVo createRole(RoleCreateVo roleCreateVo) {
    // 1. 创建角色基本信息
    PrivilegeRoleInfo privilegeRoleInfo = new PrivilegeRoleInfo();
    BeanUtils.copyProperties(roleCreateVo, privilegeRoleInfo);
    
    // 2. 生成唯一ID
    String roleId = UniqueIDGenerator.generateUniqueID();
    privilegeRoleInfo.setId(roleId);
    privilegeRoleInfo.setCreatetime(LocalDateTime.now());
    privilegeRoleInfo.setCreator(UserUtils.getUser().getUserAccount());
    
    // 3. 数据校验
    String checkRes = roleDataVerifi(privilegeRoleInfo);
    if (StringUtils.isNotBlank(checkRes)) {
        return error(checkRes);
    }
    
    // 4. 插入数据库
    privilegeRoleInfoDao.insertSelective(privilegeRoleInfo);
    
    // 5. 配置角色资源（菜单）
    if (CollectionUtils.isNotEmpty(roleCreateVo.getMenuIds())) {
        for (String menuId : roleCreateVo.getMenuIds()) {
            PrivilegeRoleResource resource = new PrivilegeRoleResource();
            resource.setRoleId(roleId);
            resource.setMenuId(menuId);
            resource.setSystemId(roleCreateVo.getSystemId());
            privilegeRoleResourceDao.insert(resource);
        }
    }
    
    return success(new RoleCreateDto().setId(roleId));
}
```

##### 3.2 更新角色
```java
public CommonReqVo updateRole(RoleUpdateVo roleUpdateVo) {
    // 1. 查询原角色信息
    PrivilegeRoleInfo roleQueryPo = privilegeRoleInfoDao.selectByPrimaryKey(roleUpdateVo.getId());
    
    // 2. 如果修改了关键字段，重新校验
    if (fieldDiff(roleUpdateVo.getName(), roleQueryPo.getName()) || 
        fieldDiff(roleUpdateVo.getOrgId(), roleQueryPo.getOrgId())) {
        // 校验通过后更新
    }
    
    // 3. 软删除旧的资源绑定
    List<PrivilegeRoleResource> oldRoleResouces = 
        privilegeRoleResourceDao.select(privilegeRoleResource);
    for (PrivilegeRoleResource old : oldRoleResouces) {
        old.setDeleteAt(LocalDateTime.now());  // 软删除
        privilegeRoleResourceDao.updateByPrimaryKey(old);
    }
    
    // 4. 添加新的资源绑定
    for (String menuId : roleUpdateVo.getMenuIds()) {
        PrivilegeRoleResource resource = new PrivilegeRoleResource();
        resource.setMenuId(menuId);
        resource.setId(UniqueIDGenerator.generateUniqueID());
        privilegeRoleResourceDao.insert(resource);
    }
    
    return success();
}
```

##### 3.3 删除角色（软删除）
```java
public CommonReqVo deleteRole(RoleDeleteVo roleDeleteVo) {
    // 1. 查询角色
    PrivilegeRoleInfo privilegeRoleInfo = 
        privilegeRoleInfoDao.selectByPrimaryKey(roleDeleteVo.getId());
    
    // 2. 校验是否已删除
    if (privilegeRoleInfo.getDeleteAt() != null) {
        return error("该ID已删除,请不要重复删除");
    }
    
    // 3. 软删除：设置删除时间
    privilegeRoleInfo.setDeleteAt(LocalDateTime.now());
    privilegeRoleInfo.setModifier(UserUtils.getUser().getUserAccount());
    privilegeRoleInfoDao.updateByPrimaryKey(privilegeRoleInfo);
    
    return success();
}
```

## 核心业务逻辑

### 1. 角色创建流程
```
用户提交创建角色请求
       ↓
校验必填字段（name, orgId, systemId）
       ↓
生成唯一ID（UUID）
       ↓
保存角色基本信息
       ↓
配置角色菜单资源
       ↓
返回角色ID
```

### 2. 角色更新流程
```
用户提交更新角色请求
       ↓
查询原角色信息
       ↓
比较是否有字段变化
       ↓
如果变化：校验新值 → 更新角色表
       ↓
软删除旧的资源绑定（设置deleteAt）
       ↓
添加新的资源绑定
       ↓
返回结果
```

### 3. 角色删除流程（软删除）
```
用户提交删除角色请求
       ↓
查询角色是否存在
       ↓
检查是否已删除
       ↓
设置deleteAt为当前时间
       ↓
返回成功
（不删除关联的资源记录）
```

## 关键知识点

### 1. 软删除 vs 硬删除
| 类型 | 特点 | 适用场景 |
|------|------|----------|
| **软删除** | 设置deleteAt标记，数据保留 | 业务数据、角色、机构 |
| **硬删除** | 直接从数据库删除 | 临时数据、日志 |

> 本模块采用**软删除**，保留数据便于追溯和恢复

### 2. 事务控制 @Transactional
```java
@Transactional(rollbackForClassName = {"Exception"})
```
- 如果方法执行出错，**自动回滚**，不会污染数据库
- 适用于"创建角色 + 配置资源"这种组合操作

### 3. 数据校验
- 角色名称不能为空
- 机构ID必须存在
- 菜单ID必须有效
- 不能重复创建同名角色

## 常见问题

### Q1: 为什么角色要关联机构？
**A**: 实现**机构隔离**，不同机构的角色互不干扰

### Q2: 一个角色可以绑定多个系统的菜单吗？
**A**: 可以！通过`PrivilegeRoleResource`表的`systemId`区分

### Q3: 删除角色会删除用户的角色绑定吗？
**A**: 不会！用户角色绑定记录会保留，但权限已经失效
