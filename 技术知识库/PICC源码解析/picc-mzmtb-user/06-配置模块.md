# 06 - 配置模块

> 🎯 一句话说明：配置API拦截器、Token校验、过滤器等基础设施

---

## 模块概述

这是系统的**安全防护模块**，类似于公司的**门禁系统**：
- 进门刷卡 → Token校验
- 权限验证 → 这个人能进这个门吗？
- 异常处理 → 不刷卡的人不能进

**生活比喻**：就像机场的**安检通道**
- 扫描机票 → 验证Token
- 检查证件 → 验证身份
- X光检查 → 权限验证

## 文件解析

### 1. APIAuthorityFilter.java - API权限过滤器 ⭐核心

> 🎯 拦截所有用户接口请求，验证Token和接口权限

#### 这是啥？（小白版）
就像公司的**门禁系统**，每个员工进门都要刷卡：
- 刷卡验证 → Token是否有效
- 权限检查 → 这个人能进这个区域吗？
- 放行/拦截 → 通过或拒绝访问

#### 核心代码解析

```java
@WebFilter(urlPatterns = {"/privilege/user/*"}, filterName = "APIAuthorityFilter")
public class APIAuthorityFilter implements Filter {
    
    @Resource
    private RedisUtil redisUtil;
    
    private String UrlPermission = "UrlPermission:";
    
    @Override
    public void doFilter(ServletRequest servletRequest, ServletResponse servletResponse, 
                          FilterChain filterChain) throws ServletException, IOException {
        
        HttpServletRequest request = (HttpServletRequest) servletRequest;
        String requestPath = request.getServletPath();  // 请求路径
        
        // 1. 获取Token（POST从Header，GET从参数）
        String token = "";
        if ("POST".equalsIgnoreCase(request.getMethod())) {
            token = request.getHeader(RequestConstant.HEAD_TOKEN);
        } else if ("GET".equalsIgnoreCase(request.getMethod())) {
            token = request.getParameter(RequestConstant.HEAD_TOKEN);
        }
        
        // 2. Token有效性校验
        if (StringUtils.isBlank(token)) {
            throw CustomException.createByMassage(999, "登录Token无效!请重新登录");
        }
        
        // 3. 从Redis获取用户信息
        User re = (User) redisUtil.get(RedisKeyConf.API_TOKEN + token);
        if (re == null || StringUtils.isBlank(re.getUserId())) {
            throw new CustomException("用户信息错误,请重新登录");
        }
        
        String userid = re.getUserId();
        
        // 4. 检查URL权限
        if (!redisUtil.exists(UrlPermission + userid)) {
            synchronized (this) {
                if (!redisUtil.exists(UrlPermission + userid)) {
                    throw new BusinessException("请重新登录");
                }
            }
        }
        
        // 5. 验证当前请求路径是否有权限
        List<String> list = (ArrayList<String>) redisUtil.get(UrlPermission + userid);
        long count = list.stream().filter(s -> s.contains(requestPath)).count();
        if (count == 0) {
            throw new CustomException("接口权限未通过!");
        }
        
        // 6. 放行
        filterChain.doFilter(servletRequest, servletResponse);
    }
}
```

#### 工作流程
```
┌─────────────────────────────────────────────────────────┐
│              API权限过滤流程                               │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  用户发起请求                                             │
│  POST /privilege/user/create                            │
│       ↓                                                 │
│  1. 获取Token                                            │
│     - POST请求：从Header获取                             │
│     - GET请求：从参数获取                                 │
│       ↓                                                 │
│  2. Token校验                                            │
│     - Redis查询：API_TOKEN:token → User对象              │
│     - 失败：返回"Token无效"                               │
│       ↓                                                 │
│  3. 获取用户权限列表                                      │
│     - Redis查询：UrlPermission:userId → [url列表]        │
│     - 失败：返回"请重新登录"                               │
│       ↓                                                 │
│  4. 验证请求路径权限                                      │
│     - 检查url列表是否包含当前请求路径                      │
│     - 不包含：返回"接口权限未通过"                          │
│       ↓                                                 │
│  5. 放行请求                                             │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 2. ApiInterceptor.java - API拦截器注解

> 🎯 用于标记Controller需要被拦截器处理

#### 这是啥？（小白版）
就像给门贴上**"需要刷卡"**的标签，告诉门禁系统："这扇门需要验证后才能进"。

#### 核心代码解析

```java
@Target({ ElementType.METHOD, ElementType.TYPE })
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface ApiInterceptor {
    String url() default "";
}
```

#### 使用方式
```java
@ApiInterceptor
@RestController
@RequestMapping("/privilege/user")
public class UserInfoApi {
    // 该Controller下的所有接口都会经过拦截器
}
```

## 安全机制解析

### 1. Token验证机制
```
用户登录 → 生成Token → 存入Redis → 返回给前端
         ↓
前端请求 → 带Token → 后端验证 → 有效则放行
```

### 2. Redis存储结构
```
API_TOKEN:{token}  → User对象
UrlPermission:{userId} → [url1, url2, url3...]
```

### 3. 权限验证流程
```java
// 1. 用户登录时，生成Token并存入Redis
redisUtil.set("API_TOKEN:abc123", user, 过期时间);

// 2. 同时存储用户的URL权限列表
redisUtil.set("UrlPermission:user001", 
    ["/privilege/user/create", "/privilege/user/query"], 
    过期时间);

// 3. 请求时验证
User user = redisUtil.get("API_TOKEN:" + token);
List<String> permissions = redisUtil.get("UrlPermission:" + user.getUserId());
boolean hasPermission = permissions.contains(requestPath);
```

## 关键知识点

### 1. Filter vs Interceptor
| 类型 | 作用范围 | 执行顺序 |
|------|----------|----------|
| **Filter** | Servlet层，所有请求 | 先执行 |
| **Interceptor** | Controller层 | 后执行 |

### 2. @WebFilter注解
- `urlPatterns`：拦截的URL路径
- `filterName`：过滤器名称
- `initParams`：初始化参数

### 3. Token安全性
- Token存储在Redis中，有过期时间
- 请求时验证Token是否存在
- Token失效后需要重新登录

### 4. 权限列表
- 存储在Redis中
- 格式：`[请求路径1, 请求路径2, ...]`
- 验证时使用`contains`判断

## 常见问题

### Q1: 为什么用Redis存储Token？
**A**: 
- 高性能，支持高并发
- 支持过期时间自动失效
- 支持分布式部署

### Q2: 如何实现单点登录？
**A**: 
- 登录时生成Token存入Redis
- 其他系统通过验证Token实现单点登录

### Q3: 权限列表为什么用URL路径匹配？
**A**: 
- 简单高效
- 支持前缀匹配（如`/privilege/user/*`）
