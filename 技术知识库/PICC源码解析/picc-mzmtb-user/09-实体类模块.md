# 09 - 实体类模块 (PO)

> 🎯 一句话说明：数据库表对应的Java实体类，是数据操作的"档案卡片"

---

## 模块概述

这是系统的**数据模型层**，类似于图书馆的**图书卡片目录**：
- 每张卡片 → 对应一本书 → 数据库中的一行数据
- 卡片上的字段 → 书的属性 → 数据库中的列

**生活比喻**：就像公司的**员工档案卡**
- 姓名、工号、部门 → 数据库字段
- 照片、指纹 → 特殊字段
- 离职日期 → 删除标记

### 1. PrivilegeUserInfo.java - 用户实体

#### 核心代码

```java
@Table(name = "privilege_user_info")
public class PrivilegeUserInfo extends BaseEntity {

    private String password;   // 密码
    private Integer sex;        // 性别
    private String tel;         // 电话
    private String email;       // 邮箱
    private String idNum;       // 身份证
    private String orgId;       // 机构ID
    private Integer isAdmin;    // 是否管理员
```

#### 字段说明
| 字段 | 类型 | 说明 | 加密 |
|------|------|------|------|
| account | String | 登录账号 | 否 |
| name | String | 姓名 | ✅ @SensitiveEncrypt |
| password | String | 密码 | ✅ SM4加密 |

| email | String | 邮箱 | ✅ @SensitiveEncrypt |

## 注解说明

| 注解 | 说明 |
|------|------|
| `@Table(name="xxx")` | 指定对应的数据库表 |
| `@Column(name="xxx")` | 指定对应的数据库列 |
| `@Data` (Lombok) | 自动生成getter/setter |
| `@NoArgsConstructor` | 无参构造函数 |
| `@AllArgsConstructor` | 全参构造函数 |

## 常见问题

### Q1: 为什么要继承BaseEntity？
**A**: 统一管理公共字段（id, creator, createtime等）

### Q2: 实体类和数据库表有什么关系？
**A**: 一对一关系，每个实体类对应一张表

### Q3: 为什么有些字段有@Column注解，有些没有？
**A**: 
- 有@Column：字段名和列名不一致
- 无@Column：字段名和列名一致（MyBatis自动映射）
