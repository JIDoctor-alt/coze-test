# 05 - 分区权限模块

> 🎯 一句话说明：管理各地区医保系统的分区权限配置，控制"能看到哪些地区的数据"

---

## 模块概述

这是系统的**地域权限控制模块**，类似于公司的**区域销售权限**：
- 北京员工 → 只能看北京的数据
- 上海员工 → 只能看上海的数据
- 区域经理 → 可以看多个区域的数据

**生活比喻**：就像医院的**科室分诊系统**
- 儿科护士 → 只能进儿科
- 骨科医生 → 只能进骨科
- 急诊科 → 所有科室都能进（特殊权限）

## 核心业务逻辑

### 1. 配置分区权限流程
```java
// 在UserInfoServiceImpl.create()中调用
setAuths(PrivilegeUserInfoDto dto, String userId) {
    
    // 1. 删除历史分区权限
    deleteHistroyUserAuth(account, userId);
    
    // 2. 查询用户选择的权限归属
    List<PrivilegeMenuAuthDto> authList = 
        privilegeAuthInfoDao.queryAuthListByRoleIdAndAuthId(
            roleIds, authIds);
    
    // 3. 遍历每个权限归属，配置分区
    for (auth : authList) {
        PersonDivision pd = new PersonDivision();
        pd.setAccount(account);
        pd.setFlag(auth.getFlag().toString());
        pd.setReady1(account + "&" + auth.getFlag());
        
        // 4. 根据auth类型设置不同权限标识
        switch(auth.getCode()) {
            case "YL_SDFP":    // 榆林手动分配
                pd.setIsmanual("1");
                break;
            case "SH_SBXIXGSQ": // 上海申报修改(审核)
                pd.setIsupdate("1");
                pd.setIdentity("2"); // 分公司
                break;
            // ... 更多配置
        }
        
        // 5. 保存分区权限
        personDivisionDao.insertSelective(pd);
    }
}
```

### 2. 查询用户分区权限
```java
public List<PersonDivision> queryUserDivisions(String account) {
    PersonDivision pd = new PersonDivision();
    pd.setAccount(account);
    pd.setDr((short) 0); // 只查有效记录
    return personDivisionDao.queryPersonDivisonList(pd);
}
```

## 关键知识点

### 1. 为什么需要分区权限？
- 不同地区的医保政策不同
- 需要控制用户只能访问本地区数据
- 满足数据安全和合规要求

### 2. flag字段的作用
- 标识用户属于哪个地市
- 用于数据隔离
- 结合unitcode实现更细粒度控制

### 3. 多权限组合
一个用户可以同时拥有多个地区的权限：
```java
// 示例：某员工可以访问宝鸡和延安
[
    {account: "zhangsan", flag: "0"},  // 宝鸡
    {account: "zhangsan", flag: "4"}   // 延安
]
```

## 常见问题

### Q1: 一个用户可以配置多个地区的权限吗？
**A**: 可以！通过多条PersonDivision记录实现

### Q2: 备案撤销权限和医保账号权限有什么区别？
**A**: 
- isfilingback：是否有权限撤销已备案的信息
- isyb：是否属于医保局账号

### Q3: 如何控制用户只能看部分地区的数据？
**A**: 通过flag字段过滤，SQL查询时加上`WHERE flag IN (用户授权的flag列表)`
