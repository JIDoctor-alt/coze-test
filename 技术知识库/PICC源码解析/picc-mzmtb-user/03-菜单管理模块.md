# 03 - 菜单管理模块

> 🎯 一句话说明：管理系统的菜单结构，让用户知道"能看到哪些功能"

---

## 模块概述

这是一个**导航系统模块**，类似于手机的**APP菜单**：
- 打开APP → 看到一排图标（菜单）
- 点击图标 → 进入对应功能页面

**生活比喻**：就像餐厅的**菜单本**
- 封面 → 系统名称
- 章节分类 → 一级菜单
- 具体菜品 → 二级菜单/按钮

### 3. MenuInfoServiceImpl.java - 菜单服务的"实际执行者"

#### 这是啥？（小白版）

#### 核心代码解析

##### 3.1 创建菜单
```java
@Transactional
public ApiResponse createMenu(PrivilegeMenuInfoVo vo) {
    // 1. 重复校验
    if (StringUtils.isNotBlank(vo.getUrl())) {
        // 检查URL是否重复
        List<PrivilegeMenuInfo> list = privilegeMenuInfoService.queryInfo(info);
        if (CollUtil.isNotEmpty(list)) {
            return ApiResponse.fail("模块链接重复，请修改后重新创建");
        }
    }
    
    // 2. 检查菜单名称是否重复
    PrivilegeMenuInfo menuInfo = new PrivilegeMenuInfo();
    menuInfo.setName(vo.getName());
    List<PrivilegeMenuInfo> privilegeMenuInfoList = privilegeMenuInfoService.queryInfo(menuInfo);
    if (CollUtil.isNotEmpty(privilegeMenuInfoList)) {
        return ApiResponse.fail("模块重复，请修改后重新创建");
    }
    
    // 3. 保存菜单基本信息
    privilegeMenuInfo.setCreator(UserUtils.getUser().getUserAccount());
    privilegeMenuInfo.setCreatetime(LocalDateTime.now());
    privilegeMenuInfoService.save(privilegeMenuInfo);
    
    // 4. 关联到机构系统
    PrivilegeOrgSystem privilegeOrgSystem = new PrivilegeOrgSystem();
    privilegeOrgSystem.setOrgId(orgId);
    privilegeOrgSystem.setSystemId(vo.getSystemId());
    privilegeOrgSystem.setMenuId(privilegeMenuInfo.getId());
    privilegeOrgSystemService.save(privilegeOrgSystem);
    
    // 5. 修改排序
    if (StringUtils.isNotBlank(vo.getParentId())) {
        privilegeMenuInfoService.modifyOrder(vo, privilegeMenuInfo);
    }
    
    return ApiResponse.ok(privilegeMenuInfo.getId());
}
```

##### 3.2 删除菜单（带校验）
```java
@Transactional
public ApiResponse deleteMenu(PrivilegeMenuInfoVo vo) {
    List<String> ids = vo.getIds();  // 支持批量删除
    
    for (String menuId : ids) {
        // 1. 查询菜单
        PrivilegeMenuInfo privilegeMenuInfoOne = 
            privilegeMenuInfoService.selectOne(privilegeMenuInfo);
        
        // 2. 检查是否已删除
        if (privilegeMenuInfoOne.getDeleteAt() != null) {
            return ApiResponse.fail("该菜单已经被删除");
        }
        
        // 3. 检查是否有子菜单
        List<PrivilegeMenuInfoDto> menus = 
            privilegeMenuInfoService.getChildrenMenus(dto);
        if (CollUtil.isNotEmpty(menus)) {
            return ApiResponse.fail("当前模块下有子模块信息，不允许删除");
        }
        
        // 4. 软删除
        privilegeMenuInfoOne.setDeleteAt(LocalDateTime.now());
        privilegeMenuInfoOne.setModifier(account);
        privilegeMenuInfoService.save(privilegeMenuInfoOne);
        
        // 5. 重新排序后续菜单
        reorderChildren(childrenMenuList);
    }
    
    return ApiResponse.ok();
}
```

##### 3.3 查询菜单树
```java
public ApiResponse getMenuTree(PrivilegeMenuInfoVo vo) {
    // 1. 获取当前用户机构ID
    String orgId = UserUtils.getUser().getOrgId();
    
    // 2. 查询该机构+系统下的所有菜单
```

## 菜单树形结构

### 树形结构示意图
```
🏢 系统（System）
│
├── 📁 用户管理 (一级菜单)
│   ├── 📄 用户列表 (二级菜单)
│   │   ├── 🔘 新增用户 (按钮)
│   │   ├── 🔘 编辑用户 (按钮)
│   │   └── 🔘 删除用户 (按钮)
│   └── 📄 角色管理 (二级菜单)
│       ├── 🔘 新增角色 (按钮)
│       └── 🔘 分配权限 (按钮)
│
├── 📁 订单管理 (一级菜单)
│   ├── 📄 订单列表
│   └── 📄 订单统计
│
└── 📁 系统设置 (一级菜单)
    ├── 📄 机构管理
    └── 📄 系统配置
```

### 数据库存储 vs 内存树形
```sql
-- 数据库存储（扁平化）
| id | parent_id | name     | level |
|----|-----------|----------|-------|
| 1   | 0         | 用户管理  | 1     |
| 2   | 1         | 用户列表  | 2     |
| 3   | 2         | 新增用户  | 3     |
| 4   | 2         | 编辑用户  | 3     |

-- 程序处理后（树形化）
{
    "id": "1",
    "name": "用户管理",
    "children": [
        {
            "id": "2",
            "name": "用户列表",
            "children": [
                {"id": "3", "name": "新增用户"},
                {"id": "4", "name": "编辑用户"}
            ]
        }
    ]
}
```

## 核心业务逻辑

### 1. 创建菜单流程
```
传入菜单信息（名称、URL、所属系统、上级菜单）
       ↓
校验：URL不能重复、名称不能重复
       ↓
生成唯一ID
       ↓
保存菜单信息
       ↓
建立菜单与机构系统的关联
       ↓
调整同级菜单的排序
       ↓
返回菜单ID
```

### 2. 删除菜单流程
```
传入菜单ID
       ↓
检查菜单是否存在
       ↓
检查是否已删除
       ↓
检查是否有子菜单（如果有则不允许删除）
       ↓
软删除菜单（设置deleteAt）
       ↓
调整同级菜单的排序（填补空缺）
       ↓
返回结果
```

### 3. 构建菜单树流程
```
查询某系统下的所有菜单
       ↓
过滤已删除的菜单
       ↓
按parentId分组
       ↓
递归构建树形结构
       ↓
返回树形JSON
```

## 关键知识点

### 1. 树形结构的两种实现方式
| 方式 | 优点 | 缺点 |
|------|------|------|
| **递归查询** | 逻辑简单 | 数据库查询次数多 |
| **一次查询+内存处理** | 查询快 | 内存占用大 |

> 本模块使用**一次查询+内存处理**方式

### 2. 排序号的作用
```java
// 同一级菜单，按orderIndex升序排列
// 1, 2, 3, 4...
// 删除中间某个菜单后，需要重新排序
```

### 3. 软删除的重要性
- 保留历史数据
- 便于数据恢复
- 避免误删导致系统异常

## 常见问题

### Q1: 为什么删除菜单要检查子菜单？
**A**: 防止误删导致用户看不到功能入口；如果要删，必须先删子菜单

### Q2: 菜单的URL是干嘛用的？
**A**: 用户点击菜单后，跳转到的页面地址

### Q3: 什么是菜单服务？
**A**: 菜单除了跳转页面，还可能关联一些"操作"，比如导出、打印
