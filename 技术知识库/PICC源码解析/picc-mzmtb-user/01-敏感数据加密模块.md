# 01 - 敏感数据加密模块

> 🎯 一句话说明：对用户敏感信息进行AES/SM4加密脱敏，保护个人隐私数据

## 模块概述

这是一个**数据安全防护模块**，类似于给用户的"隐私文件"加上"保险箱"。当系统需要存储或传输用户的敏感信息（如姓名、电话、身份证等）时，会自动进行加密处理，确保数据即使泄露也无法被直接使用。

## 文件解析

### 1. AesUtil.java - AES加密工具箱

> 🎯 **AES对称加密的核心工具类**

#### 这是啥？（小白版）
就像一把**万能钥匙**，可以：
- 把明文信息"锁"进保险箱（加密）
- 把"锁"着的信息取出来（解密）

**生活比喻**：就像医院的处方单，普通人看不懂，只有医生能看懂（解密）。

#### 核心代码解析

```java
// 1. 密钥配置（前后端约定的密码本）
@Value("${utils.aes.KEY:abcdefgabcdefg12}")
private static String KEY;  // 默认密钥，需要在配置文件中修改

// 2. 加密算法配置
@Value("${utils.aes.ALGORITHMSTR:AES/ECB/PKCS5Padding}")
private static String ALGORITHMSTR;
```

##### 主要方法

| 方法名 | 大白话解释 |
|--------|------------|
| `aesEncrypt(content)` | 把"你好"变成"乱码" |
| `aesDecrypt(encrypt)` | 把"乱码"还原成"你好" |
| `base64Encode(bytes)` | 把二进制转成字符串格式（方便传输） |
| `base64Decode(str)` | 把字符串格式转回二进制 |
| `isEncrypted(data)` | 判断这条数据是不是加密过的 |

##### 加密流程（简化版）
```
明文 → AES加密 → Base64编码 → 密文字符串
```

##### 解密流程（简化版）
```
密文字符串 → Base64解码 → AES解密 → 明文
```

#### 知识点
- **AES**：高级加密标准，美国联邦政府采用的加密算法
- **ECB模式**：电子密码本模式，最简单但安全性一般
- **PKCS5Padding**：填充方式，确保数据块对齐
- **Base64**：把二进制转成64个可打印字符，便于JSON传输

### 2. SensitiveEncrypt.java - 敏感数据标签

> 🎯 **给字段打上"敏感"标签的注解**

#### 这是啥？（小白版）
就像给文件贴上**"机密"标签**，告诉系统："这个字段需要特殊保护！"

**生活比喻**：就像快递包裹上的"易碎品"贴纸，物流人员看到就知道要小心处理。

#### 核心代码解析

```java
@Target(ElementType.FIELD)        // 作用在字段上
@Retention(RetentionPolicy.RUNTIME) // 运行时生效
@JacksonAnnotationsInside         // 继承Jackson注解能力
@JsonSerialize(using = SensitiveEncryptSerializer.class)  // 序列化时加密
@JsonDeserialize(using = SensitiveEncryptDeserializer.class) // 反序列化时解密
public @interface SensitiveEncrypt {
    boolean serialize() default true;   // 序列化时是否加密
    boolean deserialize() default true; // 反序列化时是否解密
}
```

#### 使用示例
```java
public class User {
    @SensitiveEncrypt  // 姓名需要加密
    private String name;
    
    @SensitiveEncrypt  // 电话需要加密
    private String tel;
    
    private String orgId;  // 机构ID不需要加密
}
```

#### 知识点
- **注解(Annotation)**：给代码添加元数据的标签
- **Jackson**：Java中处理JSON序列化的库
- **序列化**：对象→JSON字符串
- **反序列化**：JSON字符串→对象

### 3. SensitiveEncryptSerializer.java - 加密打包机

> 🎯 **把对象转成JSON时自动加密**

#### 这是啥？（小白版）
就像**自动打包机**，把用户的敏感信息打包成"加密包裹"再发出去。

**生活比喻**：就像银行的自动存取款机，你存钱时自动点钞、封装，你取钱时自动拆封、验钞。

#### 核心代码解析

```java
@Override
public void serialize(Object value, JsonGenerator gen, SerializerProvider serializers) {
    if (value == null) {
        gen.writeNull();  // 空值就写null
        return;
    }
    
    if (!encrypt.serialize()) {
        gen.writeObject(value);  // 不需要加密就直接写
        return;
    }
    
    // 核心：调用AesUtil加密
    String encryptedValue = AesUtil.aesEncrypt(stringValue);
    gen.writeString(encryptedValue);  // 写入加密后的值
}
```

##### 加密效果示例
```json
// 加密前
{"name": "张三", "tel": "13800138000"}

// 加密后
{"name": "k9x8y7z6w5v4u3==", "tel": "a1b2c3d4e5f6g7=="}
```

### 4. SensitiveEncryptDeserializer.java - 解密拆包机

> 🎯 **把JSON转成对象时自动解密**

#### 这是啥？（小白版）
就像**拆包裹的工具**，把加密的"包裹"拆开，还原成原始信息。

**生活比喻**：就像收快递时的扫码枪一扫，包裹信息自动录入系统。

#### 核心代码解析

```java
@Override
public Object deserialize(JsonParser p, DeserializationContext ctxt) {
    String value = p.getValueAsString();
    if (value == null) {
        return null;
    }
    
    if (!sensitiveEncrypt.deserialize()) {
        return value;  // 不需要解密就返回原值
    }
    
    try {
        return AesUtil.aesDecrypt(value);  // 核心：调用AesUtil解密
    } catch (Exception e) {
        // 如果解密失败（比如数据本来就没加密），返回原值
        return value;
    }
}
```

##### 容错处理
- 如果数据是**未加密的明文**，解密会失败，此时返回原值
- 避免因为"加密状态不一致"导致系统崩溃

## 关键知识点

### 1. 对称加密 vs 非对称加密
| 类型 | 特点 | 比喻 |
|------|------|------|
| **对称加密** | 加密解密用同一把钥匙 | 用同一把钥匙锁门和开门 |
| **非对称加密** | 公钥加密，私钥解密 | 锁公开，钥匙私有 |

> 本模块使用的是**对称加密(AES/SM4)**，速度快，适合大量数据

### 2. 为什么需要Base64编码？
- 加密后的数据是**二进制字节数组**
- JSON/字符串传输需要**可打印字符**
- Base64把二进制转成64种可打印字符的字符串

### 3. 为什么解密要容错？
- 老数据可能是**明文存储**的
- 网络传输可能有**格式损坏**
- 避免系统因个别数据异常而崩溃

---

## 安全建议

1. **密钥管理**：生产环境一定要改默认密钥
2. **定期轮换**：建议定期更换加密密钥
3. **传输安全**：HTTPS是基础，加密是增强
4. **字段选择**：不是所有字段都需要加密，评估业务需求
