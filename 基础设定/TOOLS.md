记录各种工具的使用技巧和注意事项
- **常用工具注意事项：**
  - **fetch_web工具注意事项：**
    1. **源码获取**：可直接获取GitHub上的源码文件，URL格式为https://github.com/<用户名>/<仓库名>/blob/main/<文件路径>
    2. **批量获取**：支持一次传入多个URL，适合同时获取多个相关源码文件
    3. **目标精准**：针对源码分析场景，直接定位到核心文件（如api/目录下的初始化文件、db/目录下的模型文件）可提高分析效率
    4. **项目主页获取**：直接传入GitHub项目主页URL（如https://github.com/zstar1003/ragflow-plus）可获取项目README和核心信息
    5. **官方文档抓取**：同时抓取项目主页和官方部署方案文档（如阿里云解决方案页面），可快速了解项目架构和部署实践
    6. **URL格式限制**：GitHub目录URL（如https://github.com/xxx/tree/main/xxx）可能返回"invalid link"错误，建议改用文件URL（/blob/main/）或官方文档URL
  - **search_web工具注意事项：**
    1. **关键词精准**：结合具体技术点和场景，比如分析RAGFlow权限时使用"RAGFlow tenant_id knowledgebase permission"这样的组合关键词
    2. **场景化搜索**：针对特定需求，如检索权限过滤机制，可使用"RAGFlow retrieval permission filter kb_id"这样的关键词；针对检索权限与rag目录分析，可使用"RAGFlow rag nlp search retrieval kb_id filter"、"RAGFlow knowledgebase_service accessible method"这样的关键词
    3. **多关键词组合**：一次传入多个相关查询，覆盖不同维度的技术点，提高信息获取的全面性
    4. **阿里云服务调研**：调研云服务时使用"云厂商+服务名+功能"的组合关键词，如"阿里云PAI EAS 向量检索服务"、"阿里云百炼平台 RAG知识库"、"阿里云RDS MySQL Redis 托管服务价格"，可快速获取服务特性、价格、对比信息
  - **edit_file工具注意事项：**
    1. **可用模式**：支持replace_one、replace_all、append、append_newline模式，不支持insert模式
    2. **追加内容**：当需要追加内容到文件末尾时，使用append_newline模式；若需在指定位置替换内容，使用replace_one模式并准确定位要替换的原始内容
  - **write_file工具注意事项：**
    1. **路径验证**：如果遇到"路径无效"错误，可能是因为目标路径不在当前工作目录下，可尝试使用bash命令先创建目录，再写入文件
    2. **替代方案**：当write_file工具无法写入指定路径时，可使用bash命令的重定向功能（如cat > 路径 << 'EOF' 内容 EOF）来创建文件
  - **read_file工具注意事项：**
    1. **路径限制**：read_file无法访问/tmp/等系统目录下的文件，会报"路径无效"错误
    2. **替代方案**：使用bash工具的`cat`命令读取（如`cat /tmp/xxx/file.java`），可正常访问任意路径
    3. **场景**：源码分析时项目常克隆到/tmp/，必须用bash读取而非read_file
- **特殊场景工具调用注意事项：**
  - **WSSMSIF短信服务注意事项：**
    1. 异步发送：短信发送使用异步方式，不阻塞主业务流程
    2. 异常处理：短信发送失败不影响主业务结果，仅记录日志
    3. 内容规范：短信内容需包含【人保健康】签名，符合运营商要求
    4. 频率限制：注意短信发送频率，避免被运营商限流
  - **exportYL接口调用注意事项：**
    1. 权限判断：先检查用户是否绑定医保账号，再判断是否有导出权限
    2. 敏感信息处理：根据权限决定是否对姓名、身份证号、手机号进行脱敏
    3. 异步考虑：Excel导出操作耗时较长，建议异步处理
  - **Dify平台使用注意事项：**
    1. 字符限制：集团大模型输入存在字符长度限制，需控制输入数据长度
    2. 智能体维护：以病种为单位建立智能体，需分别维护准入规则和Prompt
    3. 知识库建设：需业务及鉴定专家支持，规则库与案例库结合
  - **Drools规则引擎使用注意事项：**
    1. 规则抽离：将变化的标准从程序代码中抽离，用接近自然语言编写
    2. 动态配置：支持规则的动态修改和生效，无需重启服务
    3. 地市差异：需考虑不同地市的病种准入规则差异，设计可扩展的规则结构
  - **CXEncryptionUtis加密工具注意事项：**
    1. **字段长度**：加密后的字符串可能较长，超过数据库varchar(8000)的限制，建议将存储加密参数的数据库字段改为TEXT类型。
    2. **参数校验**：加密前确保输入参数完整，避免加密后发现参数缺失需要重新加密。
    3. **性能考虑**：加密操作可能耗时，对于批量数据建议异步处理。
  - **callService调用外部接口注意事项：**
    1. **参数校验**：调用外部接口前必须校验所有必填参数，避免因参数缺失导致外部接口返回错误。
    2. **异常处理**：捕获外部接口调用的异常，记录详细日志（包括请求参数和响应结果），避免异常扩散。
    3. **异步调用**：对于耗时较长的外部接口调用，建议使用异步方式（如CompletableFuture），避免阻塞主业务流程。
    4. **幂等设计**：如果外部接口涉及数据修改，确保接口调用的幂等性，避免重复调用导致数据重复。

- **LongCat API使用注意事项：**
  1. **配置一致性**：baseUrl与models需匹配，如使用LongCat的baseUrl（https://api.longcat.chat/openai），models应配置为LongCat系列模型（如LongCat-Flash-Lite、LongCat-Flash-Chat等），而非其他厂商模型。
  2. **API Key安全**：不要在公开场合分享包含API Key的配置，定期重新生成API Key避免泄露。
  3. **Anthropic兼容模式**：LongCat提供Anthropic兼容端点（https://api.longcat.chat/anthropic），可将Claude Code配置为使用LongCat平台，需将所有Anthropic模型映射到LongCat模型（如LongCat-Flash-Thinking-2601）。
  4. **自定义模型提供方**：使用`__provider__xxx`格式的id可动态获取该平台所有模型，简化配置，无需手动列出每个模型。
- **Java项目API文档生成技巧：**
  1. **脚本提取法**：使用Python脚本批量扫描Java文件，通过正则表达式提取@RequestMapping、@ApiOperation等注解信息，生成结构化API数据
  2. **解析优化**：针对Java方法的注解解析，优先匹配@ApiOperation再关联方法定义，提高接口信息的准确性
  3. **分类统计**：将API按业务域分类，统计每个域的类数和方法数，便于快速了解项目接口分布
  4. **文档生成**：基于提取的结构化数据，自动生成Markdown格式的API文档，包含接口路径、方法名、功能描述等关键信息
  5. **工具组合**：结合bash命令执行脚本、grep辅助验证、head查看结果等工具链，提高文档生成效率
  6. **路径解析优化**：解析@RequestMapping注解时，跳过包含"RequestMethod"的路径，避免把请求方法当成路径部分
  7. **多注解处理**：支持处理GetMapping、PostMapping、PutMapping、DeleteMapping、RequestMapping等多种HTTP注解，正确识别请求方法
  8. **编码兼容**：读取Java文件时先尝试utf-8编码，失败再使用gbk编码，适配不同文件的编码格式
  9. **敏感信息脱敏**：对文档中的手机号、身份证号等敏感信息进行脱敏处理（如手机号替换为154****7181，身份证号替换为130180********0918）
  10. **特殊场景处理**：处理类级别和方法级别的@RequestMapping组合，正确拼接完整接口路径
- **前端项目总纲索引文档生成技巧：**
  1. **核心结构模板**：固定包含8部分内容
     - 项目一句话定位（清晰说明项目用途和覆盖范围）
     - 文档导航表（列出所有子文档、行数、核心内容、推荐指数）
     - 分角色阅读路径（小白/前端开发/全栈工程师三类学习路线）
     - 核心发现摘要（架构亮点/业务特色/技术债务三个维度总结）
     - 技术栈速查（分模块展示技术栈，如核心框架/HTTP/UI/后端服务）
     - 关键文件索引（列出核心文件路径、用途、重要度）
     - 服务调用链路（用ASCII图展示完整请求流程）
     - 快速问答（整理5-10个常见问题与答案）
  2. **行数控制策略**：总长度控制在120-180行，各模块分配合理行数，用表格和代码块简化展示
  3. **内容整合方法**：从多个子文档中提取核心信息，避免重复，保持总纲的简洁性和导航性
  4. **Vue项目环境注意事项**：
     - 必须使用Node.js 14.x或16.x版本（Vue 2项目不支持17及以上版本）
     - 项目启动命令：`npm install` → `npm run serve`，默认访问http://localhost:80
  5. **文档关联技巧**：在总纲中用相对路径链接到所有子文档，便于用户快速跳转查看详细内容
