## 一、SM4密钥硬编码问题分析（最紧急）

### 🔍 问题原理分析

📖 **学习理解**：SM4是中国国密对称加密算法，密钥相当于"保险箱钥匙"。

**当前实现分析**：

```java
// SM4Util.java - 当前实现
public class SM4Util {
    // ❌ 密钥硬编码
    private static final String KEY = "1234567812345678";
}
```

**安全问题原理**：

1. **硬编码风险**：密钥随代码发布，任何人都能看到
2. **无法更换**：更换密钥需要重新编译部署
3. **所有环境相同**：测试、生产环境使用相同密钥
4. **合规问题**：违反金融行业密钥管理规范

**安全影响**：
- 攻击者获取代码后可以解密所有加密数据
- 大规模数据泄露风险

## 二、AES密钥硬编码问题分析

### 🔍 问题原理分析

📖 **学习理解**：AES是最常用的对称加密算法，密钥安全至关重要。

**当前实现分析**：

项目中的AES密钥可能也是硬编码的。

**安全问题原理**：

1. **对称加密特性**：知道密钥就能解密所有数据
2. **硬编码风险**：密钥随代码发布
3. **密钥管理缺失**：没有密钥轮换机制

---

## 三、RSA密钥硬编码问题分析

### 🔍 问题原理分析

📖 **学习理解**：RSA是非对称加密算法，私钥必须严格保密。

**安全问题原理**：

1. **私钥泄露后果严重**：私钥泄露后所有加密数据都可被解密
2. **数字签名伪造**：私钥泄露后攻击者可以伪造签名
3. **硬编码风险**：私钥随代码发布

---

## 四、URL权限模糊匹配问题分析

### 🔍 问题原理分析

📖 **学习理解**：URL权限应该精确匹配，而不是模糊匹配。

**当前实现分析**：

```java
// 当前实现使用 contains 模糊匹配
if (url.contains("/admin")) {
    // 权限校验
}
```

**安全问题原理**：

1. **模糊匹配风险**：`contains`匹配可能被绕过
2. **URL编码绕过**：使用URL编码可能绕过检查
3. **路径穿越绕过**：使用`../`可能绕过检查

**安全影响**：
- 攻击者可能绕过权限检查访问敏感接口

## 五、APIAuthorityFilter路径限制问题分析

### 🔍 问题原理分析

📖 **学习理解**：权限过滤器应该覆盖所有需要保护的接口。

**当前实现分析**：

```java
@WebFilter(urlPatterns = "/privilege/user/*")
public class APIAuthorityFilter implements Filter {
    // 只拦截 /privilege/user/* 路径
}
```

**安全问题原理**：

1. **路径覆盖不全**：很多敏感接口不在拦截范围内
2. **未授权访问**：攻击者可以直接访问未拦截的接口
3. **权限绕过**：通过改变URL路径可能绕过权限检查

## 六、权限编码硬编码问题分析

### 🔍 问题原理分析

📖 **学习理解**：权限编码"88"硬编码在代码中，难以维护。

**当前实现分析**：

```java
// 权限编码"88"硬编码
if (user.getPermission().equals("88")) {
    // 特殊权限逻辑
}
```

**安全问题原理**：

1. **魔法数字问题**：代码中直接使用数字，含义不明确
2. **难以维护**：权限规则变化需要修改代码
3. **配置缺失**：缺少权限配置管理机制

---

## 七、默认密码安全问题分析

### 🔍 问题原理分析

📖 **学习理解**：默认密码可预测是严重的安全隐患。

**当前实现分析**：

系统可能存在可预测的默认密码。

**安全问题原理**：

1. **默认密码风险**：攻击者可以猜测默认密码
2. **弱密码问题**：密码复杂度要求不足
3. **首次登录强制修改**：可能缺少强制修改密码机制

---

## 八、密码明文存储问题分析

### 🔍 问题原理分析

📖 **学习理解**：密码明文存储是严重的安全漏洞。

**当前实现分析**：

`passwordBackMD5`方法可能将密码以明文或弱加密方式存储。

**安全问题原理**：

1. **明文存储风险**：数据库泄露后密码直接暴露
2. **弱哈希问题**：MD5已经被证明不安全
3. **缺少盐值**：相同密码产生相同哈希，容易被彩虹表破解
