## 🔑 核心发现摘要

### 架构亮点
| 发现 | 说明 |
|------|------|
| **4服务协同** | 前端Vue(80/443) → 网关(9001) → 业务服务(9091) + 权限服务(9092) |
| **4 Axios实例** | axios.js(慢病)/axiosCenter.js(通用)/axiosPower.js(权限)/axiosjkgl.js(鉴权) |
| **Nginx反向代理** | 前端80/443端口统一入口，经9001网关路由分发 |

### 业务特色
| 发现 | 说明 |
|------|------|
| **21地市模块** | 按地市独立封装，延安(62文件)最大、通用模板(53文件) |
| **13个地市flag** | 运行时判断，动态渲染差异页面 |
| **340+后端接口** | 74个API文件封装，涵盖申报/审核/备案/处方等业务 |

### 技术债务
| 发现 | 说明 |
|------|------|
| **Vue 2.6遗产** | 尚未升级Vue 3，建议后续迁移 |
| **组件库分散** | mtbcomponents/mtbnewcomponents/mtbslcomponents三套并存 |

## 🔧 技术栈速查

```
核心框架
├── Vue 2.6           - 响应式前端框架
├── Vuex 3            - 状态管理（vuex-persistedstate持久化）
└── Vue Router 3      - 路由管理（含2389行子路由配置）

HTTP与网络
├── Axios             - HTTP请求（4实例模式）
├── Nginx             - 反向代理（端口80/443 → 9001）
└── AES加密           - util.js工具函数

UI组件
├── Ant Design Vue 1.x - UI组件库
├── mtbcomponents      - 基础款组件库
├── mtbnewcomponents   - 新版组件库
└── mtbslcomponents    - 商洛定制组件库

后端服务
├── 网关服务           - picc-mzmtb-gateway (9001)
├── 业务服务           - picc-mzmtb-auth (9091)
└── 权限服务           - picc-mzmtb-user (9092)
```

## 💡 快速问答

| 问题 | 答案 |
|------|------|
| Q: 如何启动项目？ | A: `npm install` → `npm run serve`，访问 http://localhost:80 |
| Q: 前后端端口是多少？ | A: 前端80/443，后端网关9001，业务9091，权限9092 |
| Q: 如何区分不同地市？ | A: 通过`src/utils/util.js`中的地市flag标识动态渲染 |
| Q: API请求走哪个实例？ | A: 慢病业务用axios.js，通用用axiosCenter.js，权限用axiosPower.js |
| Q: Node.js版本要求？ | A: **必须使用14.x或16.x**，Vue 2不支持17+ |
