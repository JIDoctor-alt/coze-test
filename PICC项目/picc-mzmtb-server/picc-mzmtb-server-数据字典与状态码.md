## 📚 阅读指南

### 🔰 小白概念解释

| 程序员术语 | 小白版解释 | 比喻 |
|-----------|-----------|------|
| **枚举（Enum）** | 菜单上的编号，每个编号对应一道菜 | 就像餐厅菜单：1=宫保鸡丁，2=鱼香肉丝 |
| **状态码** | 快递的物流状态，从发货到签收 | 像京东快递：已发货→运输中→派送中→已签收 |
| **常量** | 写在墙上的规矩，谁也不能改 | 就像交通规则：红灯停、绿灯行，永远不变 |
| **状态流转** | 快递从一个站点到下一个站点的路线 | 从北京站→石家庄站→郑州站→终点站 |
| **响应码** | 考试成绩，60分及格、100分满分 | 000000=考100分(成功)，100100=考0分(失败) |

## 第一章：响应码字典

### 1.1 系统响应码体系

> **什么是响应码？** 就是系统给用户的"考试成绩"。000000分=满分成功，其他分数=各种失败原因。

#### 核心响应码表

| 响应码 | 状态名 | 人话解释 | 触发场景 |
|--------|--------|----------|----------|
| `000000` | RES_SUCCESS | 处理成功 ✅ | 所有接口正常返回时的状态 |
| `000010` | RES_SUCCESS_DOING | 处理中 ⏳ | 异步操作正在执行，如批量处理 |
| `100100` | RES_FAIL | 错误异常 ❌ | 系统内部错误 |
| `100110` | RES_PARAM_ERROR_FAIL | 参数错误 ❌ | 传入的参数不符合要求 |
| `100120` | RES_AUTH_ERROR_FAIL | 安全验证失败 ❌ | MD5签名错误、接口权限不足 |
| `100160` | RES_DATA_STYLE_FAIL | 数据格式错误 ❌ | 无法解析的数据格式 |
| `100230` | RES_MONEY_ERROR_FAIL | 金额参数非法 ❌ | 传入负数金额、超出范围等 |
| `100300` | RES_QUERY_ERROR_FAIL | 未查询到数据 🔍 | 查询条件无结果 |
| `200100` | RES_CARDNO_FAIL | 会员卡认证失败 💳 | 卡号不存在或已注销 |
| `200120` | RES_CARD_PASSWORD_FAIL | 支付密码错误 💰 | 连续输错会被锁定 |
| `200121` | RES_CARD_NOPASSWORD_FAIL | 未设置支付密码 | 需要先设置密码才能支付 |
| `200122` | RES_CARD_PASSWORD_CH | 附属卡需要验证 | 附属卡消费需输入独立密码 |
| `200123` | RES_CARD_NO_TOKEN | 未授权免密支付 | 需要先开通免密功能 |
| `200130` | RES_CARD_LOCK_FAIL | 卡片已锁定 🔒 | 联系客服解锁 |
| `200140` | RES_CARD_NOAUTHLOCK_FAIL | 附属卡未授权 | 需要主卡授权 |
| `500120` | RES_REPEAT | 重复提交 ⚠️ | 订单已存在，请勿重复提交 |
| `500560` | RES_MONEY_OVER_BALANCE | 余额不足 💸 | 支付金额超出口袋钱 |
| `600100` | RES_EXPORT_OVER_MAXIMUM | 导出数据超限 | 导出条数超过系统限制 |

### 1.2 响应码记忆口诀

```
0开头 = 成功（像考试成绩90-100分）
1开头 = 客户端错误（参数、权限、数据格式问题）
2开头 = 会员卡问题（认证、密码、锁定等）
5开头 = 业务逻辑问题（余额、重复等）
6开头 = 系统限制（导出超限等）
```

### 2.1 申报状态（DclaStatusEnum）

> **什么是申报状态？** 就像你申请护照的过程：提交材料→审核中→通过/拒绝

| 状态码 | 状态名 | 人话解释 | 可流转到的下一个状态 |

| `0` | WBA | 未备案 ❌ | 提交备案申请 → 审核中 |
| `1` | BAWB | 备案完成 ✅ | 享受慢特病待遇 |
| `2` | BASB | 备案失败 ❌ | 修改材料后重新申请 |
| `3` | BASHZ | 备案审核中 ⏳ | 等待审核结果 |

| `5` | BACXSB | 备案撤销失败 ❌ | 联系医保局处理 |
