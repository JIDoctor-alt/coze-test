## 一、模块概览

### 1.1 先搞清楚这三个核心概念

| 概念 | 小白化理解 | 生活中的类比 |
|------|-----------|-------------|
| **处方** | 医生开的"点餐单"，上面写着吃什么药、吃多少、吃多久 | 餐厅的点餐小票 |
| **药店** | 只有通过医保认证的药房才能卖慢病药 | 外卖平台上的"合作商家认证" |
| **费用报销** | 先花钱买药→拿发票→审核→打款 | 公司报销流程 |

## 二、处方管理详解

### 2.1 处方管理是什么？

> 💡 **大白话**：处方就是医生给慢病患者开的"吃药清单"。处方管理就是管这张清单的：上传、审核、录入药品、驳回修改。

### 2.2 核心API文件一览

```
处方管理相关文件：
├── MbPrescriptionManagementApi.java      # 慢病处方管理（榆林/宝鸡/延安）
├── VipPrescriptionApi.java               # 慢病处方池（延安专用）
├── CxPrescriptionPoolApi.java            # 购药对接接口（处方池操作）
└── MbmzInfoUpdateApi.java                # 慢病人员修改（多地市处方操作）
```

### 2.3 各地市处方差异

| 地市 | 处方表 | 药品表 | 特色功能 |
|------|-------|--------|---------|
| **榆林** | TMbYLPrescription | VipDrugYL | 处方图片上传、录入完成按钮 |
| **宝鸡** | PrescriptionMain | VipDrugBJ | PC端上传处方、驳回功能 |
| **延安** | TMbYAPrescription | VipDrugYA | 处方池、用药目录、OCR识别 |
| **商洛** | TMbSLPrescription | VipDrugSL | 草稿保存、处方与药品录入 |

**方法4: UpdateButtonYL - 处方录入完成按钮(榆林)**

```java
// 一句话说明
// 点击"完成"按钮，标志处方录入工作结束

// 参数
TMbYLPrescriptionDto dto
├── accountid    // 账户ID
├── id          // 处方ID

ApiResponse (成功/失败)

⚠️ 相当于"全选确认"的逻辑
```

**方法5: rejectPrescriptionImgYL - 处方驳回(榆林)**

```java
// 一句话说明
// 审核不通过，打回让用户重新上传/修改处方

// 参数
RejectPrescriptionImgYlVo vo
├── id              // 处方ID（必填）
└── rejectreason    // 驳回原因（必填！）

// 返回值
CommonVo
├── code           // 0=成功
└── msg            // 成功/失败消息

// 易懵点
⚠️ 驳回原因必须填写！
⚠️ 驳回后处方不再参与后续流程
```

**方法2: rejectPreImgBJ - 处方单张驳回(宝鸡)**

```java
// 一句话说明
```

## 三、药店管理详解

### 3.1 药店管理是什么？

> 💡 **大白话**：不是所有药店都能卖慢病药！只有"定点药店"才行。药店管理就是管这些定点药店的：药品目录维护、订单管理、费用结算。

#### 4.3.2 药店订单服务 - 费用计算逻辑

---

**费用计算公式**

```java
// 场景：患者买药，费用如何计算？

总费用 = Σ(药品单价 × 购买数量)

报销金额 = min(总费用, 账户剩余额度)

个人自付 = 总费用 - 报销金额

账户扣除 = 报销金额  // 从慢病账户扣款
```

**示例**

```
场景：张三买药，总价500元，账户余额800元

计算：
- 总费用 = 500元
- 报销金额 = min(500, 800) = 500元
- 个人自付 = 500 - 500 = 0元  ← 全部报销！

账户变更：
- 原余额：800元
- 扣减后：800 - 500 = 300元
```

## 六、易踩坑点

### 6.1 处方管理易踩坑

| 坑点 | 说明 | 避雷方法 |
|------|------|---------|
| ⚠️ 处方状态混乱 | isnew字段含义各地市不一致 | 榆林：0=待录入，延安：0=通过，1=待审 |
| ⚠️ 图片URL拼接错误 | 返回的URL不完整 | 检查httpUrl配置是否正确 |
| ⚠️ 驳回后仍可消费 | 驳回处方没有同步到医保 | 驳回后要调用处方注销接口 |
| ⚠️ 重复录入药品 | 同一药品被多次保存 | 先查询再决定新增还是更新 |
| ⚠️ OCR识别失败 | 图片模糊导致识别不出 | 提供清晰的处方图片 |
| ⚠️ 跨月处方问题 | 跨月后处方无法使用 | 注意处方的有效期限制 |

### 6.2 药店管理易踩坑

| 坑点 | 说明 | 避雷方法 |
|------|------|---------|
| ⚠️ 余额查询不及时 | 医保账户余额有延迟 | 每次消费前重新查询余额 |
| ⚠️ 订单状态不一致 | WS接口调用失败但本地已保存 | 使用事务保证数据一致性 |
| ⚠️ 退款时间限制 | 当天数据当天退，跨月不能退 | 提前告知患者退款规则 |
| ⚠️ 药品价格冲突 | 医保限价与药店定价不一致 | 优先使用医保限价 |
| ⚠️ 定点医院未绑定 | 患者在非定点药店买药无法报销 | 买药前确认药店是否定点 |

### 6.3 费用管理易踩坑

| 坑点 | 说明 | 避雷方法 |
|------|------|---------|
| ⚠️ 余额计算错误 | 浮点数精度问题 | 使用BigDecimal计算金额 |
| ⚠️ 并发扣款超支 | 同一账户同时多笔消费 | 加锁或使用乐观锁 |
| ⚠️ 对账不平 | 报表数据与实际不符 | 每日进行账务核对 |
| ⚠️ Excel导出超时 | 大数据量导出慢 | 分页导出或异步处理 |

### 6.4 各地市差异汇总

| 地市 | 处方表 | 疾病字典 | 药品目录 | 特殊功能 |
