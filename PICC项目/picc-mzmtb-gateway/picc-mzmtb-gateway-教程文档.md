## 四、各文档核心发现摘要

### 📋 项目全貌与架构解析
- **定位**：前台服务=高级中介，不做饭只传菜
- **架构**：三服务分工明确，前台(9001)→业务(9091)+权限(9092)
- **技术栈**：Spring Boot + Apollo + Redis + BES服务器 + SM2国密

### 📋 API接口全景
- **11大业务域**：登录认证、申报管理、审批管理、发卡管理、处方管理、药店管理、专家管理、数据查询、定时任务、外部接口、系统运维
- **纯转发特性**：104个API类中大部分为纯转发，仅1个有自定义业务逻辑
- **地市标识**：BJ(宝鸡)/YA(延安)/SL(商洛)/YL(榆林)等13地市

### 📋 安全与代码质量审计
- **严重风险(4个)**：
  - SM2私钥硬编码
  - AES密钥硬编码
  - SFTP默认密码`hellolevy`明文
  - CORS允许任意来源
- **高危风险(6个)**：JWT密钥硬编码、Redis未授权、文件上传漏洞等

### 📋 Service与VO层解析
- **32个Service**：VipMbDeclareServiceImpl(PDF导出)、VipMbdeclareInfoServiceImpl(图片处理)、VipMbdeclareFileServiceImpl(FTP操作)
- **361个VO**：Request VO(点餐单)、Response VO(成品菜)
- **HttpForwardUtil**：核心工具，将请求转发到后台业务服务

### 📋 通信机制与三服务联动
- **调用链路**：前端→前台(9001)→权限服务(9092)→业务服务(9091)→数据库
- **Redis**：存储Token和Session，实现无状态认证
- **Apollo**：统一配置管理，切换环境只需改配置

### 📋 Onboarding手册
- **启动命令**：`apollo.config.sh` + `mvn spring-boot:run`
- **配置读取**：通过Apollo ID命名空间加载
- **开发规范**：遵循PICC内部编码规范
