## 🔴 SEC-GW-001：SM2私钥硬编码

### 📍 问题位置

```
文件路径：src/main/java/com/picchealth/utils/SM2Util.java
```

### 📖 学习理解

**问题原理**：SM2是中国国密非对称加密算法，私钥必须严格保密。

**学习要点**：
- 私钥相当于"家门钥匙"，必须妥善保管
- 硬编码的私钥会随代码发布，任何人都能看到
- 应该从安全的配置中心动态获取密钥
- 不同环境应该使用不同密钥

**安全影响**：
- 攻击者获取代码后可以解密所有加密数据
- 无法实现密钥轮换

## 🔴 SEC-GW-002：AES密钥硬编码

### 📍 问题位置

```
文件路径：src/main/java/com/picchealth/utils/AESUtil.java
```

### 📖 学习理解

**问题原理**：AES是对称加密算法，加密和解密使用同一把密钥。

**学习要点**：
- 密钥泄露意味着所有数据都可以被解密
- 密钥应该存储在安全的配置中心
- 应该定期更换密钥降低风险
- 不同环境使用不同密钥

---

## 🔴 SEC-GW-003：SFTP密码硬编码

### 📍 问题位置

```
文件路径：src/main/java/com/picchealth/utils/SFTPUtils.java
```

### 📖 学习理解

**问题原理**：SFTP密码是服务器登录凭证，泄露后攻击者可以访问文件服务器。

**学习要点**：
- 密码不应该有默认值
- 应该从安全的配置中心获取
- SFTP账号应该有最小权限原则
- 应该定期更换密码

---

## 🔴 SEC-GW-004：CORS配置漏洞

### 📍 问题位置

```
文件路径：src/main/java/com/picchealth/config/CorsConfig.java
```

### 📖 学习理解

**问题原理**：CORS配置过于宽松会导致跨域安全风险。

**学习要点**：
- 应该配置允许的来源白名单
- 不应该使用`*`允许所有来源
- 应该限制允许的方法和头
- 应该配置凭证携带策略

---

## 🟠 SEC-GW-005：敏感Header全转发

### 📍 问题位置

```
文件路径：网关过滤器配置
```

### 📖 学习理解

**问题原理**：网关转发所有请求头可能泄露敏感信息。

**学习要点**：
- 应该过滤不必要的请求头
- 敏感头信息应该只在内部使用
- Authorization等头应该验证后再转发

---

## 📋 学习理解总结

### 硬编码敏感信息的危害

| 危害类型 | 说明 |
|---------|------|
| 代码泄露 | 代码被访问时，敏感信息也泄露 |
| 无法更换 | 更换密钥需要重新编译部署 |
| 环境混淆 | 测试和生产使用相同密钥 |
| 合规问题 | 违反金融行业安全规范 |

### 密钥安全管理的最佳实践

📖 **学习理解**：

1. **配置中心管理**：密钥存储在Apollo等配置中心
2. **环境隔离**：不同环境使用不同密钥
3. **定期轮换**：定期更换密钥降低风险
4. **权限控制**：限制密钥的访问权限
5. **加密存储**：密钥本身也应该加密存储
